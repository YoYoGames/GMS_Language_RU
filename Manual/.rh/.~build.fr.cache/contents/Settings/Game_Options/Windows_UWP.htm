Параметры игры вWindows UWP В этом разделе описаны различные доступные вам опции, которые определяют, как будут компилироваться ваши игровые проекты UWP (Universal Windows Platform). Различными разделами являются: Общий В общем разделе опций UWP можно задать информацию о продукте: Отображаемое имя: Указывает дружественное имя приложения, которое будет отображаться пользователям (эта строка может быть локализована). Имя пакета: Это имя созданного вами пакета, которое должно совпадать с именем, используемым для ключа разработчика, созданного вами для игры (см. раздел "Упаковка" ниже). Отображаемое имя пакета: Это "короткое имя" пакета, которое в идеале должно совпадать с отображаемым именем. Отображаемое имя издателя: Это имя вашей компании или человека, которого вы хотите ассоциировать как издателя игр. Версия: Номер версии вашей игры. После настройки вы можете выбрать различные ориентации , в которых может быть запущена ваша игра для наилучшего восприятия пользователем. Доступны следующие варианты: Ландшафт Портрет Перевернутый пейзаж Перевернутый портрет По умолчанию все они будут выбраны, и вам следует снять выделение с тех, которые вам не нужны. После установки параметров ориентации вы можете выбрать различные разрешения , которые потребуются вашему UWP-приложению, из следующих вариантов: Включить разрешение на доступ в Интернет: Помечает игру как имеющую доступ к Интернету. По умолчанию эта функция выключена. Включить разрешение микрофона: Помечает, что игра имеет доступ к микрофону устройства. По умолчанию эта функция выключена. Включить песочницу IAP: Помечает игру как использующую "песочницу" для покупок в приложениях. По умолчанию эта функция выключена. Семейства устройств Здесь вы можете выбрать, на какие устройства Families вы хотите нацелить экспорт Windows UWP при создании исполняемого файла. По по умолчанию отмечена опция "Поддерживать универсальное семейство устройств", что означает, что приложение будет поддерживать все устройства и семейства. Чтобы ограничить приложение поддержкой только определенных семейств устройств, следует снять флажок с опции "Универсальное" и установить флажки для соответствующих семейств устройств. соответствующие семейства устройств. Также можно поддерживать семейство устройств Universal, но ограничить другие семейства устройств только определенными версиями платформ - в этом случае нужно установить флажок для семейства устройств Universal, а также отметить другое семейство устройств, для которого требуется поддержка другой версии платформы, и ввести сведения о версии для этого семейства в соответствующие текстовые поля. Отображать курсор: Когда эта опция включена, в вашей игре будет отображаться курсор ОС по умолчанию. Start Fullscreen: Если флажок установлен, игра будет запускаться в полноэкранном режиме, в противном случае она будет запускаться в оконном режиме. По умолчанию эта опция выключена. Разрешить полноэкранное переключение: Если этот параметр установлен, пользователь может переключаться из полноэкранного режима в оконный и обратно, используя стандартные сочетания клавиш ОС Ubuntu. По умолчанию эта функция выключена. Масштабирование: Здесь вы можете выбрать сохранение соотношения сторон (таким образом, комната 4:3 будет иметь формат "letter boxed" на экране 16:9) или полное масштабирование (растягивание изображения на весь экран). ВНИМАНИЕ! Выключениеповерхности приложения отключает все опции масштабирования, установленные в Опциях игры Windows, до тех пор, пока она не будет снова включена. На сайте также есть кнопка Предварительный просмотр которая сгенерирует страницы текстур для этой платформы и затем откроет окно, чтобы вы могли посмотреть, как они выглядят. Это может быть очень полезно, если вы хотите увидеть, как структурированы текстурные страницы и предотвратить создание текстурных страниц большего (или меньшего) размера, чем необходимо. ПРИМЕЧАНИЕ: Имейте в виду, что чем больше размер страницы текстур, тем менее совместима ваша игра на ПК с более низкими техническими характеристиками. Изображения В первой части раздела "Изображения" у вас есть возможность установить различные плитки , которые ваша игра будет использовать на различных устройствах, на которых может работать UWP. Вы можете задать следующие параметры (и все они должны быть файлами формата *.png ): Маленькое изображение плитки - это изображение логотипа, которое появляется рядом с названием приложения в строке поиска и других областях пользовательского интерфейса. Его размер должен составлять 44 x 44 пикселя. Изображение средней плитки должно быть размером 71x71 пиксель. Следующее изображение плитки - это плитка с логотипом магазина , ее размер должен составлять 50 x 50 пикселей. Широкоеизображение плитки - это изображение, которое появляется, когда плитка имеет широкий формат и должно быть 310 x 150 пикселей. Если это изображение не предоставлено, плитка может отображаться только в квадратном формате и не может принимать уведомления основанные на широких типах шаблонов. Так как пользователь может решить, какой формат использует плитка, рекомендуется включить плитку с широким изображением. Также обратите внимание, что если предоставлено широкое изображение, плитка может изначально отображаться в широком формате. Большая плитка изображения представляет собой формат большой живой плитки и должна иметь размер 310 x 310 пикселей. Обратите внимание, что это будет использоваться только на настольных ОС. Упаковка GameMaker Studio 2 поставляет ключ разработчика по умолчанию для разработки UWP для Windows, но рекомендуется сгенерировать свой собственный и указать его здесь. Для этого создайте пустой проект в Visual Studio, затем сохраните в безопасном месте копию автоматически сгенерированного файла *.pfx, который будет включен в пустой проект (это и есть ключ разработчика). Затем вы должны связать этот файл *.pfx здесь. После ссылки на файл необходимо нажать на связанную с ним кнопку Install , а затем следовать инструкциям в открывшейся командной строке. Если вы этого не сделаете, то использование кнопок Run/Debug (или нажатие F5) будет неудачным и ваша игра не сможет быть протестирована на платформе UWP (однако, если вы используете Create Executable, процесс сертификационной установки будет запущен автоматически, если *.pfx еще не зарегистрирован). ПРИМЕЧАНИЕ: Когда дело дойдет до загрузки вашего приложения в магазин, вам нужно будет получить правильный *.pfx для вашего названия. Это делается через ассоциацию вашего приложения с Магазином Windows в Visual Studio, что заставляет Visual Studio генерировать новый *.pfx специально для приложения. Стоит также отметить, что вы можете связать любое приложение с Windows Store в Visual Studio, Это означает, что вы можете связать пустой проект с приложением Windows Store и просто скопировать сгенерированный *.pfx в безопасное место, которое затем можно использовать в UWP Game Options. Xbox Live Раздел Xbox Live в опциях UWP-игры - это место, где вы можете активировать свой UWP-проект для использования на консоли Xbox One. Активация проекта для Xbox позволяет вам делать следующее при запуске проекта на Xbox One: Вход в Xbox Live из приложения Использовать идентификатор пользователя Xbox Live каким-либо образом в проекте (например, показывать геймерский тег или фотографию геймера) Однако прежде чем вы сможете выполнить эти действия, вам необходимо убедиться, что вы нажали на флажок Включить Xbox Live, а если вы являетесь участником программы Microsoft Creators Program, вам также следует отметить опцию Creators Program App. Если вы участвуете в программе Creators Program, вам нужно будет добавить идентификатор названия и идентификатор конфигурации службы (более подробную информацию о них можно получить в компании Microsoft). Обратите внимание, что в настоящее время для тестирования функциональности Xbox Live в вашем проекте необходимо использовать Create Executable, а не Run, так как для вывода UWP Xbox необходимо использовать сертификат подписи Windows Store, который вы предоставили в качестве части Packaging (см. выше), а не временный сертификат, поставляемый с GameMaker Studio 2, который будет использоваться при нажатии кнопки Run. После включения XBox Live вы можете использовать функции XBox Live в своем проекте, но учтите, что для базовых функций вам нужно быть участником программы Creator Program, а для расширенных функций вам понадобится учетная запись ID@XBox.

Параметры игры HTML5 В этом разделе описаны различные доступные вам опции, которые определяют, как будут компилироваться ваши игровые проекты HTML5. Различными разделами являются: Общий В этом разделе есть поля для заполнения следующей информации о вашей игре: Заголовок браузера: Здесь вы можете указать текст, который будет использоваться для строки заголовка вкладки вашей игры в браузере. Версия: Номер версии для вашего проекта. Имя папки: Имя папки, в которой будут храниться файлы вашей HTML5-игры (по умолчанию "html5game"). Выходное имя: Это имя выходного HTML-файла, которое может быть изменено на любое другое, кроме установленного по умолчанию "index.html". Ниже полей с информацией об игре у вас также есть следующие опции: Вывод отладки в консоль: Это позволяет отправлять все отладочные данные в консоль JS вашего браузера. По умолчанию эта функция выключена. Отобразить курсор: Это отключает курсор мыши, когда он находится над холстом игры. По умолчанию эта функция включена. Отображение предупреждения "Running Outside Server": Если вы тестируете игру локально (т.е. нажимаете на индексный файл с вашего компьютера, а не с сервера - не рекомендуется), то вы обычно получаете предупреждение, информирующее вас о том, что некоторые вещи не будут работать так, как вы ожидаете, из-за безопасности браузера для локальных файлов. Вы можете отключить это предупреждение, сняв флажок. По умолчанию она включена. Ниже общих настроек находятся дополнительные параметры: Включенный файл index.html: Эта функция предназначена для добавления пользовательского файла "index.html", который будет использоваться вместо предопределенного файла, создаваемого GameMaker Studio 2 по умолчанию. Это может быть очень полезно, если у вас есть дизайн веб-сайта. или вы хотите включить определенные функции во все ваши HTML5-сборки. Используемый файл должен быть включен в проект игры как Included File. Расширение панели загрузки: Это позволит изменить стандартную панель загрузки на ту, которая была добавлена в ваш проект GameMaker Studio 2 в качестве расширения. Prepend output .js: Здесь вы можете дополнить вывод *.js любым дополнительным JavaScript, необходимым для вашей игры. Например, приложение Chrome Store может потребовать, чтобы вы добавили window.localStorage = undefined; в выходной JS, для чего вы будете использовать эту опцию. Аналитика В этом разделе вы можете добавить аналитику в свою игру, что позволит вам использовать определенные продукты сторонних производителей для отслеживания вашей игры и ее продажи или игры. Вы можете указать, использовать ли Flurry Analytics или Google Analytics для отслеживания вашей игры. В связи с характером этих функций, одновременно активной может быть только одна из них, и после того, как вы отметите Enable Flurry Analytics или Enable Google Analytics, вы также должны добавить присвоенный вам уникальный ID в соответствующую область. Информацию о функциях, которые вы можете использовать для отправки аналитических данных, смотрите здесь. Графика Здесь вы можете изменить следующие детали, связанные с тем, как будет отображаться ваша игра (обратите внимание, что не все браузеры будут отображать вашу игру одинаково, поэтому проверьте эти параметры на максимально возможном количестве браузеров, чтобы убедиться, что они дают желаемый эффект): Разрешить переключение в полноэкранный режим: Установка этого флажка позволит пользователю переключать игру из оконного режима в полноэкранный, используя / + "F" на Mac и F10 на Windows. Это включено по умолчанию. Интерполировать цвета между пикселями: Включает линейную интерполяцию, которая, по сути, "сглаживает" пиксели. Для четкой пиксельной графики этот параметр должен быть выключена, но если у вас есть красивые альфа-смешения и сглаженные края графики, ее лучше оставить включенной. По умолчанию эта функция включена. Центрировать игру в браузере: Установка этого флажка автоматически центрирует экран игры в браузере, в котором запущена игра. По умолчанию эта опция выключена. Использовать встроенные частицы: Эта опция экспортирует спрайты, необходимые для встроенной системы частиц, вместе с вашей игрой. Эти спрайты хранятся в отдельных страницах текстур, что означает, что они не являются наиболее эффективным методом генерирования частиц для этой платформы, поэтому рекомендуется (если не используется WebGL) снять флажок с этой опции и использовать свои собственные спрайты частиц, которые будут храниться на основных страницах текстур для вашей игры. Эта опция включена по умолчанию. Использовать встроенный шрифт: Эта опция экспортирует встроенный шрифт GameMaker Studio 2 вместе с остальными файлами игры. Это немного увеличит размер текстурных страниц и потребует небольшого количества дополнительной памяти и хранилища, поэтому, если ваша игра не использует встроенный шрифт Arial, эту опцию следует отключить. По умолчанию он включен. WebGL: Выберите, отключить ли поддержку WebGL, сделать ее необязательной (автоопределение) или сделать ее обязательным требованием для вашей игры. Следует отметить, что для тестирования производительности при любых обстоятельствах, вам следует отключить эту функцию, а затем обычно устанавливают автоопределение для публикации. Если сделать его обязательным требованием, это может значительно снизить совместимость вашей игры с различными типами браузеров. По умолчанию установлено автоопределение. Масштабирование: Ваша игра может быть настроена на автоматическое масштабирование холста рисунка для поддержания соотношения сторон в браузере, или вы можете выбрать полный масштаб. Вариант полного масштабирования не выводить игру на полный экран в браузере, а растягивать нарисованное, чтобы оно соответствовало размеру холста, определенному первой комнатой игры. По умолчанию это соотношение сторон сохраняется. Наконец, есть возможность задать размер страницы текстуры. По умолчанию (и наиболее совместимый) размер - 2048x2048, но вы можете выбрать любой размер от 256x256 до 8192x8192. На сайте также есть кнопка Предварительный просмотр которая сгенерирует страницы текстур для этой платформы и затем откроет окно, чтобы вы могли посмотреть, как они выглядят. Это может быть очень полезно, если вы хотите увидеть, как структурированы текстурные страницы и предотвратить создание текстурных страниц большего (или меньшего) размера, чем необходимо. Более подробную информацию о текстурных страницах можно найти здесь. ПРИМЕЧАНИЕ: Помните, что чем больше размер страницы текстуры, тем меньше совместимость вашей игры с различными браузерами и устройствами. Изображения В этом разделе вы можете определить Favicon и Splash Screen. Иконка должна быть создана в виде *.ico и может иметь размер 16x16, 32x32, 48x48 или 64x64 пикселя, глубину цвета 8 бит, 24 бит или 32 бит. Заставка должна быть размером с холст, на котором будет располагаться ваша игра, и она будет перекрывать любые если вы отметите опцию Use Splash Screen (т.е. у вас может быть загрузочная полоса ИЛИ заставка, но не обе). Разрешается использовать изображения меньшего или большего размера, но они будут масштабироваться под размер. Заставка должна быть изображением *.png. Стоит отметить, что в GameMaker Studio 2 есть инструмент Project Image Generator, который можно использовать для автоматического создания всех изображений, необходимых для различных целевых платформ, на которые компилируется ваша игра. для которых компилируется ваша игра. Если вы используете этот инструмент, вам следует пересмотреть созданные изображения, чтобы убедиться, что они соответствуют вашим требованиям. Социальная Социальная вкладка используется для настройки общения с Facebook в вашей игре. Если вам необходимы функции Facebook, вам необходимо установить флажок "Использовать Facebook", а затем ввести Facebook App ID и Facebook App Display Name. Обратите внимание, что доступ ко всем функциям Facebook осуществляется через расширение, и когда вы впервые отметите флажок " Использовать Facebook ", вам будет предложено загрузить и установить расширение Facebook со страницы Рынок (возможно, вам потребуется войти в свой аккаунтYoYo, прежде чем оно будет загружено).

Ключевые элементы управления После добавления дорожки параметров можно добавлять и удалять клавиши параметров к определенным кадрам для установки значения этого параметра. События Подобно обычным объектам, последовательности могут иметь события, которые могут запускать определенный код, назначенный им. Код назначается в виде скриптовой функции, которая может принимать никаких аргументов и будет вызвана при наступлении события. События добавляются нажатием кнопки Добавить событие , в результате чего откроется следующее окно: Здесь перечислены следующие события: Create - Это событие происходит при первом создании экземпляра последовательности, и это самое первое, что происходит в последовательности, помещенной в комнату через редактор комнат при входе в комнату. Уничтожение - Это событие выполняется, когда экземпляр последовательности уничтожается, и будет запущено до события Очистка (см. ниже). Clean Up - Это событие будет вызываться после любого события, которое удаляет экземпляр последовательности из комнаты. Таким образом, оно будет вызвано, если последовательность будет уничтожена, если комната закончится, или если игра закончится, и предназначено для выполнения любую задачу, которую вам нужно выполнить один раз, когда экземпляр последовательности удаляется из игры по любой причине. Если вы уничтожили экземпляр последовательности, используя, например, layer_sequence_destroy(), то это событие будет вызвано после события Destroy (см. выше). Begin Step / Step / End Step - Событие Step Event - это событие, которое проверяется на каждом шаге (кадре) игры, пока существует экземпляр последовательности, и состоит из трех частей: begin, step и конец. Для большинства вещей стандартное событие шага вполне подходит, но иногда вы хотите немного больше контроля над тем, какой код выполняется и в какое время, поэтому для этого вам предоставляются события шага Begin и End, и эти события всегда будут срабатывать в одном и том же порядке на каждом шаге (кадре) игры. Важно! Если последовательность поставлена на паузу, то эти события не сработают, а когда игра возобновится, они сработают на следующем кадре после того, как последовательность снова начнет воспроизводиться. Также обратите внимание, что порядок событий не зависит от направления головки воспроизведения, и даже если последовательность воспроизводится в обратном направлении, события все равно будут запускаться как Начало, Шаг и Конец. Асинхронное событие: Это эквивалент объекта Asynchronous System Event. Важно! Если последовательность приостановлена, то это событие не будет вызвано. Каждому событию можно назначить одну функцию, которая будет вызываться при срабатывании события. Вы можете назначить функцию с помощью поля ввода, а нажатие на кнопку со стрелкой откроет редактор сценария для редактирования функции. Вы также можете нажать кнопку Create New внизу, чтобы создать новый ресурс сценария с уже определенными и готовыми к заполнению "шаблонными" функциями. Обратите внимание, что вы можете изменить имена функций на любые, какие вам нужны, и необязательно использовать предопределенные имена, и вы также можете удалить все определения функций, которые вам не нужны. Важно отметить, что функции, используемые для событий последовательности, не могут принимать никаких аргументов. Последовательность событий также можно добавлять и редактировать с помощью кода. Добавление дорожек параметров Кнопку Add Parameter можно использовать для добавления одной дорожки параметров к выбранной дорожке актива. Как уже упоминалось ранее на этой странице, дорожки параметров дорожки параметров могут быть различных типов в зависимости от актива, к которому они относятся. Ниже приведен список всех доступных дорожек параметров и их назначение, а также активы, к которым они могут быть применены: Позиция: Этот параметр дорожки может быть добавлен к любому из четырех типов активов последовательности(спрайты, объекты, звуки или последовательности) и используется для установки пространственной позиции в последовательности для данного актива. актива. Он принимает значения X и Y. Вращение: Как и параметр положения, параметр вращения доступен для всех активов последовательности и может быть использован для установки вращения в градусах для актива. Следует помнить, что GameMaker Studio 2 рассчитывает вращение против часовой стрелки, начиная с 0° справа, поэтому 90° - это вверх, 180° - влево и 270° - вниз. Origin: Параметр origin доступен для активов спрайта, объекта и последовательности и может использоваться для установки положения X и Y начала актива. Изменение начала координат повлияет на то, как актив будет позиционирование, поворот и масштабирование. Масштаб: Параметр scale доступен для активов спрайтов, объектов и последовательностей и используется для установки горизонтального и вертикального масштаба актива. Это влияет на то, как актив будет отображаться в IDE и и вычисляется как скалярное значение, где 100% - это размер актива по умолчанию, 50% - половина размера, а 200% - двойной размер. Обратите внимание, что у вас есть возможность связать значения горизонтального и вертикального масштаба, нажав на кнопку кнопку "Связать" . Это означает, что изменение одного значения автоматически изменит и другое. Индекс изображения и скорость изображения: эти параметры доступны только для спрайтов и объектов и могут использоваться для изменения текущего кадра анимации (см. страницу об индексе_изображения для подробнее), а также текущую скорость анимации, которая задается с помощью FPS и работает так же, как значение, установленное для используемого спрайта в редакторе спрайтов. Умножение цвета: Этот параметр доступен только для спрайтов и объектов и может быть использован для изменения текущего цвета наложения для рисуемого спрайта. Это эквивалентно установке значения image_blend и любые цвета, заданные здесь, будут "умножены" (т.е. смешаны) с цветом спрайта, а значение white (#ffffffff) означает, что смешивания не должно происходить. Обратите внимание, что при работе с этим типом дорожки вы можете задать или щелкните на образце цвета, чтобы открыть GameMaker Studio 2 Colour Picker. Обратите внимание, что цвета задаются в формате ARGB, поэтому вы можете установить значение для альфа, используя этот тип параметра (Colour Picker также позволяет установить значение альфа). Это будет эквивалентно установке параметра image_alpha для спрайта. Громкость: Параметр громкости применяется только к дорожкам звукового актива и может использоваться для установки усиления звука (фактическое усиление звукового актива не будет изменено, так как это, по сути, эмиттер, который используется для дорожки актива в секвенции). дорожки в последовательностях). Значение громкости рассчитывается в процентах, где 0% - отсутствие громкости, а 100% - полная громкость. Pitch: Параметр pitch применяется только к дорожкам звукового актива и может использоваться для установки высоты звука (как и в случае с громкостью, фактическое усиление звукового актива не изменяется, поскольку это, по сути, эмиттер, который используется для дорожки актива в последовательностях). Значение высоты тона рассчитывается в процентах, 100% - это "естественная" высота тона звука, меньше 100% - понижение, больше 100% - повышение. Falloff: Параметр falloff применяется только к дорожкам звуковых активов и в настоящее время не реализован.

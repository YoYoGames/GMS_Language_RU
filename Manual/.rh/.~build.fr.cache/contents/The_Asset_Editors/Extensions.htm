Редактор расширений Пакет расширения может быть использован для добавления ряда дополнительных функций и константв язык GameMaker Language(GML) или как способ передачи игровых активов или даже целых проектов между людьми. Если расширение предназначено для добавления дополнительной функциональности, то используемые функции могут связать вашу игру с внешними SDKили исходным кодом, написанным на других языках (например, Java для Android), а также с родным языком GML. Как правило, сторонние расширения будут от GameMaker Marketplace и те, на которые вы подписались будут перечислены в окне Marketplace Library и могут быть установлены оттуда. Если вы не получили пакет расширений из Marketplace, вы все равно можете добавить его в проект, снова щелкнув правой кнопкой мыши в любом месте браузера активов и выбрав пункт Добавить существующее, а затем перейдя к местоположению расширения, чтобы выбрать его. Если у вас есть *.GMEZ, созданный в предыдущей версии GameMaker Studio, вы можете добавить их, выбрав правой кнопкой мыши пункт меню Import Extension. ПРИМЕЧАНИЕ: Устаревший формат расширений *.GEX больше не поддерживается и не может быть импортирован в проекты GameMaker Studio 2. После добавления расширения в Asset Browser необходимо дать ему имя и номер версии, а также указать, на какие платформы вы хотите его экспортировать. Сняв галочки с выбранных платформ, вы можете указать GameMaker Studio 2 не экспортировать расширение как часть исполняемого файла для этих платформ. Также необходимо убедиться, что свойства расширения соответствуют целевой платформе (платформам), и отредактировать цели экспорта для файлов расширения в соответствии со следующим: Все цели: Расширения GML. Это расширения, созданные с использованием языка GameMaker Language(GML) и совместимые со всеми целевыми платформами. Windows(Standard, UWP, XBoxOne): Вы можете использовать файлы windows DLL (Dynamic Link Libraries). Обратите внимание, что если вы установили флажок "Use x64 Windows Runtime " в опциях игры Windows , то все DLL для проекта должны быть совместимы с 64-битной системой. 32-разрядные DLL не будут работать с 64-разрядным исполняемым файлом. macOS: Цель macOS может использовать стандартные файлы DyLib для расширений. Ubuntu (Linux): Для цели Ubuntu вам необходимо убедиться, что ваше расширение использует файлы SO. HTML5: Эти платформы могут использовать расширения JavaScript. Android: Для этой платформы требуется, чтобы расширение было написано на Java или Android Framework SDK. iOS / tvOS: Расширения для этой платформы написаны на Objective C и требуют наличия файла *.mm и *.h, или, если используется какой-либо Frameworks, предварительно скомпилированного файла *.a native. PlayStation: Для PlayStation 3 или 4 ваши расширения должны быть файлами PRX, причем для каждой цели требуется свой PRX, чтобы соответствовать спецификации цели (PS3 или PS4). Если вы работаете над PSVita, расширение должно быть SUPRX. ПРИМЕЧАНИЕ: Некоторые расширения должны следовать определенным соглашениям об именовании, чтобы GameMaker мог правильно определить платформу и архитектуру процессора, для которых они предназначены. Более подробную информацию смотрите в разделе Создание расширений. Поскольку одно расширение может иметь различные связанные файлы, можно создать одно расширение для нескольких платформ, в этом случае необходимо указать все совместимые платформы в главном редакторе расширений (т.е. установить флажок и затем в свойствах отдельных файлов для каждой части расширения указать платформу, на которую эти файлы должны быть экспортированы. Чтобы изменить целевую платформу для определенного файла расширения или заполнителя, необходимо сначала открыть свойства расширения (дважды щелкните на расширении в браузере активов или используйте и выберите Открыть свойства), а затем в свойствах расширения выберите справа файл ресурса, который вы хотите изменить: ПРИМЕЧАНИЕ: Если вы экспортируете в целевую систему macOS или Ubuntu, то вы можете установить опцию "Uncompress as zip file", чтобы сохранить разрешения файлов для расширения. После установки расширения вы можете использовать функции и константы для него так же, как и встроенные функции и константы, используемые в языке GameMaker. Если ваша игра является кроссплатформенной (например, она предназначена для запуска как на платформе Windows и Android ), то вам следует использовать константы и функции ОС для проверки платформы перед вызовом функций, специфичных для расширения, чтобы избежать проблем, например: switch(os_type) { case os_android: call_android_extension_function(x, y, z); break; case os_windows: call_windows_extension_function(x, y, z); break; } Обратите внимание, что для разных платформ ваши расширения могут использовать общие имена функций, поэтому если у вас есть расширение с разными файлами для (например) всплывающего сообщения для Windows и для Android, оба файла расширения могут использовать одно и то же имя функции до тех пор. если они оба настроены на экспорт только на соответствующие платформы. Если вы хотите создать свои выигранные расширения, то основную необходимую информацию вы можете найти на следующих страницах: Создание расширения Расширения дляAndroid Расширения дляiOS / tvOS YoYo Games Расширения сторонних разработчиков В старых версиях GameMaker было встроено множество функций сторонних разработчиков, но это было неэффективно и трудно поддерживать, поскольку если какая-то конкретная функциональность была изменена сторонним разработчиком, то вся IDE GameMaker и Время выполнения пришлось бы обновлять. Поэтому в текущих версиях для добавления сторонней поддержки в ваши игры используются пакеты расширений. Это означает, что вы можете добавить только те дополнительные функции, которые вам необходимы, и даже можете адаптировать и редактировать их в соответствии со своими специфическими потребностями. Большинство из этих расширений будут загружены и установлены для вас при выборе определенных опций игры (например, Facebook или Google Mobile Ads), но вы можете найти дополнительную информацию о них в соответствующем разделе для каждой платформы в Базе знаний YoYo Games: Платформы и установкаGameMaker Studio 2 Сами расширения можно найти на следующей странице Marketplace: YoYo Games Marketplace Assets Обратите внимание, что эти расширения поставляются с демонстрацией и файлом справки (в формате *.pdf ), который объясняет, как их использовать, и поэтому их функции не рассматриваются в руководстве.

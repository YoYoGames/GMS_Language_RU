Порядок проведения мероприятий Рассматривая события в GameMaker Studio 2, следует отметить, что точный порядок, в котором ВСЕ события будут происходить на каждом этапе, не может быть четко определен, просто потому, что это зависит от внутренней работы GameMaker Studio2, и это может быть изменено в процессе развития программы. 2, а это может меняться по мере развития программы. Однако есть определенные события, которые всегда будут происходить в одном и том же порядке. Первый набор событий, которые всегда будут происходить одинаково, - это события, происходящие при первом входе в комнату. Различные события будут происходить в следующем порядке: Переменные объекта / переменные экземпляра инициализируются (это переменные, которые вы можете определить в IDE редактора объектов или редактора комнат). Если у вас есть объект с переменными, то они будут созданы до всего остального, а затем все экземпляры, которые имеют переменные, будут определены после, так что вы можете перезаписать переменные объекта переменными конкретного экземпляра в редакторе комнаты. редактор. событиеCreate , за которым следует код создания экземпляра каждого экземпляра - таким образом, по мере создания каждого экземпляра будет выполняться сначала событие Create, а затем код создания экземпляра прежде чем перейти к следующему создаваемому экземпляру. Событие начала игры - это событие срабатывает один раз в самой первой комнате игры для всех экземпляров, помещенных в комнату из редактора комнат (обратите внимание, что вызов функции game_restart() вызовет это событие снова. Код создания комнаты - это одноразовый код, написанный в редакторе комнат для первого входа в комнату. Событие начала комнаты для всех экземпляров - одно из событий категории "другие", которое будет происходить для всех экземпляров, постоянных или иных Стоит отметить, что вы также можете установить порядок создания конкретных экземпляров в самом редакторе помещений, перемещая их вверх или вниз по списку в окне Instance Layer Properties. Кроме этих конкретных событий, единственный известный порядок, который всегда будет происходить одинаково, независимо ни от чего, относится к событиям трех шагов и различным событиям жеребьевки . Они всегда остаются неизменными, поэтому если у вас есть код, который зависит от конкретного времени на каждом этапе игры, вам следует использовать их: Событие начала шага Событие шага (обратите внимание, что событие шага выполняется непосредственно перед тем, как экземпляры будут помещены в свои новые позиции) Событие конца шага Все события для рисования также всегда обрабатываются в следующем порядке (за исключением события Window Resize , которое запускается по-другому): Мероприятие перед жеребьевкой Событие начала рисования Ничейное событие Событие окончания рисования Событие после жеребьевки Событие начала рисования графического интерфейса Событие рисования графического интерфейса Конечное событие Draw GUI Назад: События объекта Следующее: Другие события © Copyright YoYo Games Ltd. 2021 Все права защищены

Сохранить / Загрузить Это событие будет вызвано обратным вызовом некоторых функций, связанных с загрузкой и сохранением буферов в файлы, а также при загрузке или выгрузке аудио из памяти. Само событие будет содержать встроенную async_load DS map, которая будет заполнена ключами, необходимыми для конкретной функции. Они перечислены в разделах ниже. Буферы Когда вы используете функции buffer_save_async() или buffer_load_async() асинхронное событие будет инициировано, когда передача данных будет завершена. Это событие заполнит карту async_load следующими парами ключ/значение "id": идентификатор асинхронной функции, возвращаемый используемой функцией. "status": вернет true, если данные были сохранены/загружены правильно, и false в противном случае. Это позволяет опросить ход сохранения/загрузки и вывести сообщение или сменить комнату и т.д. по завершении процесса. Аудиогруппы При работе с аудиогруппами вы можете загружать и выгружать их из памяти с помощью функций audio_group_load() и audio_group_unload(). При использовании функции load это событие сработает, когда полный набор аудиофайлов, установленных для группы, будет загружен в память и заполнит карту следующими парами ключей файлов, установленных для группы, был загружен в память и заполнит карту следующими парами ключ/значение: "type": это говорит нам о типе вызываемого события и будет "audiogroup_load" для загрузки аудио. "group_id": вернет идентификатор аудиогруппы, которая была загружена (как определено в редакторе аудиогрупп). Когда все аудио загружено для группы, срабатывает это событие, и его можно использовать для смены комнат, воспроизведения музыкальной дорожки и т.д....

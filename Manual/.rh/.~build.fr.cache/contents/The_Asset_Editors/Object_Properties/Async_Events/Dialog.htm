Диалог Асинхронное событие Dialog срабатывает только тогда, когда оно получает обратный вызов от одной из специальных асинхронных диалоговых функций, например get_login_async() (пожалуйста см. эту функцию для расширенного примера кода использования этого события). Эти события - те, которые запрашивают какой-либо тип пользовательского ввода, который может быть именем, данными для входа, числом или цветом и т.д.. Поскольку большинство устройств не любят зацикливаться в ожидании ответа, они должны быть асинхронными, и GameMaker Studio 2 будет продолжать работать в фоновом режиме, пока в этих функциях открыт диалог и пока они не получат требуемый пользовательский ввод, который вызовет это событие. Это означает, что если диалог открыт во время выполнения игры, вам следует принять меры для приостановки игры до тех пор. пока не будет получен обратный вызов диалога. Снова возвращается карта DS Map с идентификатором, хранящимся в специальной переменной async_load. Значения, хранящиеся в этой карте, зависят от используемой функции, и для получения более подробной информации следует обратиться к отдельным записям по каждой функции в данном руководстве. ПРИМЕЧАНИЕ: Функции асинхронных диалогов предназначены только для отладки и тестированияи не должны использоваться в выпущенных играх. Для этих целей вы должны создать пользовательский интерфейс для получения ввода от игроков, чтобы иметь полный контроль над тем, как выглядят и ведут себя диалоги.

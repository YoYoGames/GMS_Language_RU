ПредпочтенияAndroid Android предпочтения разделены на различные разделы, перечисленные ниже: В верхней части настроек вы можете установить максимальный размер кучи памяти JDK (по умолчанию 2 ГБ). Далее необходимо настроить различные пути , чтобы GameMaker Studio 2 знала, где искать различные инструменты, необходимые для создания конечного исполняемого пакета и тестирования игры. Это разделено на три раздела для Android SDK, Android NDK и Java JDK. Эти пути должны были быть заполнены автоматически, но если у вас возникнут проблемы, вы можете нажать кнопку справа, чтобы открыть проводник файлов и вручную выбрать местоположение каждого элемента. Подробную информацию о том, какие версии требуются для каждого из элементов SDK, можно найти здесь. Пути должны проверяться по мере их применения и сообщение покажет " Found" или сообщит вам о каких-либо ошибках (в этом случае вам следует пересмотреть всю информацию, чтобы убедиться в ее правильности). Ниже представлены различные варианты упаковки: Всегда выполнять полную установку APK: Отметив этот флажок, GameMaker Studio 2 удалит все предыдущие версии вашей игры с вашего устройства, а также все связанные с ней файлы (например, *.ini ) с вашего тестового устройства перед установкой новой версии игры. Установить на пакет: Когда эта опция отмечена, компиляция конечного исполняемого пакета *.apk также автоматически установит его на выбранное в данный момент целевое устройство. Наконец, мы переходим к хранилищу ключей. Это файл, который будет использоваться для "подписи" всех ваших приложений Android , поэтому заполните все данные правильно и помните, что этот файл будет необходим для создания и обновления всех ваших приложений Android в будущем, что означает, что после установки . рекомендуется сделать скриншот используемых настроек и создать резервную копию конечного файла.. Если вы потеряете этот файл, вы не сможете обновить ни одну из существующие игры, которые были загружены в магазин. Для создания нового файла keystore необходимы следующие данные: Filename: Это имя файла хранилища ключей. ПРИМЕЧАНИЕ: Это не ваше имя или название компании, а имя генерируемого файла! Если у вас уже есть файл keystore из предыдущих проектов, вы можете нажать здесь кнопку браузера файлов и направить GameMaker Studio 2 к ранее созданному файлу keystore (или нажать кнопку Import ). В этом случае вы все равно должны заполнить остальную необходимую информацию (точно так же, как это было при первом создании файла), но вы не должны нажимать кнопку Generate Keystore ! Это приведет к созданию нового файла keystore, который перезапишет предыдущий. Общее имя: Обычно это ваше имя. Пароль: Ваш пароль безопасности для файла keystore, который должен состоять не менее чем из шести символов. Псевдоним: Это другое имя, которое может быть использовано для файла keystore и должно отличаться от имени, введенного выше. Alias Password: Ваш псевдонимный пароль безопасности для файла keystore, который должен состоять не менее чем из шести символов. Организационное подразделение: Отдел в компании, частью которого вы являетесь. Организация: Название вашей компании. Местонахождение: Название города или населенного пункта, в котором вы находитесь. Страна: Стандартный двухбуквенный код страны, в которой вы находитесь. Как только эта информация будет заполнена и вас все устроит, нажмите кнопку с надписью Generate Keystore for GameMaker Studio 2, чтобы создать необходимый файл keystore. Это может занять некоторое время, но после завершения процесса обычно не требуется не нужно будет менять его снова. Обратите внимание, что вы можете нажать кнопку Import , чтобы импортировать ранее созданный файл keystore (см. раздел Filename , выше), а также получить хэш SHA1 файла keystore, нажав кнопку Generate Key Hash . При нажатии этой кнопки будут сгенерированы два специальных хэш-ключа безопасности, непосредственно связанных с вашим файлом keystore, которые требуются некоторым веб-сайтам для корректной работы (например, Facebook или Google Play для хэша Sha1). Если вам это понадобится, просто нажмите кнопку а затем скопируйте и вставьте сгенерированные хэш-ключи.

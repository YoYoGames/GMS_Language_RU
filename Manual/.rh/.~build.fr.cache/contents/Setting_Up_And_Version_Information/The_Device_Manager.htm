Диспетчер устройств Доступ к диспетчеру устройств осуществляется нажатием на кнопку с карандашом в списке целей: При нажатии на эту кнопку откроется Диспетчер устройств, где вы можете настроить различные устройства для целевых платформ, на которых вы хотите компилировать и тестировать на них (обратите внимание, что доступные платформы будут зависеть от имеющейся у вас Лицензии, и не все из них могут быть видны). Ниже вы найдете подробную информацию о настройке устройств для каждой из целевых платформ: Android Здесь вы можете нажать на кнопку Добавить новое устройство, чтобы добавить новое устройство Android в список вручную, или вы можете нажать кнопку Detect Device, чтобы GameMaker Studio 2 автоматически определила все устройства, подключенные через проводное соединение. Вы также можете настроить Android Virtual Device, нажав на кнопку с надписью Запустить AVD. Откроется менеджер виртуальных устройств Android, где можно создать новое AVD (см. здесь для получения дополнительной информации). После того как устройство будет создано, его нужно запустить из менеджера AVD, а затем в GameMaker Studio 2 нажать на кнопку Detect Device (Определить устройство ), и оно должно быть автоматически добавлено в список устройств: После добавления устройства (физического или виртуального), вы можете проверить соединение с помощью кнопки Проверить соединение или удалить его из списка, нажав кнопку Удалить устройство. HTML5 Менеджер устройств HTML5 предназначен для добавления различных браузеров в список устройств для таргетинга при запуске. Для начала вам следует нажать кнопку Detect Browsers, что приведет к созданию устройств для каждого из установленных на вашем компьютере браузеров. Если вашего браузера нет в списке, вы также можете нажать кнопку Add New Browser, а затем дать устройству имя и указать GameMaker Studio 2 путь к исполняемому файлу браузера. Дополнительные доступные поля могут быть настроены для управления тем, как браузер запускает ваши проекты, с помощью следующих опций: Аргументы командной строки браузера: Здесь вы можете добавить любые аргументы командной строки, которые будут добавлены к пути браузера. Вы можете найти список этих аргументов в следующих ссылках на основные популярные браузеры: Chromium (Chrome) Firefox Internet Explorer Адрес веб-сервера: Адрес веб-сервера сервера (по умолчанию 127.0.0.1). Вам может понадобиться изменить этот параметр в зависимости от спецификаций различных API, например, Facebook потребует, чтобы этот параметр был установлен на http://localhost (и затем внести его в белый список в консоли приложений). Порт веб-сервера: установите номер порта для веб-сервера (по умолчанию 51264). Разрешенные IP-адреса веб-сервера / диапазоны CIDR IP: Установите разрешенный веб-сервер, имена хостов и диапазоны CIDR IP при необходимости. iOS Прежде чем настраивать устройства iOS, необходимо настроить настроить устройство Mac и проверить его подключение (подробности см. ниже). После этого вы можете выбрать Mac, который вы хотите настроить, из выпадающего меню в верхней части окна, а затем нажать кнопку Detect Device, чтобы GameMaker Studio 2 автоматически нашла все подключенные устройства iOS, включая симуляторы. После подключения информация об устройстве будет показана в списке в этом окне, и вы можете изменить имя устройства, чтобы сделать его более удобным для пользователя (это изменение влияет только на то, как устройство будет представлено в диспетчере целей, и ни на что другое). tvOS Прежде чем настраивать устройства tvOS, необходимо настроить устройство Mac и проверить его подключение (подробности см. ниже). После этого вы можете выбрать Mac, который вы хотите настроить, из выпадающего меню в верхней части окна, а затем нажать кнопку Detect Device, чтобы GameMaker Studio 2 автоматически нашла все подключенные устройства tvOS, включая симуляторы. После подключения информация об устройстве будет показана в списке в этом окне, и вы можете изменить имя устройства, чтобы сделать его более удобным для пользователя (это изменение влияет только на то, как устройство будет представлено в диспетчере целей, и ни на что другое). macOS Чтобы добавить устройство Mac в менеджер, необходимо сначала нажать на кнопку на кнопку с надписью Добавить новое устройство, которая добавит устройство и создаст несколько пустых полей, которые необходимо заполнить для включения соединения. Сначала вам необходимо дать имя устройству, чтобы его можно было идентифицировать в списке целей, а затем указать имя хоста (или IP) компьютера Mac. Затем необходимо указать имя пользователя , с которым вы хотите установить связь, и пароль , который он использовал. При желании вы также можете изменить Install Folder на компьютере Mac на то место, где вы предпочитаете, чтобы GameMaker Studio 2 хранила файлы сборки ваших проектов. Когда все будет готово, можно нажать кнопку Проверить соединение, чтобы убедиться, что GameMaker Studio 2 может взаимодействовать с компьютером Mac. ПРИМЕЧАНИЕ: Если вы разрабатываете для iOS, вы должны сначала сделать это и установить рабочее соединение с Mac, который вы хотите использовать для разработки. ПРИМЕЧАНИЕ: Вы должны быть включены Удаленный вход на целевом компьютере mac. Это можно сделать, перейдя в Предпочтения -> Общий доступ и Включить удаленный вход для пользователя, которому вы хотите предоставить возможность входа. Ubuntu(Linux) Чтобы добавить устройство Ubuntu(Linux) в менеджер, необходимо сначала нажать на кнопку " Добавить новое устройство ". добавит устройство и создаст несколько пустых полей, которые необходимо заполнить для включения соединения. Сначала нужно назвать устройство, чтобы его можно было идентифицировать в списке целей, а затем указать имя хоста (или IP) машины Ubuntu (или виртуальной машины). Затем вы должны указать имя пользователя , на которого вы хотите связать с ним, и пароль , который он использовал. Также необходимо указать папку установки на машине Ubuntu, которую вы хотите использовать для установки ваших проектов. Когда все это будет сделано, вы можете нажать на кнопку Test Connection, чтобы убедиться, что GameMaker Studio 2 может взаимодействовать с машиной Ubuntu. Обратите внимание, что конечный исполняемый файл, созданный GameMaker, будет зависеть от архитектуры устройства, на которое он нацелен, которая может быть либо x64, либо ARM. Например, если вы подключите Raspberry Pi 3 или 4, созданный исполняемый файл будет 32-разрядным ARM-исполняемым файлом, а если вы подключите ПК с Linux, то это будет 64-разрядный x64-исполняемый файл. Windows UWP Прежде чем добавить устройство для цели Windows 10 UWP, необходимо включить режим разработчика и настроить портал устройств на устройстве. Если вы этого не сделаете, GameMaker Studio 2 не сможет взаимодействовать с устройством и не сможет правильно создавать UWP-приложения. Самый простой способ включить режим разработчика - это просто ввести в поиске Windows " Режим разработчика", а затем щелкнуть на открывшемся окне "Настройки". Откроется следующее окно: Вам нужно выбрать опцию Режим разработчика (что может потребовать загрузки и установить некоторые дополнительные пакеты), а затем выбрать расположенную под ним опцию Enable Device Portal. Портал устройства - это то, что позволит GameMaker Studio 2 взаимодействовать с ПК с Windows 10, и когда он будет включен, вам потребуется указать имя пользователя и пароль: Обратите внимание, что имя пользователя и пароль должны быть уникальными для портала и не должны быть теми, которые используются в вашей учетной записи Microsoft или другими учетными данными Windows. После ввода этой информации нажмите кнопку Apply (Применить ), затем закройте менеджер настроек и вернитесь в GameMaker Studio 2. Для получения дополнительной информации о Портале устройств см. статью Microsoft: Портал устройств для настольных компьютеров. Теперь вы можете открыть диспетчер устройств в GameMaker Studio 2 и настроить устройство (или устройства) для использования: Чтобы добавить устройство, совместимое с Windows 10 UWP, в менеджер, необходимо сначала нажать на кнопку " Добавить новое устройство ". добавит устройство и создаст несколько пустых полей, которые необходимо заполнить для включения соединения. Сначала необходимо назвать устройство, чтобы его можно было идентифицировать в списке целей, а затем указать хост - это IP-адрес устройства Windows 10 в сети, на котором будет размещена игра, с суффиксом " :50443" для задания номера порта. После этого добавьте имя пользователя и пароль , которые вы установили в Device Portal (как упоминалось ранее). Вы можете оставить PIN-код пустым, так как он не требуется для UWP Windows 10 Desktop. Более подробную информацию о настройке для различных выходов UWP (Desktop, XboxOne) можно найти в службе поддержки YYG: Настройка UWP. В нижней части всех окон различных устройств имеется опция Очистить устройства, при нажатии на которую все подключенные устройства будут удалены из списка.

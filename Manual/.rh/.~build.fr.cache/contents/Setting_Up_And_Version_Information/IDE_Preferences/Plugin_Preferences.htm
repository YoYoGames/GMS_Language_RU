Предпочтения плагина Параметры плагинов используются для управления любыми дополнительными плагинами, которые были добавлены в IDE GameMaker Studio 2. По умолчанию здесь есть один раздел, посвященный плагину Source Control для интеграции проектов GameMaker Studio 2 с Git. Контроль исходных текстов(Git) В этом разделе рассматриваются настройки, которые необходимо установить, чтобы получить Git SCM плагина для работы с GameMaker Studio 2. Во-первых, у вас есть набор G универсальных опций: Автоматически открывать окно вывода элементов управления исходным кодом: Если этот флажок установлен, окно вывода контроля исходных текстов будет автоматически открываться при изменении чего-либо. По умолчанию эта опция включена. Verbose Logging: Включите ведение подробного журнала для выходного журнала Git. По умолчанию эта функция выключена. Далее следует Раздел " Идентификация пользователя " со следующими полями, которые необходимо заполнить: Имя пользователя для фиксации: Здесь вы должны ввести имя пользователя, которое вы будете использовать для фиксации в Git (оно должно быть таким же, как имя пользователя, указанное при настройке вашего Git-репозитория ). Электронная почта для фиксации: Электронная почта, связанная с этим Git-идентификатором. После настройки личности пользователя вы можете добавить новую комбинацию имени пользователя/пароля для аутентификации, нажав на кнопку с надписью Add new User/Pass authentication, в результате чего откроется следующее окно с различными полями для заполнения: Вы также можете Добавить новую пару ключей SSH для аутентификации, нажав на кнопку с этой надписью. В результате откроется следующее окно для заполнения: Ниже этих кнопок вы можете настроить инструмент Инструмент слияния и Diff Tool для SCM с помощью этих опций: Путь к инструменту слияния: Здесь вы указываете полный путь к файлу инструмента слияния Git. Опции для инструмента слияния: В этом поле вы можете добавить дополнительные команды, которые будут выполняться при использовании инструмента слияния. Path to Diff Tool: Здесь вы указываете полный путь к файлу для инструмента Git diff. Опции для инструмента Diff Tool: В этом поле вы можете добавить дополнительные команды, которые будут выполняться при использовании инструмента Diff Tool. Наконец, после их установки у вас остается последний вариант: Автоматически включать контроль исходных текстов: Если этот флажок установлен, контроль исходных текстов будет автоматически включаться для всех новых проектов.

Предпочтения компиляции Параметры компиляции касаются определенных опций, связанных с выводом результатов при компиляции вашей игры для тестирования или создания пакета, а также определенных областей процесса компиляции. Опции включают в себя: Автоматически открывать окно синтаксической ошибки при ее обнаружении: Включить или отключить уведомление о синтаксических ошибках при компиляции. По умолчанию этот параметр выключен. Показывать окно компиляции во время компиляции: По умолчанию окно компиляции будет показано при компиляции ваших проектов, даже если документ компиляции закрыт (он будет открыт для компиляции). Если снять флажок, это поведение будет отключено и окно компиляции будет отображаться, если оно открыто. Показывать команды компилятора в выводе компиляции: Этот флажок установлен по умолчанию и заставляет компилятор показывать все соответствующие команды в окне компиляции. Если снять этот флажок, вывод будет меньше и содержать меньше информации. Показывать подробный вывод компилятора: Установка этого флажка заставит компилятор показывать дополнительные команды, связанные с процессом компиляции. Обычно это требуется только при отладке и может быть запрошено службой поддержки, если у вас возникнут проблемы. По умолчанию он выключен (и это рекомендуемая настройка). Use Subst for drive paths: Если этот параметр отмечен, GameMaker Studio 2 будет создавать - и удалять снова, когда это больше не требуется - виртуальные диски для использования в качестве замены полных путей к файлам. Эти виртуальные диски обходят любую потенциальную проблему слишком длинных для ОС путей к файлам (особенно при компиляции для Android или использовании расширений), поэтому рекомендуется включить эту опцию. Однако если у вас возникают проблемы с созданием виртуальных дисков в GameMaker Studio 2, вы можете отключить эту опцию. вы можете отключить эту опцию. По умолчанию эта опция включена. Показывать пакет в проводнике по завершении сборки: Установка этого флажка заставит GameMaker Studio 2 открыть окно проводника ОС в том месте, где был создан окончательный исполняемый пакет. По умолчанию этот параметр включен. Очищать окно вывода при запуске компиляции: По умолчанию окно компилятора будет очищать все предыдущие сообщения из окна вывода компиляции в начале каждой сборки, однако снятие флажка сохранит предыдущие журналы компиляции. Это включено по умолчанию и является рекомендуемой конфигурацией, поскольку большие журналы компилятора могут замедлить время компиляции. Максимальное количество параллельных заданий компиляции: Это значение устанавливает максимальное количество ядер процессора для компиляции проекта. Значение по умолчанию равно 8, и обычно его не нужно изменять, но если вы работаете на ноутбуке или менее мощной машине, вы можете уменьшить это значение. вы можете уменьшить это значение.

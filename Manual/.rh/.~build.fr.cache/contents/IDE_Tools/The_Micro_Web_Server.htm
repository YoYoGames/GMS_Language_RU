Micro Web Server Основной функцией этого небольшого инструмента является подключение и передача файлов на некоторые из доступных целевых платформ для GameMaker Studio 2 (например, на HTML5). Когда вы запускаете игру на целевой платформе, для которой требуется Micro Web Server, у вас появится откроется новое окно, когда игра будет передана на выбранное устройство: ПРИМЕЧАНИЕ: При первом запуске игры на объекте, требующем Micro Web Server, возможно появление предупреждения брандмауэра. В этом случае следует разрешить веб-серверу взаимодействовать через брандмауэр. Веб-сервер покажет определенное количество отладочной информации, связанной с файлами, которые были обслужены, и имеет кнопку для ее очистки в нижней части. Рядом с кнопкой "Очистить" указан путь к корневому JS-файлу проекта, а под ним находится параметр " Скорость сети ". По умолчанию она установлена на "Без ограничений", но нажатие на нее открывает список различных сетевых опций: Изменение этого параметра также изменит параметр Размер пакета вправо. Вы также можете отредактировать этот параметр вручную, в этом случае для параметра Скорость сети будет установлено значение "Пользовательская". Установка значения, отличного от максимального, эффективно снизит скорость передачи данных по сети и позволит вам тестировать игры в среде с низкой пропускной способностью. Вы также можете увидеть список разрешенных IP-адресов вместе с номером используемого порта , но их нельзя редактировать в Micro Web Server, и вместо этого они должны быть установлены в настройках диспетчера устройств для данного целевого устройства.

Настройка плагина GIT Когда мы начинаем использовать плагин GIT, нам нужно установить идентификатор, под которым GIT будет делать коммит, а это значит, что нам нужно перейти в раздел Плагины - Контроль исходных текстов (Git ) в Настройках и добавить данные для аутентификации. Этот идентификатор будет использоваться для всех будущих проектов, и GameMaker Studio 2 предоставляет вам два различных способа его установки. установить: Имя пользователя и пароль: Нажав на кнопку "Добавитьнового пользователя/пасс аутентификации", вы увидите следующее окно, в котором можно ввести URL репозитория , имя пользователя и пароль. пароль: Если вы хотите, чтобы это повлияло только на определенное хранилище, то введите URL хранилища в верхнее поле. Однако, если вы хотите вы хотите аутентификацию по всему домену (например, "bitbucket.org" или "github.com"), то просто укажите домен и ничего больше. Когда что-то должно проверить аутентификацию, он будет искать сначала соответствие определенному хранилищу, а затем соответствие домену. Это позволит вам установить аутентификацию по умолчанию для домена, а затем переопределить ее с помощью специфических деталей для определенных хранилищ. ПРИМЕЧАНИЕ: Если вы используете внешний репозиторий, то вы должны использовать имя пользователя и пароль, связанные с учетной записью, настроенной для этого репозитория, но если вы хотите использовать локальный репозиторий, то вы можете использовать любое имя пользователя и пароль. Пары ключей SSH: Нажав на кнопку "Добавитьновую аутентификацию пары ключей SSH", вы увидите следующее окно, в котором можно ввести URL репозитория и пароль, а затем указать пути к файлам открытого и закрытого ключей. Как и при аутентификации по имени пользователя/паролю, при использовании пар ключей SSH можно использовать аутентификацию по домену и репозиторию. Если у вас уже есть SSH Keypair, вы можете задать здесь пути, а также пароль, необходимый для доступа к ним. Обратите внимание, что если вам нужен быстрый доступ к открытому ключу, кнопка копирования автоматически скопирует его содержимое в буфер обмена. Если у вас еще нет пары ключей SSH, нажатие на кнопку Create KeyPair вызовет диалоговое окно файла, в котором будет указано, где разместить закрытый ключ, и рядом с ним будет создан открытый ключ - при этом пути к открытому и закрытому ключам будут заполнены автоматически. Эти настройки теперь будут использоваться для связи с репозиторием (IE: Git Hub, Bit Bucket и т.д.) при обработке запросов на клонирование, push и pull. Однако вам все еще необходимо настроить параметры для каждого проекта, что можно сделать, открыв Основные параметры и нажав на опцию Включить контроль исходных текстов. Это активирует SCM для текущего проекта. После включения контроля исходников (и его применения) в верхней части IDE появится новое контекстное меню со следующими опциями (каждая из которых подробно описана в остальных разделах руководства). каждая из которых более подробно описана в остальных разделах этого руководства): ПРИМЕЧАНИЕ: Если вам нужно снова отредактировать данные аутентификации, просто дважды щелкните на записи, и откроется окно с деталями для редактирования, но вы не сможете переименовать URL. Если вам нужно изменить URL, вам нужно будет удалить данные аутентификации и добавить их заново. Чтобы удалить данные аутентификации, нажмите на кнопку закрытия слева от строки в главном окне Предпочтений.

Параметры репозитория После настройки учетных данных GIT и включения функции Source Control для вашего проекта, вам необходимо связать IDE GameMaker Studio 2 с репозиторием, что делается одним из трех способов: Создание репозитория проекта Вставьте текущий проект во внешний репозиторий Клонирование существующего репозитория В первом случае, при создании репозитория проекта, будет создан GIT-репозиторий непосредственно там, где находится ваш проект. На языке GIT это "непустойрепозиторий", поэтому Push/Pull в данном случае не имеют смысла. не имеют смысла в этом случае, но вы все равно можете фиксировать и отслеживать изменения проекта, а также возвращать изменения и ревизии, если это необходимо. Чтобы настроить его, вам просто нужно выбрать Source Control > Create Project Repository в главном меню. Однако, возможно, у вас уже есть существующий проект, и вы хотите поместить его во внешнее хранилище для отслеживания и совместного использования. среди вашей команды. Для этого необходимо, чтобы внешний репозиторий был уже создан, а также чтобы в настройках идентификации были заданы правильные имя пользователя и пароль для доступа к нему. Если у вас все готово, нажмите Source Control > Import Project into Repository и заполните URL: Наконец, в качестве удобной функции вы можете клонировать внешний репозиторий через IDE. Опять же, вам понадобится имя пользователя и пароль для хранилища, к которому вы собираетесь подключиться, установленные в ваших настройках идентификации, а затем это просто нажмите Source Control > Clone Repository. Заполните URL (используя вариант HTTPS, а не SSH) и укажите, куда вы хотите его отправить: После завершения клонирования автоматически откроется браузер файлов, чтобы вы могли открыть проект, если захотите.

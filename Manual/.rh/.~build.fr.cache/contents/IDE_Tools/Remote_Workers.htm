Удаленные работники Когда вы компилируете свою игру, вы обычно используете рабочий "Local", то есть компьютер, на котором установлена GameMaker Studio 2. Однако при некоторых обстоятельствах вам может понадобиться использовать другой компьютер для сборки ваших проектов. Например, если вы работаете на Mac и хотите создать проект для Windows, то вам нужно настроить удаленного рабочего и поручить сборку компьютеру с Windows. По сути, когда вы используете удаленного работника, GameMaker Studio 2 будет отправлять все все необходимые файлы с локальной машины, на которой она запущена, на машину удаленного работника, а затем использует эту машину для создания окончательного исполняемого файла проекта (либо сохраняя его на удаленной машине, либо возвращая его на локальную машину). Однако прежде чем вы сможете воспользоваться этой функцией, вам необходимо установить определенные инструменты на машину сборки и выполнить некоторые настройки, о которых мы расскажем здесь. ВАЖНО! Вы можете настроить удаленного работника между локальным Windows PC и удаленным Windows PC, или между локальным Mac PC и удаленным Windows PC, но в настоящее время вы не можете настроить локальный Windows (или Mac) PC и удаленный Mac PC, т.е. вы не можете иметь удаленного работника Mac, только Windows. Начальная настройка Чтобы настроить удаленного работника, необходимо сначала установить приложение инструмента сборки Remote Worker на компьютер, который будет использоваться. Вы можете найти этот инструмент, зайдя в меню Build Menu в GameMaker Studio 2 на локальной машине и выбрав опцию "ShowRemote Worker Installer": В качестве альтернативы вы можете щелкнуть значок "глаз" в диспетчере целей и получить установщик Remote Worker таким образом: Программа установки будет называться примерно так: "GameMakerStudio-Remote-Installer-X.X.X.XXX.exe". и вам нужно скопировать его на компьютер, который вы хотите использовать в качестве удаленного рабочего (вам не нужно устанавливать GameMaker Studio 2 на удаленном рабочем, только эту программу). После того как вы скопируете его на ПК сборки, установите его, следуя подсказкам на экране, и по завершении работы программы установки ОС может запросить доступ через брандмауэр к вашей сети. После того, как вы предоставите доступ, программа установки откроет страницу в вашем веб-браузере, и вы должны записать IP-адрес и номер порта, указанные на этой странице, поскольку они понадобятся вам на следующем этапе. Теперь вам нужно вернуться на машину, где установлена GameMaker Studio 2 и щелкнуть на кнопке Target Manager , чтобы открыть окно Target Manager, перейдите в раздел Worker и нажмите кнопку Add Worker . После этого откроется следующее окно: Здесь вам нужно дать имя вашему удаленному работнику (это то, что будет в списке Worker в Target Manager), а затем в разделе Host Name добавить адрес вместе с портом, который был показан и отмечен ранее с рабочей машины. Если все прошло правильно, вы должны получить сообщение о том, что вы можете успешно подключиться к машине удаленного работника. После того, как вы установили Remote Worker и подключились к основному компьютеру с IDE, вам нужно будет перейти к настройке Local Build Settings для каждой платформы (см. раздел ниже для более подробной информации), прежде чем вы сможете скомпилировать свой проект. Удаленные настройки На странице настроек отображаются различные пути и параметры сборки для приложения Remote Worker. В верхней части страницы настроек указаны различные пути, которые будут использоваться приложением Remote Worker. прежде чем делать что-либо еще, их следует пересмотреть и установить альтернативные пути, если это необходимо (в целом, пути по умолчанию должны быть в порядке). Вы можете получить доступ к странице настроек Remote Worker любым из следующих способов: Нажмите на ссылку "Настройки" в верхней части страницы Remote Worker, которая открывается в браузере вашей удаленной машины: Используйте RMB на значке в трее Remote Worker, чтобы открыть настройки в браузере удаленной машины: На локальной машине откройте диспетчер целей и нажмите на иконку "cog", чтобы открыть настройки рабочего в окне на локальной машине: Откроется страница настроек, где вы можете увидеть (и установить) различные опции, позволяющие задать пути для сборки вашего проекта: Локальные настройки сборки После проверки путей, по которым будет работать приложение Remote Worker, вы можете установить (при необходимости) различные настройки локальной сборки. Для разных платформ потребуются различные инструменты сборки, которые должны быть установлены на компьютере сборщика, а удаленный работник должен иметь возможность использовать их на странице настроек. удаленный работник должен иметь возможность использовать их для сборки ваших игр через страницу настроек . Первые настройки здесь - это общие настройки: Здесь вы можете настроить порт для Help Server и порт для отладчика . Их можно изменить, если у вас возникнут проблемы с подключением при использовании модуля отладки, но в целом следует оставить значения по умолчанию. Теперь вы можете перейти к настройке различных опций для платформ, на которых может работать удаленный работник: Если у вас есть доступ к целевой платформе Windows, вы сможете сразу же выполнить сборку с помощью удаленного рабочего на ПК с Windows, используя опцию виртуальной машины (VM). Однако если вы хотите использовать YoYo Compiler (YYC), то вам также необходимо установить путь к Visual Studio на компьютере для сборки. Для получения подробной информации о том, как настроить ПК с Windows для компиляции ваших проектов, пожалуйста, обратитесь к следующей статье службы поддержки: Настройка для Windows Компиляция После того как вы настроили Remote Worker и выполнили настройки локальной сборки (см. раздел выше), вы можете начать использовать его для компиляции ваших игр. Для этого нужно просто выбрать Remote Worker из списка Target Manager в списке Target Manager, выбрать платформу для тестирования/компиляции, а также любые другие опции (например, тип компиляции или конфигурацию), а затем нажать Play или Create Executable . ПРИМЕЧАНИЕ: Доступные платформы в диспетчере целей будут меняться в зависимости от возможностей компьютера удаленного работника, и не все цели будут доступны. Также обратите внимание, что для удаленных работников не существует цели "Тест". Если вы просто воспроизводите проект, то машина Remote Worker скомпилирует и запустит его, но если вы создаете исполняемый файл, то Remote Worker скомпилирует проект, а затем вернет файл *.zip на локальную машину для сохранения. Следует отметить, что ни кнопка Debug , ни кнопка Clean не будут доступны при использовании Remote Worker. Также обратите внимание, что в настоящее время вы можете выполнять только одну удаленную сборку за раз. После установки и запуска Remote Worker на вашей удаленной машине он появится на панели задач в виде значка, и вы можете щелкнуть его правой кнопкой мыши, чтобы вызвать следующие опции: Открыть: Откройте приложение Remote Worker в браузере по умолчанию. Показать журнал: Откройте файл журнала для удаленного работника в браузере. Настройки сети: Здесь вы можете отфильтровать, какие машины вы хотите иметь возможность отправлять сборки на машину удаленного работника. Каждый фильтр может быть фильтром в стиле CIDR или именем хоста машины для разрешения. Вы должны знать, что если вы хотите, чтобы несколько машин подключались к одному удаленному работнику, или если вы смешиваете типы подключения (т.е. проводное и Wi-Fi), то вам может понадобиться отредактировать диапазон разрешенных IP-адресов. Редактировать настройки: Откройте меню настроек удаленного работника (описано в разделе Настройки, выше). Выход: Выйти из приложения Remote Worker. Это означает, что вы больше не сможете подключиться (или скомпилировать) к приложению с локальной машины, пока не перезапустите приложение Remote Worker.

Основная структура кода Код пишется блоками, и типичный блок кода состоит из набора инструкций, называемыхоператорами, которые затем интерпретируются GameMaker Studio 2 и используются для того, чтобы что-то произошло в вашей игре. Это "что-то" может быть простым, как сложение 2 и 2 для получения 4, или сложным, как заставить врага убежать, когда его здоровье становится ниже определенного значения. Фактическая структура программы может сильно варьироваться в зависимости от используемых функций, но в общих чертах она выглядит следующим образом: <высказывание>; <высказывание>; ... Высказывания должны разделяться символом ';' для предотвращения ошибок с объявлениями переменных и для поддержания чистоты и порядка в коде, и они могут состоять из объявлений переменных, выраженийи вызовов определенных функций. Вы также можете "сгруппировать" утверждения в блок с помощью фигурных скобок {}, чтобы они выполнялись вместе, как в следующем примере условный пример: if (<условный>) { <условие>; <утверждение>; ... } ПРИМЕЧАНИЕ: Язык GameMaker Language также может принимать begin и end вместо фигурных скобок {}, хотя это, как правило, не самый распространенный способ: если (<условно>) begin <высказывание>; <высказывание>; ... end Вот более наглядное представление того, как может выглядеть блок кода, на этот раз созданный как сценарий в редакторе сценариев GameMaker Studio 2: Существует несколько различных типов операторов, выражений, условий и функций, все они подробно рассматриваются в последующих разделах руководства.

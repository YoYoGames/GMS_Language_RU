столкновение_эллипса Collision_ellipse использует первые четыре аргумента (x1,y1,x2,y2) для определения ширины и высоты эллипса в пределах текущей комнаты, а затем проверяет, не находится ли какой-либо объект, определенный аргументом "obj" находится в столкновении с этой областью. Это столкновение может быть проверено как точное или нет, и вы также можете выбрать, проверять ли сам экземпляр, выполняющий код, или нет. Рассмотрим следующее изображение: Здесь экземпляр в центре использует эллипс столкновения для проверки наличия шаровых объектов. Теперь синие объекты не имеют точной границы, и, как вы можете видеть, даже если спрайт не касается эллипса, столкновение все равно может произойти (даже если вы установите параметр exact в функции на true), поскольку граничная область этого спрайта выходит за пределы эллиптической области, определенной collision_circle. С другой стороны, зеленые шарики будут считаться столкнувшимися только в том случае, если реальный спрайт выйдет за пределы определенного эллипса. Помните, что для точного столкновения должны учитываться как спрайт объекта, так и функция столкновения. Следует также отметить, что возвращаемым значением функции может быть id любого из экземпляров, считающихся столкнувшимися. Обратите внимание, что вместо индекса объекта вы можете указать экземпляр id для проверки конкретного экземпляра, или ключевые слова экземпляра all, или other (в зависимости от события и текущей области видимости кода). Синтаксис: collision_ellipse(x1, y1, x2, y2, obj, prec, notme); Аргумент Описание x1 Координата x левой стороны эллипса для проверки. y1 Координата y верхней стороны эллипса для проверки. x2 Координата x правой стороны эллипса для проверки. y2 Координата y нижней стороны эллипса для проверки. obj Объект для проверки столкновений экземпляров. prec Основана ли проверка на точных столкновениях(true, что медленнее) или на его граничной области в целом(false, быстрее). notme Должен ли вызывающий экземпляр, если он релевантен, быть исключен(true) или нет(false). Возвращается: Идентификатор экземпляра или нет Пример: if collision_ellipse(50, 50, 200, 100, obj_Player, false, true) { instance_create_layer(obj_Player.x, obj_Player.y, "Effects", obj_Splash); } При этом будет проверена эллиптическая зона в пределах 50x, 50y и 200x, 100y для объекта "obj_Player". Если произойдет столкновение с этим объектом, то будет создан экземпляр "obj_Splash" в координатах x/y объекта obj_Player.

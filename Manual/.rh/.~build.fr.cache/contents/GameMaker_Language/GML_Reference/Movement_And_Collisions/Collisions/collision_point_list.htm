список_точек_столкновения Эта функция аналогична функции collision_point() только вместо обнаружения одного столкновения за один раз, она обнаруживает несколько столкновений. Вы указываете x/y позицию точки, которую нужно и можно задать точность проверки (в этом случае все проверяемые экземпляры должны иметь точные маски столкновений), а также указать, должна ли проверка включать вызывающий экземпляр или нет. Вы также предоставляете список DS, поэтому id значения всех экземпляров, которые сталкиваются с вызывающим экземпляром, будут будут добавлены в конец данного списка. Вы можете очистить список перед вызовом этой функции, чтобы он содержал только результаты этого вызова функции. У вас также есть возможность упорядочить экземпляры на основе расстояний от заданной точки до их начала. Функция возвращает количество найденных экземпляров или 0, если не найдено ни одного. Обратите внимание, что вместо индекса объекта можно использовать ключевое слово instance all, чтобы проверить все экземпляры в текущей комнате, которые могут включать экземпляр, выполняющий код (в зависимости от аргумента notme ). Синтаксис: collision_point_list(x, y, obj, prec, notme, list, ordered); Аргумент Описание x Координата x точки, которую нужно проверить. y Координата y точки для проверки. obj Объект для проверки столкновений экземпляров. prec Основана ли проверка на точных столкновениях(true, что медленнее) или на его граничной области в целом(false, быстрее). notme Должен ли вызывающий экземпляр, если он релевантен, быть исключен(true) или нет(false). список Список DS, используемый для хранения идентификаторов сталкивающихся экземпляров. заказал Должен ли список быть упорядочен по расстоянию(true) или нет(false). Возвращается: Integer (Число экземпляров, с которыми произошло столкновение) Пример: var _list = ds_list_create(); var _num = collision_point_list(x, y, obj_Enemy, false, true, _list, false); if _num > 0 { for (var i = 0; i < _num; ++i;) { instance_destroy(_list[| i]); } } ds_list_destroy(_list); Приведенный выше код проверит позицию вызывающего экземпляра на наличие столкновений с экземплярами "obj_Enemy". Если столкновения есть, то предварительно созданный список просматривается, и каждый экземпляр, который был в столкновении, уничтожается.

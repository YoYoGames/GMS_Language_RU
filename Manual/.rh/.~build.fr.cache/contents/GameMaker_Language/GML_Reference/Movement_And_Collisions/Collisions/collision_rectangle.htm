прямоугольник столкновения Collision_rectangle использует первые четыре аргумента (x1,y1,x2,y2) для определения области в текущей комнате, а затем проверяет, не столкнулся ли какой-либо объект, определенный аргументом "obj", с этой областью. областью. Это столкновение может быть проверено как точное или нет, и вы также можете выбрать, проверять ли сам экземпляр, выполняющий код, или нет. Рассмотрим следующее изображение: Здесь экземпляр в центре использует прямоугольник столкновения для проверки на наличие шариковых объектов. Теперь синие объекты не имеют точной границы, и, как вы можете видеть, даже если спрайт не касается прямоугольника, столкновение все равно произойдет (даже если вы установите параметр exact в функции на true). поскольку ограничивающая область этого спрайта выходит за пределы прямоугольника collision_rectangle. С другой стороны, зеленые шарики будут считаться столкнувшимися только в том случае, если сам спрайт выйдет за пределы прямоугольника. Помните, что для учета точных столкновений и спрайт объекта, и функция столкновения должны иметь значение "Точное". Следует также отметить, что возвращаемым значением функции может быть id любого из экземпляров, считающихся столкнувшимися. Обратите внимание, что вместо индекса объекта вы можете указать экземпляр id для проверки конкретного экземпляра, или ключевые слова экземпляра all, или other (в зависимости от события и текущей области видимости кода). Синтаксис: collision_rectangle(x1, y1, x2, y2, obj, prec, notme); Аргумент Описание x1 Координата x левой стороны прямоугольника для проверки. y1 Координата y верхней стороны прямоугольника для проверки. x2 Координата x правой стороны прямоугольника для проверки. y2 Координата y нижней стороны прямоугольника для проверки. obj Объект для проверки столкновений экземпляров. prec Основана ли проверка на точных столкновениях(true, что медленнее) или на его граничной области в целом(false, быстрее). notme Должен ли вызывающий экземпляр, если он релевантен, быть исключен(true) или нет(false). Возвращается: Идентификатор экземпляра или нет Пример: var inst; inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true); if inst != noone { with (inst) instance_destroy(); } Этот короткий код использует collision_rectangle для проверки области в комнате от 50x, 50y (верхняя левая часть прямоугольника) до 200x, 200y (нижняя правая часть прямоугольника) на наличие экземпляра объекта под названием "obj_ball". Возвращаемое значение сохраняется во временной переменную, которая затем проверяется, является ли это значение идентификатором экземпляра или ключевым словом noone.

mp_grid_create С помощью этой функции вы можете создать mp_grid для функций планирования движения. Она возвращает индекс, который должен быть сохранен в переменной для использования во всех других вызовах функций MP-сетки. Координаты x и y указывают на положение левого верхнего угла сетки, hcells и vcells указывают количество горизонтальных и вертикальных ячеек, которые будут созданы в сетке, а ширина и высота ячеек указывают размер ячеек в пикселях. Вы можете создать и поддерживать несколько структур сетки одновременно но имейте в виду, что каждая сетка требует памяти, и чем больше ячеек, тем больше памяти она требует. Также обратите внимание, что функции MP достаточно требовательны к процессору, поэтому чем чаще вы будете их вызывать, тем больше ячеек потребуется. вы их вызываете и чем выше разрешение ячеек, тем больше вероятность падения производительности. Синтаксис: mp_grid_create(left, top, hcells, vcells, cellwidth, cellheight): Аргумент Описание слева Начальная x-координата mp_сетки в помещении топ Начальная координата y сетки mp_grid в помещении hcells Количество горизонтальных ячеек, которые будет содержать сетка mp_grid vcells Количество вертикальных ячеек, которые будет содержать сетка mp_grid ширина ячейки Ширина (в пикселях) каждой отдельной ячейки сетки mp_grid высота ячейки Высота (в пикселях) каждой отдельной ячейки сетки mp_grid Возвращается: Реально Пример: global.grid = mp_grid_create(0, 0, room_width / 32, room_height /32, 32, 32); Приведенный выше код создает глобальную переменную "grid", затем генерирует mp_grid и присваивает ее индекс (id) этой переменной для использования во всех дальнейших вызовах функции mp_grid. Сетка покрывает комнату с разрешением ячеек 32x32 пикселя. Это означает. что, например, в комнате размером 640x480 сетка будет содержать 300 ячеек: 20 горизонтальных ячеек (640 / 32), умноженные на 15 вертикальных (480 /32), дают 300 ячеек.

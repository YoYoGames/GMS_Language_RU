виртуальный_ключ_добавить Эта функция позволяет привязать "прикосновения" к области экрана к событиям клавиатуры. Это означает, что если вы назначили область виртуальной клавише, то все прикосновения к этой области будут вызывать событие клавиатуры, соответствующее клавише, которую вы назначили этой области. Каждой виртуальной клавише можно присвоить переменную, которая затем может быть использована в дальнейших функциях виртуальных клавиш для их отображения, скрытия и удаления. Эти ключи назначаются для каждой комнаты и будут автоматически удаляться GameMaker Studio 2 при смене комнаты. Фактическое положение виртуальной клавиши основано на положении экрана, а не на положении комнаты, поэтому значения x/y являются абсолютными на экране. Это означает, что вам не нужно беспокоиться об использовании представлений или относительных координат помещения, и можно просто определить ключи в событии Create объекта (виртуальный ключ нужно определить только один раз для каждой комнаты), а затем нарисовать спрайты ключей в событии Draw GUI, при этом размер слоя GUI должен быть равен размеру экрана. Синтаксис virtual_key_add(x, y, w, h, keycode); Аргумент Описание x Координата x (левая сторона) виртуальной клавиши на экране y Координата y (верхняя сторона) виртуальной клавиши на экране w Ширина виртуальной клавиши h Высота виртуального ключа код ключа Какое событие клавиши клавиатуры должно быть вызвано при касании этой области Возвращается: индекс виртуального ключа Пример: global.Left = virtual_key_add(32, 32, 64, 64, 64, vk_left); Приведенный выше код создает виртуальную клавишу размером 64x64 пикселя, расположенную на экране по адресу (32, 32), которая будет вызывать событие vk_left при касании, и присваивает индекс этой виртуальной клавиши глобальной переменной.

sha1_string_unicode В криптографии SHA-1 - это криптографическая хэш-функция, разработанная Агентством национальной безопасности США и используемая в нескольких широко распространенных приложениях и протоколах, таких как популярный протокол SHA-1. Git где она используется для проверки изменений файлов. Эта функция принимает на вход строку в формате unicode и возвращает 160-битный дайджест сообщения в формате ASCII, например: sha1 input string = "The quick brown fox jumps over the lazy dog" sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12" Таким образом, вы можете генерировать безопасный ключ, который можно хранить и использовать для проверки целостности информации, отправляемой на внешний сервер (или получаемой с него) (например). ПРИМЕЧАНИЕ: Существует два формата кодировки sha1 - UTF-8 и unicode. Оба формата предусмотрены для облегчения взаимодействия с различными настройками сервера, но чаще всего используется unicode. ПРИМЕЧАНИЕ: SHA-1 не является полностью безопасным и может быть взломан. См. эту страницу для получения дополнительной информации. Синтаксис: sha1_string_unicode(string) Аргумент Описание строка Строка для хэширования. Возвращается: Строка Пример: var hash, str; str = base64_encode(game_data); hash = sha1_string_unicode(str); http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?hash=" + hash); http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?data=" + str); Приведенный выше код кодирует строку в base64, а затем генерирует хэш sha1.

sha1_string_utf8 В криптографии SHA-1 - это криптографическая хэш-функция, разработанная Агентством национальной безопасности США и используемая в нескольких широко распространенных приложениях и протоколах, таких как популярный Git где она используется для проверки изменений файлов. Эта функция принимает на вход строку UTF-8 (которая имеет переменное количество байт на символ) и возвращает 160-битный дайджест сообщения в формате ASCII, например: sha1 input string = "The quick brown fox jumps over the lazy dog" sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12" Таким образом, вы можете генерировать безопасный ключ, который можно хранить и использовать для проверки целостности информации, отправляемой на внешний сервер (или получаемой с него) (например). ПРИМЕЧАНИЕ: Существует два формата кодировки sha1 - UTF-8 и unicode. Оба формата предусмотрены для облегчения взаимодействия с различными настройками сервера, но чаще всего используется unicode. ПРИМЕЧАНИЕ: SHA-1 не является полностью безопасным и может быть взломан. См. эту страницу для получения дополнительной информации. Синтаксис: sha1_string_utf8(string) Аргумент Описание строка Строка для хэширования. Возвращается: Строка Пример: var hash, str; str = json_encode(hiscore_map); hash = sha1_string_utf8(str); ini_open("local.ini"); ini_write_string("info", "0", hash); ini_close(); get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str) Приведенный выше код кодирует карту DS в строку JSON. Затем генерируется хэш sha1 и сохраняется в ini-файле, чтобы впоследствии его можно было использовать для проверки целостности JSON в случае получения той же информации от сервера.

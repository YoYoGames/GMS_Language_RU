ds_queue_destroy Эта функция удалит заданную структуру данных очереди из памяти, освободив используемые ею ресурсы и удалив все содержащиеся в ней значения. Эту функцию всегда следует использовать, когда вы закончили использовать очередь DS, чтобы предотвратить утечку памяти. которые могут замедлить работу и привести к краху вашей игры. ВАЖНО! Когда вы создаете структуру данных, значение индекса для ее идентификации является целым числом, начинающимся с 0. Это означает, что структуры данных разных типов могут иметь одинаковое значение индекса, поэтому, если вы сомневаетесь, вам следует использовать значение индекса ds_exists() функцию перед обращением к ним. Также обратите внимание, что индексы используются повторно, поэтому значение индекса уничтоженной структуры данных может быть использовано вновь созданной, поэтому мы рекомендуем всегда устанавливать переменную, в которой хранился индекс DS, в -1 после уничтожения. Синтаксис: ds_queue_destroy(id); Аргумент Описание id Идентификатор структуры данных для удаления. Возвращается: Н/Д Пример: if (lives == 0) { ds_queue_destroy(AI_queue); AI_queue = -1; room_goto(rm_Menu); } Приведенный выше код проверит значение встроенной глобальной переменной "lives" и, если оно равно 0, уничтожит очередь DS, индексированную в переменной "AI_queue", а затем сменит комнаты.

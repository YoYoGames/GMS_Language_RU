отладочное_событие Эта функция генерирует пользовательское событие отладки, которое будет отображаться в окне Graph View отладчика, когда игра выполняется в режиме отладки. Если вам требуется, чтобы сообщения отображались, когда игра не находится в режиме отладки режиме, то вам следует использовать show_debug_message(). Стоит отметить, что функция также принимает две зарезервированные строки, чтобы помочь выполнить отладку с помощью внешних приложений, таких как Visual Studio. Эти строки: "OutputDebugOn" - Это включает вызов OutputDebugString для Windows что означает, что весь отладочный вывод - все, что вы видите в окне Output - может быть просмотрен Visual Studio или сторонними приложениями. " BreakOnError" - Эта опция предназначена только для сборок Windows YYC и означает, что проекты не будут отображать обычный экран ошибки кода, если среда выполнения обнаружит ошибку, а просто продолжат работу и аварийно завершатся. Это позволяет вам увидеть трассировку стека в Visual Studio при отладке. Синтаксис: debug_event(string) Аргумент Описание строка Пользовательская строка события отладки для использования. Возвращается: Н/Д Пример: if !surface_exists(global.EffectsSurface) { debug_event("Recreating Effects Surface"); global.EffectsSurface = surface_create(room_width, room_height); } Приведенный выше код проверяет, существует ли поверхность, и если нет, то в графическом представлении отладчика запускается событие отладки (игра должна быть запущена в режиме отладки, чтобы это было видно) и поверхность создается заново.

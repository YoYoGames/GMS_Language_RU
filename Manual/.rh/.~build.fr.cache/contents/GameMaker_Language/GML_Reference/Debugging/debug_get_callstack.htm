debug_get_callstack Эта функция генерирует массив строк в виде "стека вызовов", где сначала перечисляется текущий сценарий, а затем все другие сценарии, которые были запущены для того, чтобы текущий сценарий был выполнен. Точный формат строк зависит от от выбранной целевой платформы, но в основном это будет имя сценария/события, затем двоеточие : и номер строки, как показано здесь: "gml_Script_script2:1" "gml_Script_script1:32" "gml_Script_script0:64" "gml_Object_object0_Create_0:1" Функция позволяет передавать необязательный аргумент, который является максимальной глубиной возвращаемого стека вызовов. Это значение представляет собой количество скриптов, включенных в возвращаемый массив, начиная с текущего скрипта. Если этот аргумент не то будет возвращен полный стек вызовов. Обратите внимание, что возвращаемый массив всегда будет содержать 0 в своей последней позиции, после перечисления сценариев стека вызовов. Синтаксис: debug_get_callstack([maxdepth]) Аргумент Описание [maxdepth]. Максимальная глубина стека вызовов Возвращается: Массив Пример: if debug_mode { if keyboard_check(vk_escape) { var _a = debug_get_callstack(4); for (var i = 0; i < array_length_id(_a); ++i;) { show_debug_message(_a[i]); } } } Приведенный выше код проверяет, включен ли режим отладки, и если да, то проверяет, удерживается ли клавиша.

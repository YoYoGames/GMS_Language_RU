view_surface_id С помощью этой переменной можно установить содержимое данного порта просмотра для рисования на поверхности или получить идентификатор текущей поверхности, если она была назначена порту просмотра. При работе с поверхностями часто требуется, чтобы вы захватили всю видимую область экрана на поверхность, и поэтому вы назначаете ее порту представления с помощью этой переменной. Это означает, что все, что отображается в выбранном представлении, теперь будет рисоваться на назначенной поверхности. Теперь вид не будет рисоваться на экран, что означает, что вам нужно будет либо: Включите новый вид и рисуйте поверхность только в этом виде (используя вид_текущий для проверки того, какой вид рисуется) Нарисуйте поверхность в событии Draw GUI экземпляра, поскольку слой GUI не зависит от представлений. Вы также можете прочитать эту переменную, чтобы получить индекс поверхности, которая была назначена выбранному виду, или она вернет -1, если поверхность не была назначена, и обычно поверхность, используемая для этой функции, должна быть размером самой камеры вида (не порта вида). В расширенном примере ниже показана базовая настройка для захвата вида и его отрисовки в событии Draw GUI, а для получения дополнительной информации о поверхностях см. раздел " Поверхности". Обратите внимание, что вы также можете установить порт представления на поверхность с помощью функции view_set_surface_id(). Синтаксис: view_surface_id[0...7]; Возвращается: Real (индекс поверхности или -1, если поверхность не была назначена) Расширенный пример: В этом расширенном примере мы создадим поверхность и назначим ее порту view port[0], чтобы она захватывала вид камеры, назначенный этому порту, а затем нарисуем ее на экране в событии Draw GUI. Для начала нам нужно инициализировать нашу переменную surface в событии Create Event экземпляра контроллера: surf = -1; Мы установили переменную surface в -1, так как все функции surface должны использоваться в событиях Draw, чтобы предотвратить странные ошибки или неопределенное поведение. Итак, после этого в главном событии Draw мы получим следующее: if !surface_exists(surf) { surf = surface_create(camera_get_view_width(view_camera[0]), camera_get_view_height(view_camera[0])); view_surface_id[0] = surf; } Поверхности непостоянны, то есть они могут быть удалены из памяти в любой момент из-за управления памятью ОС и других вещей, поэтому здесь мы проверяем, существует ли наша поверхность, и если нет, то создаем ее и присваиваем виду 0. Теперь все, что будет отображаться в представлении 0, будет рисоваться на созданной нами поверхности. Если вы ничего не сделаете в этот момент, то при запуске игры вы получите пустой экран, так как все рисование происходит на поверхности, но сама поверхность нигде не рисуется. Поэтому теперь нам нужно нарисовать поверхность на экране в событии Draw GUI следующим образом: if surface_exists(surf) { draw_surface_stretched(surf, 0, 0, display_get_gui_width(), display_get_gui_height()); } Теперь этот код нарисует поверхность, которую мы создали из растянутой камеры обзора, чтобы заполнить весь слой GUI.

HTTP В этом разделе перечислены все различные функции асинхронного HTTP, доступные в GameMaker Studio 2. Эти функции будут генерировать асинхронное HTTP-событие во всех экземплярах, в которых оно есть: http_request http_get http_get_file http_post_string Обратите внимание, что вышеуказанные функции http_* могут работать не так, как ожидается, из-за проблем с безопасностью междоменных соединений. Это означает, что запросы к вашему серверу или попытки загрузить ресурсы из других доменов будут блокируются и могут возвращать пустые результаты или ошибки 404. Одним из способов обойти эту проблему является использование PHP на стороне сервера, который разрешает определенным доменам доступ к вашему серверу (это также способ защиты ваших ресурсов и блокировки серверов, не включенных в PHP). которые не включены в список разрешенных PHP). Ниже приведен пример кода, который можно использовать для этого: $http_origin = $_SERVER['HTTP_ORIGIN']; if ($http_origin == "http://127.0.0.1:51268") { header("Access-Control-Allow-Origin: *"); } Для запросов загрузки изображений, в которых важно определить или установить их кросс-оригинальный тип, существуют следующие функции: http_get_request_crossorigin http_set_request_crossorigin В большинстве случаев эти функции не нужно использовать, но если игра хранится на защищенном сервере - где для доступа к определенным активам может потребоваться базовая аутентификация и при загрузке возникают ошибки безопасности - установка типа cross-origin для запросов изображений на "use-credentials" может оказаться необходимой. типа для запросов изображений на "use-credentials" может оказаться необходимой (в отличие от настройки по умолчанию "anonymous").

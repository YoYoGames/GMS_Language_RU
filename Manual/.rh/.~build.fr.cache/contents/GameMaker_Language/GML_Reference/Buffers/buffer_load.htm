буферная_загрузка Эта функция используется для загрузки буфера, который был ранее сохранен с помощью buffer_save() функций, а также любые Включенные файлы или файлы загруженных извне. Она возвращает идентификатор нового буфера, который создается функцией как "растущий" буфер (см. здесь) с выравниванием по байтам 1. Этот идентификатор должен храниться в переменной и использоваться во всех последующих обращениях к этому буферу. обращениях функции к этому буферу. Если по какой-либо причине загрузка не удалась, функция вернет -1, а в окне вывода компилятора появится сообщение о том, что загрузка не удалась. Обратите внимание, что на HTML5 любые буферы, загруженные из локального хранилища, будут декодированы с использованием base64 (поскольку они сохраняются как строки base64), однако использование этой функции для Included Files или любых внешних файлов не будет использовать декодирование base64, что должно быть хорошо для большинства случаев. ПРИМЕЧАНИЕ: Важно, чтобы вы удаляли из памяти любые динамически созданные ресурсы, подобные этому, когда они вам больше не нужны, чтобы предотвратить утечку памяти, поэтому, когда вы закончите работу с созданным буфером, вы должны освободить его снова использование buffer_delete(). Синтаксис: buffer_load(filename); Аргумент Описание имя файла Имя файла для загрузки. Возвращается: настоящий Пример: player_buffer = buffer_load("Player_Save.sav"); Приведенный выше код загружает в память ранее сохраненный буфер, создавая новый буфер.

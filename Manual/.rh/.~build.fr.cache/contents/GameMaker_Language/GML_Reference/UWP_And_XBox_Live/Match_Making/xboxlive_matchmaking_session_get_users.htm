xboxlive_matchmaking_session_get_users Эта функция создаст и заполнит карту DS деталями пользователей в указанной сессии, или -1, если произошла ошибка. Вы можете получить указатель идентификатора сессии из async_load DS map, который возвращается в асинхронном социальном событии, когда вы создали или нашли сессию (см. xboxlive_matchmaking_create() для более подробной информации). Возвращаемая карта DS будет содержать следующие пары ключ/значение: " num_results" - содержит количество пользователей в сессии " userId "- содержит ID пользователя (< index> - это значениеот 0 до "num_results" - 1) " userIsOwner "- содержит 1, если пользователь является хозяином, иначе 0 (< index> - значениеот 0 до "num_results" - 1) Синтаксис: xboxlive_matchmaking_session_get_users(session_id); Аргумент Описание идентификатор сессии Указатель идентификатора сессии для использования Возвращается: Идентификатор запроса (целое число) Пример: var session_map = xboxlive_matchmaking_session_get_users(global.SessionID); if (session_map != -1) { for (var i = 0; i < session_map[? "num_results"]; i++) { if (session_map[? "userIsOwner" + string(i)] == 1) { global.OwnerID = session_map[? "userId" + string(i)]; } } } Вышеприведенные действия позволят получить пользовательские данные для всех пользователей, участвующих в сеансе подбора, и установить глобальную переменную на ID владельца сеанса.

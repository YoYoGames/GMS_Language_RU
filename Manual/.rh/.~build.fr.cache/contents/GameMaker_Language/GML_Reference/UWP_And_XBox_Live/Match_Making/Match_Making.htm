Сопоставление Matchmaking (многопользовательская игра) для Xbox Live и UWP требует некоторой настройки, частично через конфигурационный файл, который определяет, какие сетевые соединения вы собираетесь использовать, и частично через настройку шаблонов сессий и бункеров matchmaking на Xbox Портал разработчиков. GameMaker Studio 2 поддерживает это с помощью цели UWP и позволяет сделать следующее: Создание многопользовательской сессии Найти (и/или присоединиться) к многопользовательской сессии Получать уведомления о том, что соответствующие игроки присоединяются или покидают сессию, а также получать данные об их сетевом подключении Получайте уведомления, когда хозяин сеанса переходит к другому пользователю Получение списка пользователей в сессии Оставить сессию Подключение к сети Xbox Live опирается на так называемую Ассоциацию безопасных устройств, которая представляет собой договор, определяющий, как именно устройство подключается к другим устройствам и какие данные оно может передавать с помощью соединения. это соединение. Они настраиваются с помощью файла, который необходимо включить в проект. Пример этого файла вы можете найти здесь. Основные важные пункты в файле - это протокол и номера портов для описания каждого сокета, поскольку они должны соответствовать тому, что вы фактически установили в коде. После создания файла сетевого манифеста его следует добавить в проект в качестве включенного файла. Затем, внутри UWP Game Options, на вкладке Xbox One General, он должен быть выбран в комбинированном поле "Включенный файл для манифеста сетевой конфигурации". Чтобы создать многопользовательскую сессию, необходимо указать, какой шаблон сессии будет использоваться. Эти шаблоны сессий определяются в XDP, но пример показан ниже: Имя сессии: MatchSession Версия контракта: 107 Шаблон сессии (текст JSON): { "constants": { "system": { "version": 1, "maxMembersCount": 12, "capabilities": { }, "memberInitialization": { "externalEvaluation": false, "membersNeededToStart": 1 } }, "custom": { } } Описание полей можно найти в разделе "Обзор сессии" в файле справки XDK. Следующие функции доступны для XboxLive matchmaking (обратите внимание, что вам также потребуется использовать функции GameMaker Studio 2 Networking Functions для соединений и т.д.): xboxlive_matchmaking_start xboxlive_matchmaking_stop xboxlive_matchmaking_create xboxlive_matchmaking_find xboxlive_matchmaking_session_get_users xboxlive_matchmaking_join_session xboxlive_matchmaking_session_leave GameMaker Studio 2 предоставляет дополнительный набор функций подбора игроков, которые можно использовать для приглашения других игроков в сети присоединиться к сессии или для принятия приглашения присоединиться к чужой сессии. Доступны следующие функции следующие: xboxlive_matchmaking_send_invites xboxlive_matchmaking_set_joinable_session xboxlive_matchmaking_join_invite Когда игрок принимает приглашение от другого игрока, происходит одно из двух событий: Если ваша игра запущена, вы получите социальное асинхронное событие с картой async_load DS, содержащей следующие ключи/значения: " id" - Это значение будет установлено в MATCHMAKING_INVITATION. "invitation_id" - Это уникальное значение идентификатора сессии. "invitation_host" - Это уникальное значение ID сеанса для ведущего сеанса. Вы должны сохранить значения ID и Host, затем покинуть все необходимые сессии и вызвать функцию xboxlive_matchmaking_join_invite(), передав в качестве аргументов значения ID приглашения и хоста приглашения в событии Step. Если ваша игра в данный момент не запущена, она будет запущена ОС. Вы получите асинхронное событие типа Social Asynchronous Event с тем же набором ключей, что и перечисленные выше. Сохраните их в переменных, затем подключитесь к Xbox Live matchmaking и, наконец, вызовите xboxlive_matchmaking_join_invite(). После вызова xboxlive_matchmaking_join_invite() пользователь попытается присоединиться к сессии, и вы получите асинхронные социальные события, как если бы вы вызвали xboxlive_matchmaking_find() на этой сессии.

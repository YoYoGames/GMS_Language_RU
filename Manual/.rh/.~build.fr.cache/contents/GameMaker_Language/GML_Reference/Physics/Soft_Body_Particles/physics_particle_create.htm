физика_частицы_создать С помощью этой функции вы можете создать одну частицу в любом месте вашей игровой комнаты, установив определенные флаги и свойства. Флаги" являются возвращаемым значением комбинации следующих констант: Постоянная Описание phy_particle_flag_water Свойства по умолчанию для частицы мягкого тела. phy_particle_flag_zombie Частица-зомби - это частица, которая будет уничтожена после одного шага, при этом все остальные будут помечены таким образом. phy_particle_flag_wall Это определяет частицу как статическую, по существу создавая ее как неподвижный объект в физическом моделировании, поскольку она будет оставаться в фиксированном положении независимо от того, что с ней столкнется. Вы должны использовать этот флаг, а не устанавливать плотность в 0. phy_particle_flag_spring Пружинные частицы создают эффект прикрепления друг к другу, как будто пружиной. Частицы, созданные с помощью этого флага, "соединяются" попарно, причем каждая частица соединяется с той, которая была ближе всего к ней в момент создания. Попав в пару, частицы не меняют "партнеров", и чем дальше внешняя сила отталкивает их друг от друга, тем с большей силой они столкнутся, когда эта внешняя сила будет удалена. Обратите внимание, что независимо от того, насколько далеко парные частиц друг от друга, связь между ними не разрывается. phy_particle_flag_elastic Упругие частицы деформируются и могут также отскакивать при столкновении с другими жесткими телами в физическом моделировании. phy_particle_flag_viscous Вязкая частица - это частица, которая проявляет "цепкость" или "липкость", как масло. Вязкие частицы больше слипаются и склеиваются. phy_particle_flag_powder Частицы порошка создают эффект рассеивания, подобный тому, который можно наблюдать при использовании песка или пыли. phy_particle_flag_tensile Растягивающие частицы используются для создания эффекта поверхностного натяжения, или тугой кривизны на поверхности тела жидкости. Они могут использоваться, например, для создания поверхностного натяжения, которое можно наблюдать на капле воды. Как только натяжение частицы отскакивают, как если бы они были упругими, но при этом продолжают притягиваться друг к другу. В результате частицы, отскакивая, образуют скопления. phy_particle_flag_colourmixing Частицы, смешивающие цвета, принимают некоторые цвета других частиц, с которыми они сталкиваются. Обратите внимание, что если только одна из двух сталкивающихся частиц является смешивающей цвет, другая частица сохраняет свой цвет до столкновения. Эти флаги используют битовую маскировку для создания конечного выходного значения, которое затем проверяется для установки различных основных свойств частицы (при этом базовым свойством всегда является свойство phy_particle_flag_water). Например, если вы хотите смоделировать вязкую жидкость с поверхностным натяжением, вы будете использовать побитовую комбина цию или "|", чтобы отметить соответствующие биты, как показано в примере ниже. Таким образом, вы можете установить различные свойства (отличные от глобальных) для каждой отдельной созданной частицы. Кроме флагов, вы также можете задать положение в комнате, в котором будет создана частица, начальную горизонтальную и вертикальную скорость, начальный цвет и альфа, а также категорию пользователя. Значения цвета и альфа используются только при рисовании частицы с помощью функции physics_particle_draw()но значение категории используется во многих других функциях. Это значение представляет собой произвольное целое число, которое вы присваиваете каждой частице, которая и может использоваться для установки флагов в дальнейшем для этой конкретной категории, а также для ряда других вещей. Например, если вы определили частицы "вода" и "масло", то вы присвоите всем частицам, отмеченным как "вода", флаг "масло". частицам, помеченным как "вода", пользовательскую категорию 1, а всем частицам, созданным как "масло", пользовательскую категорию 2, что позволит вам позже выбрать одну категорию для изменения ее флагов или получения конкретных данных. ПРИМЕЧАНИЕ: Вы можете использовать любое целочисленное значение для категории, кроме 0, которое зарезервировано симуляцией и используется для выбора всех категорий в других функциях (например, для рисования). Функция возвращает уникальное значение индекса (или ID) для частицы, которое может быть сохранено в переменной для использования в других функциях, напрямую ссылающихся на частицу. Синтаксис: physics_particle_create(flags, x, y, xv, yv, col, alpha, category) Аргумент Описание флаги Флаги для установки на частицу. x Позиция x для создания частицы. y Позиция y для создания частицы. xv Начальная горизонтальная скорость. yv Начальная вертикальная скорость. col Базовый цвет, который будет использоваться для частицы. альфа Базовая альфа, используемая для частицы. категория Определенная пользователем категория, к которой принадлежит частица. Возвращается: Реально Пример: var flags = phy_particle_flag_water | phy_particle_flag_viscous | phy_particle_flag_tensile; var x_vel = lengthdir_x(5, image_angle); var y_vel = lengthdir_y(5, image_angle); var p = physics_particle_create(flags, x, y, x_vel, y_vel, c_white, 1, 1); Приведенный выше код создаст переменные для установки типа частицы и ее начальной скорости, а затем использует их для создания частицы в позиции экземпляра x/y.

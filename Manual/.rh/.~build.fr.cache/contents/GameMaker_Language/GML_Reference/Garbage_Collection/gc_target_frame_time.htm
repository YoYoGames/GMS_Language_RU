gc_target_frame_time С помощью этой функции вы можете указать сборщику мусора, что он должен стремиться тратить на выполнение каждого кадра не более указанного времени. Функция принимает значение времени, указанное в микросекундах (где 1 000 000 микросекунд равны одной секунде), а целевое время кадра по умолчанию составляет 100 микросекунд. Обратите внимание, что это просто целевое значение, поскольку некоторые операции все равно должны выполняться полностью в одном кадре и иногда могут занимать больше времени, чем целевое время. Также обратите внимание, чтоувеличение целевого времени заставит сборщик мусора быстрее реагировать на быстрые изменения в использовании памяти и быстрее освобождать память, хотя на практике это вряд ли потребуется в большинстве случаев. Также обратите внимание, что установка целевого времени кадра в 0 не приведет к полной остановке сборщика - он все равно будет выполнять минимальный объем работы каждый кадр. Чтобы полностью отключить сборщик, используйте функцию gc_enable(). Синтаксис: gc_target_frame_time(time); Аргумент Описание время Целевое время - в микросекундах - за которое сборщик мусора должен отработать каждый кадр Возвращается: Н/Д (не определено) Пример: if (gc_get_target_frame_time() != 50) { gc_target_frame_time(50); } Приведенный выше код проверяет текущее значение времени кадра для сборщика мусора, и если оно не равно 50 микросекундам, то оно устанавливается на это значение.

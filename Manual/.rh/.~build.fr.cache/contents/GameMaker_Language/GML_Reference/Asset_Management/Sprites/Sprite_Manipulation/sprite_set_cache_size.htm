sprite_set_cache_size При выполнении смешивания изображений HTML5 не может делать это динамически, как это можно было бы сделать в исполняемом файле. Поэтому GameMaker Studio 2 вынуждена временно сохранять копию изображения с блендингом и загружать ее. Эта функция устанавливает, сколько копий данного спрайта может быть кэшировано до перезаписи старых копий. По умолчанию - 4. Это применяется ко всем подизображениям спрайта. Синтаксис: sprite_set_cache_size(ind, max); Аргумент Описание инд Индекс спрайта для изменения размера кэша. max Максимальное количество кэшированных копий спрайта, которое может быть сохранено. Возвращает Н/Д Пример: sprite_set_cache_size(sprite0, 2); Это установит кэш спрайта для всех подизображений спрайта0 в 2 копии.

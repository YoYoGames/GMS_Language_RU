sprite_set_cache_size_ext При выполнении смешивания изображений HTML5 не может делать это динамически, как это можно было бы сделать в исполняемом файле. Поэтому GameMaker Studio 2 вынуждена временно сохранять копию изображения с блендингом и загружать ее. Эта функция устанавливает, сколько копий данного спрайта может быть кэшировано до перезаписи старых копий. По умолчанию - 4. Это применяется к одному заданному подизображению спрайта. Синтаксис: sprite_set_cache_size_ext(ind, index, max); Аргумент Описание инд Индекс спрайта для изменения размера кэша. индекс Субизображение ind для изменения размера кэша. max Максимальное количество кэшированных копий спрайта, которое может быть сохранено. Возвращает Н/Д Пример: sprite_set_cache_size_ext(sprite0, 0, 2); Это установит кэш спрайта первого подизображения спрайта0 в 2 копии.

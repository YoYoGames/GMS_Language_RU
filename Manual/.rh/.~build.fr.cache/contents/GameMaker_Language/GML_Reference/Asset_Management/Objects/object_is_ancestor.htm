object_is_ancestor Эта функция может использоваться для проверки объекта и определения того, является ли он предком другого объекта или нет. Таким образом, оба аргумента должны быть индексами объектов, а не идентификаторами экземпляров, и первый аргумент - это всегда объект, который вы хотите узнать, является ли он дочерним по отношению ко второму аргументу. хотите узнать, является ли он дочерним по отношению ко второму аргументу, который всегда является объектом, который вы хотите проверить как предка (родителя). Синтаксис: object_is_ancestor(obj, par); Аргумент Описание obj Объект, который проверяется как дочерний. Объект, который проверяется в качестве предка (родителя). Возвращается: Булево Пример: if object_is_ancestor(object_index, obj_Enemy) { instance_destroy(); } Приведенный выше код проверяет, является ли экземпляр, выполняющий код, дочерним объектом объекта "obj_Enemy", и если да, то он уничтожается.

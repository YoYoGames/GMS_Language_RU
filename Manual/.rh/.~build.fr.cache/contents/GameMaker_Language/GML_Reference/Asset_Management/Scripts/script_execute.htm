script_execute Эта функция изначально была разработана для динамического вызова активов сценария, но с введением пользовательских функций ее теперь можно использовать двумя способами: Первый способ - это поддержка унаследованных сценариев, где каждый сценарий был одной функцией, а название сценария было также названием функции. Используемый таким образом, он будет работать, как и раньше, и будет запускать созданный пользователем актив скрипта и передавать заданные аргументы в него. Обратите внимание, что в данном случае вызывается не скрипт, а одноименная функция внутри скрипта. Второй способ - это вызов функции из актива сценария, который содержит несколько функций или определений переменных. В этом случае нужно указать имя функции и аргументы, которые она требует, а не имя актива сценария. Обратите внимание, что это работает только для функций сценария, но не для методов, поскольку только функции сценария имеют присвоенный им идентификатор сценария. Несмотря на то, что эта функция предназначена в основном для поддержки наследия, она не является устаревшей, так как все еще имеет свое место в сетевых ситуациях и ситуациях удаленного вызова процедур. Также стоит отметить, что эта функция медленная, и ее частое использование может негативно сказаться на производительности. ПРИМЕЧАНИЕ: Эта функция не может использоваться со встроенными функциями. Синтаксис: script_execute(scr, arg0, arg1, arg2..., etc...); Аргумент Описание scr Имя функции/скрипта, который вы хотите вызвать arg0, arg1, arg2..., etc... Различные аргументы, которые вы хотите передать функции/скрипту Возвращается: Будет зависеть от возвращаемого значения из вызываемого скрипта/функции Пример: script_execute(choose(move_up, move_down, move_left, move_right), irandom(5)); Приведенный выше пример кода будет использовать script_execute для выбора одной из четырех определенных пользователем функций (в данном случае связанной с движением) и передаст ей в качестве первого аргумента случайное целое число.

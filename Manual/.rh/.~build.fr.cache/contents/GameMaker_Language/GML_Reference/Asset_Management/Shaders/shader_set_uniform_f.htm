shader_set_uniform_f С помощью этой функции можно установить значение (или значения) константы шейдера. Предварительно вы должны получить "хэндл" константы с помощью функции shader_get_uniform()и вы должны знать тип константы, чтобы передать ей правильное количество значений с плавающей точкой, т.е. если у вас vec2, то в функцию нужно передать два значения. Исключением из этого правила является работа с 32-битными значениями цвета. Вы можете передать в шейдер до восьми 32-битных цветовых значений, и они будут автоматически преобразованы в vec4 значения 0 - 1. Однако для этого вы должны использовать имя константы шейдера in_Colour для одного цвета или in_Colour0, in_Colour1 и т.д. вплоть до in_Colour7 для нескольких входных значений. Следующие примеры показывают, как это будет выглядеть в самом шейдере: //Один цвет атрибут vec2 in_Position; атрибут vec4 in_Colour; атрибут vec2 in_TextureCoord; //Множество цветов атрибут vec2 in_Position; атрибут vec4 in_Colour0; атрибут vec4 in_Colour1; атрибут vec2 in_TextureCoord; Обратите внимание, что если вы хотите делать сжатые нормали и т.д., используя 4-байтовый формат цвета, то вам также придется использовать формат "in_Colour", как показано выше. ПРИМЕЧАНИЕ: Все формы должны быть установлены после вызова функции shader_set()и до вызова shader_reset(). Синтаксис: shader_set_uniform_f(handle, value1 [, value2, value3, value4]); Аргумент Описание ручка Ручка константы шейдера, которую нужно установить. значение1 ... значение4 Значение (или значения) с плавающей точкой, на которое нужно установить константу шейдера. Возвращается: Н/Д Пример: shader_set(shader_Glass); shader_params= shader_get_uniform(shader_glass, "u_vParams"); shader_set_uniform_f(shader_params, 0.1, 0.8, 0.25); draw_self(); shader_reset(); Приведенный выше код получит хэндл шейдерной константы "u_vParams" (a vec3), затем установит эту константу в заданные значения.

постоянный Эта переменная может быть прочитана, чтобы узнать, помечен ли экземпляр как постоянный или нет, или она может использоваться для установки постоянства в true (постоянный) или false (не постоянный) для данного экземпляра. Постоянный экземпляр - это экземпляр, который "переносится" из комнаты в комнату, то есть (например) его нужно создать только один раз в начале игры, и он будет присутствовать во всех последующих комнатах. С постоянством следует быть осторожным, поскольку легко потерять отслеживание постоянных экземпляры, что может привести к проблемам на более поздних этапах разработки игры. Когда вы создаете экземпляр с включенной персистентностью в комнате, эта переменная будет установлена в true, и ему будет присвоен слой или глубина (в зависимости от того, какую функцию вы использовали для создания экземпляра объекта). Когда комната будет изменена, и если следующая комната не имеет слоя с тем же именем или глубиной, что и назначенный слой, то для экземпляра будет создан новый слой, сохраняющийся между комнатами. Если при создании экземпляра вы присвоили слою имя, то новый слой будет назван так же, как и исходный слой, а если вы назначили глубину для экземпляра, то новый слой будет назван "_layer_XXX", где "XXX" - шестнадцатеричное значение, используемое для присвоения слою уникального имени. имя. Эти слои будут удалены из комнаты при завершении работы комнаты. Еще один момент, касающийся слоев и постоянных экземпляров, заключается в том, что если при создании экземпляра вы присвоили ему именованный слой, а в следующих комнатах есть еще один слой с с тем же именем, то сохраняемый экземпляр будет назначен слою с тем же именем, независимо от глубины слоя. Наконец, если сохраняемый экземпляр перемещается в комнату со слоем, находящимся на той же глубине, на которой был создан экземпляр, он не будет назначен этому слою, а вместо него будет создан новый слой на той же глубине (в соответствии с соглашением об именовании, описанным ранее). У постоянного объекта по-прежнему будут срабатывать события Game Start, Game End, Room Start и Room End, однако если вы перезапустите игру (например, с помощью функции game_restart() ), все постоянные объекты будут будут удалены и будут существовать только при повторном создании игрой. Также обратите внимание, что если вы деактивируете постоянный объект, он больше не будет переходить из одной комнаты в другую, если только не будет повторно активирован до срабатывания события Room End. Синтаксис: стойкий; Возвращается: Булево Пример: if !persistent { persistent = true; } Приведенный выше код проверит экземпляр на предмет того, отмечена ли персистентность как false, и если экземпляр НЕ персистентен, то флаг персистентности будет установлен в true.

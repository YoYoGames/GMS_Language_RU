экземпляр_изменения Вы можете использовать эту функцию для изменения одного экземпляра объекта на другой экземпляр другого объекта и при этом решить, следует ли выполнять события уничтожения и очистки исходных экземпляров и событие создания новых экземпляров. Таким образом, вы можете заставить (например) бомбу превратиться во взрыв - в этом случае аргумент "perf", вероятно, будет верен, поскольку вы захотите, чтобы бомба выполнила свои события уничтожения и очистки, а взрыв - событие создания. или персонаж игрока может измениться на другого персонажа - в этом случае аргумент "perf" будет ложным, поскольку вы не хотите, чтобы экземпляры выполняли свои события Create и Destroy/Clean Up. Стоит отметить, что изменение экземпляра означает, что вы не должны выполнять никаких дальнейших действий с этим экземпляром до следующего шага, в частности, пытаться получить доступ к переменным и т.д., так как это приведет к ошибке. По сути, вы изменяете экземпляр, но он фактически недоступен до конца текущего шага, поэтому прямой доступ к содержащимся в нем переменным (например, вызов obj_Changed.x) не сработает. ВНИМАНИЕ! При изменении экземпляра с включенной физикой физические свойства не будут перенес ены на новый экземпляр. Поэтому вы должны иметь код для "переноса" физического состояния текущего экземпляра в новый экземпляр, или определить физические свойства нового экземпляра. или определить физические свойства нового экземпляра в событии Create Event или в редакторе объектов. По этой причине рекомендуется не использовать эту функцию с экземплярами с включенной физикой, а использовать комбинацию instance_destroy() и instance_create_layer(). Синтаксис: instance_change(obj, perf); Аргумент Описание obj Новый объект, в который превратится вызывающий объект. перф Выполнять ли события создания и уничтожения этого нового объекта (true) или нет (false). Возвращается: Н/Д Пример: if keyboard_check(vk_enter) { instance_change(obj_Player_Swimming, false); exit; } Приведенный выше код изменит вызывающий экземпляр на экземпляр "obj_Player_Swimming" без выполнения события уничтожения исходного экземпляра и события создания нового экземпляра, когда пользователь нажмет клавишу "Enter", выйдя из блока кода по завершении работы. после завершения работы.

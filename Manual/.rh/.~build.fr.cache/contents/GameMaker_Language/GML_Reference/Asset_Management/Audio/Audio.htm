Аудио GameMaker Studio 2 имеет полноценный звуковой движок, основанный на звуковых форматах *.ogg, *.mp3 и *.wav. Звуки этих типов, добавленные в IDE, могут быть использованы в вашей игре с помощью основных аудиофункций, показанных ниже. Для более сложных вещей, чем базовые звуковые эффекты или воспроизведение одного музыкального фрагмента, вы можете обратиться к расширенным аудиофункциям, которые позволяют изменить способ воспроизведения звука. Есть также ряд более специализированных функций, предназначенных для потоковой передачи аудио, позиционированию аудио - для получения 3D-звука - и группировке аудио. ВАЖНО! При использовании звука в HTML5-цели следует помнить, что не все браузеры поддерживают Web Audio, поэтому при запуске ваш проект может не воспроизводить звук. Вы можете получить общее представление о поддержке Web Audio по следующей ссылке: Могу ли я использовать Web Audio? Следующие функции предназначены для работы со звуком наиболее простым и понятным способом, избегая использования эмиттеров и позволяя пользователю легко генерировать звуки и воспроизводить музыку, поскольку эти звуки всегда генерируются в позиции слушателя (более подробную информацию о слушателе см. в разделе по ссылке ниже) и поэтому, как правило, не зависят от каких-либо изменений в слушателе: audio_exists audio_get_name audio_get_type audio_play_sound audio_play_sound_at аудио_пауза_звук audio_pause_all audio_resume_sound audio_resume_all audio_stop_sound audio_stop_all audio_is_playing audio_is_paused audio_create_stream audio_destroy_stream Следующие функции предназначены для обеспечения большего контроля над работой звукового движка и над тем, как воспроизводимые через него звуки будут "слышны" слушателю. Поэтому рекомендуется, чтобы вы хорошо знали, как работает остальной звукового движка GameMaker Studio 2, прежде чем использовать следующие функции: audio_sound_set_track_position audio_sound_get_track_position audio_sound_set_listener_mask audio_sound_get_listener_mask аудио_звук_длительность audio_sound_pitch audio_sound_get_pitch audio_sound_is_playable audio_falloff_set_model аудио_звук_усиление audio_sound_get_gain audio_master_gain audio_set_master_gain audio_get_master_gain номер аудио_канала аудио_отладка Дополнительные расширенные функции, такие как настройка слушателей, запись звука или синхронизация нескольких аудиодорожек во времени, можно найти в подразделах, перечисленных ниже: Аудиоизлучатели Слушатели аудио Аудиогруппы Аудио буферы Синхронизация звука Веб-аудио При создании игр для целевой платформы HTML5 звуковой движок требует поддержки Web Audio, а это, в свою очередь, означает, что иногда звук не будет воспроизводиться, когда или как вы ожидаете. Это происходит потому, что контекст Web Audio может быть не запущен или может перестать работать во время воспроизведения игры. Причины этого сильно различаются в разных браузерах и даже в разных версиях одного и того же браузера, поэтому обнаружение изменения состояния контекста веб-аудио очень важно, например: для обнаружения когда контекст меняется и приостановить/запустить зацикленное аудио, например, фоновую музыку. Чтобы помочь решить эту проблему, в GameMaker Studio 2 есть два отдельных способа обнаружения изменения состояния контекста Web Audio, либо с помощью следующей функции: audio_system_is_available Вы также можете использовать асинхронное системное событие, которое будет срабатывать всякий раз, когда изменяется статус Web Audio. В этом событии вы получите встроенную async_load DS map, заполненную ключом " event_type", который в свою очередь будет содержать строку " audio_system_status", если это аудиособытие. Когда этот ключ существует, также будет существовать дополнительныйключ"status", который будет будет либо "доступен", либо"недоступен". Обратите внимание, что это событие будет срабатывать на всех платформах, но на всех, кроме HTML5, оно будет срабатывать только один раз при запуске игры, когда аудиодвижок будет впервые инициализируется.

актив_имеет_теги С помощью этой функции вы можете проверить, присвоена ли одна или несколько строк тегов любому активу из браузера активов. Вы передаете либо имя актива (в виде строки), либо его индекс, а также либо одну строку тегов, либо массив, где каждый элемент является одной строкой тегов. Если вы указываете значение индекса актива, то вам необходимо указать дополнительный аргумент тип актива (константа), так как активы разных типов могут иметь одинаковый индекс, хотя они не могут иметь одинаковое имя. Доступные типы активов перечислены в таблице ниже: Постоянная Описание актив_объекта Данное имя относится к объекту. актив_спрайт Данное имя относится к спрайту. актив_звук Данное имя обозначает звук. комната_активов Данное имя относится к помещению. актив_плитки Данное название относится к набору плиток. путь_актива Данное имя относится к пути. актив_скрипт Данное имя относится к сценарию. asset_font Данное название относится к шрифту. временная шкала активов Данное имя относится к временной линии. asset_shader Данное имя относится к шейдер. asset_animationcurve Данное название относится к кривой анимации. последовательность активов Данное имя относится к последовательности. Если функция работает успешно и все заданные теги присутствуют для актива, то она возвращает true, в противном случае возвращается false. Если вам нужно проверить наличие не всех тегов, а любого из выбранных, используйте функцию функцию asset_has_any_tag(). Синтаксис: asset_has_tags(name_or_index, tags, [asset_type]); Аргумент Описание имя_или_индекс Название актива (строка) или значение его индекса (целое число). теги Одна строка тега актива или массив с различными тегами актива. [asset_type] ВАРИАНТ! Тип актива для проверки тегов, используется только при указании индекса для первого аргумента. Возвращается: Булево Пример: var _a = array_create(3); _a[0] = "враг"; _a[1] = "уровень_" + string(global.Level); _a[2] = "босс"; if asset_has_tags(object_index, _a, asset_object) { instance_create_layer(0, 0, "Overlay", obj_Boss_Text); } Приведенный выше код создаст массив тегов, затем проверит, все ли из них применены к данному объекту, и если да, то создаст экземпляр другого объекта.

ОС и компилятор При создании кроссплатформенных игр часто бывает крайне важно получить подробную информацию об операционной системе, которую использует устройство, на котором работает ваша игра. Такие вещи, как язык, версия или состояние сети, могут быть использованы для адаптации вашей игры к системе, на которой она работает, и для обеспечения наилучшего опыта конечного пользователя. GameMaker Studio 2 имеет ряд специальных функций, которые можно использовать для получения необходимой информации. Следующие функции и переменные можно использовать для получения определенных сведений об операционной системе или браузере, в котором запущена ваша игра: os_browser os_device os_type os_version os_is_paused os_is_network_connected os_get_config os_get_language os_get_region os_get_info os_powersave_enable os_lock_orientation os_check_permission os_request_permission Некоторые функции предварительно компилируются при запуске игры (т.е. они используются для определения того, как будет скомпилирована конечная игра), а некоторые также используются после компиляции игры для получения конкретных сведений о среде выполнения. Здесь вы можете найти полный список этих функций: GM_build_date GM_version GM_runtime_version gml_release_mode gml_pragma счётчик параметров строка_параметра environment_get_variable В Windows разрешение планировщика потоков Windows по умолчанию устанавливается на 1 мс при запуске игры. Для получения и изменения разрешения планировщика потоков Windows во время выполнения доступны следующие функции: scheduler_resolution_get scheduler_resolution_set Наконец, при работе с расширениями вам может понадобиться возможность определять и вызывать внешние функции во время выполнения, и тогда вы будете использовать следующее: ПРИМЕЧАНИЕ: Это унаследованные функции, и вы должны определить любые функции расширения в самом редакторе расширений.

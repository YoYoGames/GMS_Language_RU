внешнее определение Эта функция может использоваться для определения вызова внешней функции к определенной dll (для Windows) или dylib (для Mac). Этот файл может быть как включенным файлом, так и частью расширения. Вы указываете имя (и путь) файла, затем имя функции. которую вы хотите определить. Далее необходимо определить используемое соглашение о вызове (см. список констант ниже), а также тип ожидаемого результата (также константа, как указано ниже). Наконец, вы должны указать число аргументов, которые функция может принимать (от 0 до 15), и для каждого аргумента необходимо указать его тип. Обратите внимание, что для функций с 4 и более аргументами все они должны быть типа ty_real. ПРИМЕЧАНИЕ: Эта функция предназначена только для dll или dylib, которые были добавлены как "Included Files" в IDE GameMaker Studio 2. Это не будет работать с файлами, добавленными в качестве расширений, поскольку для них требуется определение функций в самом пакете расширения. в самом пакете расширения. Постоянная Описание dll_cdecl Это вызов по умолчанию в C, C++ dll_stdcall Это стандартный вызов WinAPI( только дляWindows dll ) ty_real Аргумент вещественного числа ty_string строковый аргумент с нулевым окончанием Синтаксис: external_define(dll, name, calltype, restype, argnumb, argtype[0], argtype[1], ...argtype[10]) ; Аргумент Описание dll Имя файла dll (строка) имя Имя функции (строка) тип звонка Используемая конвенция вызова restype Тип ожидаемого результата argnumb Количество аргументов (0 - 10) argtype[0 ... 10] Различные типы используемых аргументов Возвращается: Real (Уникальное значение идентификатора функции) Пример: my_funcion = external_define("MyDLL.dll", "MyMin", dll_cdecl, ty_real, 2, ty_real, ty_real); В приведенном выше примере кода будет определена внешняя функция под названием "MyMin" с двумя аргументами.

Ошибки бегунов Даже после проверки синтаксиса в редакторе кода и последующей проверки кода компилятором на наличие ошибок компилятора, все равно бывают случаи, когда что-то может пойти не так. В большинстве случаев это приведет к возникновению ошибки выполнения виртуальной машины (VM) (также называемой исключениемвремени выполнения), которая выглядит следующим образом: Ошибки бегущей строки обычно более серьезны, чем ошибки компиляции, поскольку это означает, что в коде что-то серьезно не так. в вашем коде, что ни редактор кода, ни компилятор не смогли обнаружить, и поэтому вы должны обращать внимание на все такие ошибки. При возникновении такой ошибки вы можете воспользоваться кнопкой Copy на всплывающем окне, чтобы скопировать ошибку в буфер обмена которую затем можно вставить в текстовый файл (или куда-либо еще) для дальнейшего использования. Структура этой ошибки выглядит следующим образом: Здесь показано событие, в котором произошла ошибка, а также объект (если применимо). Здесь показана фактическая ошибка. Таким образом, здесь показан масштаб ошибки и тип ошибки. Первое значение в скобках будет ID экземпляра, на котором выполнялся код - или это может быть одно из специальных значений, перечисленных в таблице ниже - а второе является внутренним значением, которое YoYo Games использует для выявления ошибок и может быть проигнорировано. В случае с изображением выше, это говорит о том, что переменная _image не была установлена (т.е. не существует до обращения к ней) на объекте obj_EA_DEMO_Control. Здесь показан номер строки, на которой возникла ошибка, а также фрагмент кода, показывающий позицию. Обратите внимание, что это не всегда на 100% точно из-за способа компиляции игры, но этого должно быть достаточно, чтобы помочь вам найти точное место проблемы. Как упоминалось выше, некоторые сообщения об ошибках будут идентифицироваться не по значению ID экземпляра, а по отрицательному значению. DoConv Ошибка выполнения Типы данных Это означает ошибку при преобразовании одного типа данных в другой. DoAdd Ошибка выполнения Математика Что-то пошло не так при использовании выражения сложения (+). DoMul Ошибка выполнения Математика Что-то пошло не так при использовании выражения умножения (*). DoSub Ошибка выполнения Математика Что-то пошло не так при использовании выражения вычитания (-). DoSub Execution Engine - Невозможно выполнить операцию над строковым типом Математика Вы пытаетесь вычесть переменные не того типа (например, вычесть строку из real). DoDiv Ошибка выполнения Математика Что-то пошло не так при использовании выражения деления (/ или div). DoDiv Execution Engine - Невозможно выполнить операцию над строковым типом Математика Вы пытаетесь разделить переменные не того типа (например, разделить строку на вещественную). DoDiv Деление на ноль Математика Вы пытаетесь разделить на 0 (обратите внимание, что это произойдет только при использовании целочисленного деления, деление (ненулевого) вещественного числа на 0 даст бесконечность в качестве ответа, деление нуля на ноль даст NaN в качестве ответа). Вы можете проверить наличие этих значений с помощью (is_infinity) и (is_nan) DoMod Ошибка выполнения Математика Что-то пошло не так при использовании выражения modulo (mod). DoMod Execution Engine - Невозможно выполнить операцию над строковым типом Математика Вы пытаетесь использовать modulo (mod) для переменных неправильного типа (например, mod строку на вещественную). DoAnd Ошибка выполнения Побитовая Что-то пошло не так при использовании побитового выражения "и" (&). DoAnd Execution Engine - Невозможно выполнить операцию над строковым типом Побитовая Вы пытаетесь использовать побитовое "и" (&) для переменных неправильного типа (например, пытаетесь "и" строки с вещественным числом). DoOr Ошибка выполнения Побитовая Что-то пошло не так при использовании побитового выражения "или" (|). DoOr Execution Engine - Невозможно выполнить операцию над строковым типом Побитовая Вы пытаетесь использовать "или" (|) для переменных неправильного типа (например, пытаетесь "или" строку с вещественным числом). DoXor Ошибка выполнения Побитовая Что-то пошло не так при использовании побитового выражения "xor" (^). DoXor Execution Engine - Невозможно выполнить операцию над строковым типом Побитовая Вы пытаетесь использовать "xor" (^) для переменных неправильного типа (например, пытаетесь "xor" строки с вещественным числом). DoShl Ошибка выполнения Побитовая Что-то пошло не так при битовом сдвиге влево (<<) значения. DoShl Execution Engine - Невозможно выполнить операцию над строковым типом Побитовая Вы пытаетесь выполнить левый битовый сдвиг (<<) переменных не того типа (например, пытаетесь выполнить битовый сдвиг строки). DoShr Ошибка выполнения Побитовая Что-то пошло не так при битовом сдвиге вправо (>>) значения. DoShr Execution Engine - Невозможно выполнить операцию над строковым типом Побитовая Вы пытаетесь выполнить правый битовый сдвиг (>>) переменных не того типа (например, пытаетесь выполнить битовый сдвиг строки). DoNeg Ошибка выполнения Отрицание Вы пытаетесь превратить тип переменной в отрицательный, когда этот тип не допускает такой операции. DoNot Ошибка выполнения Отрицание Вы пытаетесь "не" тип переменной, когда этот тип не допускает такой операции. Push Получение переменной (var) Стек Данная переменная не определена или неизвестна. With Невозможно использовать global в операторе with С Вы пытались использовать "global" в качестве переменной в операторе "with", т.е: with (global)    {    //do something;    } With Невозможно использовать local в операторе with С Вы пытались использовать "local" в качестве переменной в операторе "with", т.е: with (local)    {    //do something;    } DoCall Ошибка типа исполнительного механизма Двигатель Это неопределенная ошибка в виртуальной машине. В этом случае необходимо подать отчет об ошибке (как это сделать, см. в меню Справка. Stack Overflow - Двигатель Переполнение стека происходит, когда в стеке вызовов используется слишком много памяти, и когда ваша игра пытается использовать больше места, чем доступно в стеке вызовов (то есть, когда она пытается получить доступ к памяти за пределами стека вызовов, что по сути является переполнение буфера), стек, как говорят, переполняется, что приводит к аварийному завершению программы.

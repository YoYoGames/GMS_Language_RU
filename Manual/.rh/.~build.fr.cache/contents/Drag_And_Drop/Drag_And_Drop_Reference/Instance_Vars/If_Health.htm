Если здоровье Это действие используется для проверки значения переменной экземпляра health с помощью определенного выражения. Вы указываете тип выражения для проверки и значение для проверки текущего здоровья, а оператор "if" будет всегда будет возвращать либо true, либо false в зависимости от используемых выражений и значений. Доступными выражениями являются: Equals to - Переменная и значение в точности равны. Меньше - Переменная меньше значения Больше чем - Переменная больше, чем значение Less than or Equal to - Переменная меньше или равна значению Greater than or Equal to - Переменная больше или равна значению Если вы отметите аргумент Not, то вышеприведенные выражения будут отрицательными, например, "equals to" станет " not equals to",таким образом, вы будете проверять, не равно ли значение здоровья заданному значению. ВАЖНОЕ ПРИМЕЧАНИЕ: Из-за проблем с точностью плавающей точки, проверка на точное равенство двух значений может вернуть ложное значение, так как одно из них может быть равно 1, а другое - 1.00000000000001. другое может быть 1.00000000000001. Этого можно избежать, используя действие Decimal to Integer перед проверкой или используя выражения "больше чем" или "меньше чем". Обратите внимание, что для добавления действий в блок "if", их следует опустить сбоку от действия, как показано на изображении ниже: Эти действия теперь будут выполняться, если "if" оценивается в true, в то время как любые действия, отброшенные в другом месте, будут выполняться после блока "if". Синтаксис действия: Аргументы: Аргумент Описание Не Устанавливается для проверки, если выражение не имеет значения true. Выражение Тип выражения, используемый для проверки. Значение Значение, по которому проверяется здоровье. Пример: Приведенный выше код блока действий добавит 25 к переменной health, а затем проверит, не стала ли переменная больше 100. переменная теперь больше 100, и если это так, то она будет установлена на 100.

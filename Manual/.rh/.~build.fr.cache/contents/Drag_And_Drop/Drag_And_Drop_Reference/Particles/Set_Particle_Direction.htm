Установить направление частиц Это действие используется для установки направления частицы, которое может быть статичным или увеличиваться/уменьшаться с течением времени. Вы предоставляете уникальное значение ID типа для установки размера (полученное при создании типа с помощью действия Create Particle Type) и затем задаете диапазон начальных направлений, затем величину увеличения и, наконец, "вихляние". Направление, в котором создается частица, рассчитывается так: 0 - вправо, 90 - вверх, 180 - влево и 270 - вниз. -45 и максимальное значение 45 будет создавать частицы, движущиеся, например, только вправо. Опция увеличения направления частицы позволяет ввести значение, которое будет добавляться к базовому значению направления каждый игровой кадр времени жизни частицы. Таким образом, если установить это значение, например, 0,1, то направление будет прибавляться к 0,1 каждый игровой кадр. Обратите внимание, что здесь можно использовать и отрицательные значения, чтобы заставить частицу двигаться в противоположном направлении вниз, поэтому положительные значения заставят частицу вращаться против часовой стрелки, а отрицательные - по часовой. Обратите внимание, что минимальный принимаемый декремент направления - -0,01, а минимальный инкремент - 0,01. Наконец, вы можете установить параметр "wiggle". Это значение, которое будет генерироваться случайным образом между 0 и заданным вами значением, а затем добавляться или вычитаться к направлению на каждом шаге жизни частиц, заставляя нарисованное изображение колебаться в направлении. Очевидно, что большие значения более выражены, чем меньшие, и это значение может быть даже отрицательным с максимальным диапазоном от -20 до 20. Синтаксис действия: Аргументы: Аргумент Описание Тип Уникальное значение ID для типа частицы, чтобы задать направление Мин. дир Минимальное направление для типа частицы (от -360 до +360, по умолчанию 0) Макс. дир Максимальное направление для типа частиц (от -360 до +360, по умолчанию 360) Увеличить Значение, добавляемое к направлению типа частиц в каждом игровом кадре (может быть отрицательным, минимум +/-0.01). Wiggle Максимальное значение, которое будет произвольно добавляться или вычитаться из направления частиц за шаг (+/-20). Пример: Приведенный выше код блока действий создаст новый тип частиц и присвоит его уникальное ID значение глобальной переменной. Затем он переходит к установке всех свойств для типа частицы.

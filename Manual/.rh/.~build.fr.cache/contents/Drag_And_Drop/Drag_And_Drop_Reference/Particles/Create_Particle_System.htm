Создать систему частиц Это действие инициализирует систему частиц, чтобы вы могли создавать и использовать в ней типы частиц и излучатели. Вы задаете имя слоя в виде строки - например, "Particle Layer" (который должен существовать в редакторе комнат) - и затем отмечаете систему как систему как постоянную или нет. Если система отмечена как постоянная, то она останется активной во всех комнатах после создания, и ей будет назначен управляемый слой, если исходный слой не существует в последующих комнатах. Если флаг постоянства не установлен, то система будет автоматически уничтожена в конце комнаты, в которой она была создана, как и все эмиттеры, связанные с ней. Если вы хотите вручную удалить систему из памяти, вам необходимо использовать действие Уничтожить систему частиц. Вам также необходимо предоставить целевую переменную, которая будет содержать уникальное значение ID системы частиц. Это значение ID будет использоваться в других функциях для установки свойств системы или для указания GameMaker Studio 2, какую систему использовать при создании частиц. Вы можете отметить эту переменную как временную локальную переменную, но это будет означать, что она будет отброшена в конце сценария действия или события, то есть система все еще будет существовать, но у вас не будет никаких средств доступа к ней, поэтому при ее использовании следует соблюдать осторожность. (в общем случае системы никогда не нужно хранить во временной локальной переменной, но для некоторых вещей, таких как притягивание частиц к поверхности, это может потребоваться). ПРИМЕЧАНИЕ: Постоянная система частиц после создания остается в памяти системы на все время игры, поэтому если вам нужно воссоздать ее в любой момент (например, при вызове действия Restart Game ), то сначала необходимо уничтожить текущую систему, иначе произойдет утечка памяти. Утечка памяти происходит, когда ресурс добавляется в память, а затем "отсылается", то есть GameMaker Studio 2 больше не имеет возможности взаимодействовать с этим ресурсом. этим ресурсом. Это приводит к засорению памяти недоступными ресурсами, что, в свою очередь, может вызвать проблемы в вашей игре. Чтобы уничтожить постоянную систему частиц, если она больше не нужна, используйте действие Destroy Particle System. Синтаксис действия: Аргументы: Аргумент Описание Слой Слой, в котором будут рисоваться частицы в этой системе Постоянный Должна ли система частиц оставаться активной после завершения текущей комнаты Цель Целевая переменная для хранения уникального значения ID созданной системы Пример: В приведенном выше блоке действий создается глобальная переменная, затем создает новую систему частиц и присваивает ее уникальное значение ID глобальной переменной. Созданная система частиц помечена как постоянная, поэтому она останется активной даже после завершения текущей комнаты.

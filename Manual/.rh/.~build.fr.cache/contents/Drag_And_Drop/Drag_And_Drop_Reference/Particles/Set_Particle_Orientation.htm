Установить ориентацию частиц Это действие используется для установки ориентации частиц - угла, под которым будут рисоваться изображения частиц, - которая может быть статичной или увеличиваться/уменьшаться с течением времени. Вы предоставляете уникальное значение ID типа для установки размера (полученное при вы создали тип с помощью действия Create Particle Type), затем задаете диапазон начальных ориентаций, затем величину увеличения и, наконец, "вихляние". Ориентация, в которой создается частица рассчитывается так: 0 - вправо, 90 - вверх, 180 - влево и 270 - вниз, поэтому при минимальном значении 0 и максимальном значении 360 частицы будут испускать частицы, каждая из которых будет повернута на случайную величину вокруг 360° (обратите внимание, что это не влияет на направление частиц). Опция увеличения ориентации частицы позволяет ввести значение, которое будет добавляться к базовому углу ориентации каждый игровой кадр времени жизни частицы. Таким образом, при установке этого значения, например, 0,1, к углу изображения будет добавляться 0,1 каждый игровой кадр. Обратите внимание, что здесь можно использовать и отрицательные значения, чтобы заставить частицу вращаться в противоположном направлении, поэтому положительные значения заставят частицу вращаться против часовой стрелки, а отрицательные - по часовой. Обратите внимание, что минимальное значение ориентации -0.01, а минимальное увеличение - 0.01. Вы также можете установить параметр "wiggle" для ориентации частицы. Это значение будет генерироваться случайным образом в диапазоне от 0 до заданного вами значения и затем добавляться или вычитаться к углу изображения частицы на каждом этапе ее жизни, в результате чего нарисованное изображение будет колебаться при вращении. Очевидно, что большие значения более заметны, чем меньшие, и это значение может быть даже отрицательным с максимальным диапазоном от -20 до 20. Наконец, вы можете выбрать ориентацию относительно направления частицы или нет. Это означает, что пока частица имеет заданное направление (и скорость), изображение частицы будет ориентировано на основе этого вектора. Имейте в виду, что если скорость частицы установлена на уменьшение и достигает 0 во время жизни частицы, отсутствие скорости устанавливает направление на значение по умолчанию 0°, и поэтому относительная ориентация заставит изображение частицы "перепрыгнуть" на другой угол ориентации. Синтаксис действия: Аргументы: Аргумент Описание Тип Уникальное значение идентификатора типа частицы для установки ориентации Минимальный угол Минимальная ориентация для типа частицы (от -360 до +360, по умолчанию 0). Максимальный угол Максимальная ориентация для типа частицы (от -360 до +360, по умолчанию 360) Увеличить Значение, добавляемое к ориентации типа частиц в каждом игровом кадре (может быть отрицательным, минимум +/-0.01). Wiggle Максимальное значение для случайного добавления или вычитания из ориентации частиц за шаг (+/-20). Относительно направления Устанавливать ли угол изображения частицы относительно направления движения частицы (включено) или нет (отключено). Пример: Приведенный выше код блока действий создаст новый тип частиц и присвоит его уникальное ID значение глобальной переменной. Затем он переходит к установке всех свойств для типа частицы.

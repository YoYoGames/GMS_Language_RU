Библиотека действий буферов Буфер - это область физической памяти, используемая для временного хранения данных, пока они перемещаются из одного места в другое или манипулируются каким-либо образом, например, вы можете использовать буфер для хранения данных состояния игры. В GameMaker Studio 2 есть ряд определенных действий, связанных с созданием, изменением и загрузкой/сохранением буферов, но перед их использованием обратите внимание на следующие моменты: При создании буфера он автоматически очищается и заполняется до 0. Аргумент "size" функций буфера всегда указывает на размер буфера в байтах . БуферыDnD™всегда выровнены по 1 байту. Выравнивание" относится к тому, как данные хранятся в буфере, и используется при использовании функций буфера GML , но для Drag And Drop это не имеет значения. Данные в буферах считываются и записываются последовательно (то есть: вы вызываете действие Write Buffer три раза, тогда оно запишет первые данные, переместится вперед, справа вторые данные, переместится вперед и затем запишет последние данные). последние данные), с чем-то, называемым значением "seek", чтобы сказать вам, где в буфере будет происходить следующее чтение/запись. Все данные записываются в буфер и возвращаются из буфера в виде строки, поэтому, если вы храните какие-либо вещественные числа, вам придется использовать одно из действий с типом данных для преобразования их из строки в значение. Память, используемая для создания буферов, является системной, поэтому даже когда игра не имеет фокуса, все данные, хранящиеся в буфере, должны быть в безопасности, однако если приложение будет закрыто или перезапущено, они будут потеряны. Стоит также отметить, что буферы можно также можно сохранять в виде файлов, а затем загружать их обратно в игру с помощью действий с файлами.

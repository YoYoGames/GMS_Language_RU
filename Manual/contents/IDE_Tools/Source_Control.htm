Контроль источников

Управление контролем исходного кода (SCM) - это название, данное методу работы с подверсиями или резервными копиями ваших проектов через репозиторий и локальный источник. По сути, SCM-решение - это независимый программный пакет. который контролирует все аспекты поддержания, изменения и сравнения версий вашего проекта в процессе работы над ним. Это особенно полезно для тех, кто работает в команде и нуждается в возможности контролировать, кто что делает, и не беспокоиться о потере данных или не беспокоиться о потере данных или внесении изменений, которые впоследствии придется отменять, но и индивидуальные пользователи могут воспользоваться этой мощной, но гибкой системой.

Существуют различные инструменты SCM, а GameMaker Studio 2 поставляется с плагином Git, который можно использовать сразу же, и вам не нужно устанавливать никаких дополнительных пакетов, поскольку все это поставляется вместе с самим плагином. Ниже вы можете найти ссылки на небольшое руководство по установке этого плагина и использованию инструментов SCM в проекте.

ВАЖНО! Мы рекомендуем вам иметь некоторые рабочие знания о том, как работает контроль исходных текстов, прежде чем пытаться использовать его в GameMaker Studio 2.

  • Настройка плагина Source Control Git Plugin
  • Параметры репозитория
  • Стандартный рабочий процесс
  • Реверсирование файлов
  • Клонирование хранилища
  • Конфликты
  • Настройка внешних инструментов слияния/разделения

Когда вы активируете Source Control для любого проекта (подробнее см. раздел Настройка Git-плагина Source Control выше), GameMaker Studio 2 отобразит следующее контекстное меню в в верхней части IDE:

Здесь мы опишем каждый из доступных вариантов (большинство из них более подробно описаны в разделах учебника выше):

  • Создать репозиторий проекта: Эта опция позволяет создать локальный репозиторий в том же каталоге, где находятся файлы проекта.
  • Импорт проекта в репозиторий: Эта опция позволяет импортировать проект во внешний репозиторий.
  • Клонировать репозиторий: Эта опция позволит вам клонировать хранилище из источника в новое место назначения.
  • Зафиксировать изменения: С помощью этой опции вы можете отобразить измененные файлы в вашем проекте и затем зафиксировать их в репозитории.
  • Push Changes: После выполнения фиксации или слияния эта опция используется для переноса изменений в основной репозиторий.
  • Pull Changes: С помощью этой опции вы можете обновить локальный репозиторий, извлекая измененные файлы из главного репозитория.
  • Просмотреть историю: Откроется окно истории, где можно просмотреть всю историю версий проекта и выбрать откат определенных путей файлов или целых ревизий.
  • Показать конфликты: Это откроет окно Conflicts и перечислит все конфликтующие файлы, которые могут существовать в проекте, позволяя вам разобраться с ними либо в IDE GameMaker Studio 2, либо с помощью специального инструмента слияния.

xboxlive_matchmaking_find

Эта функция будет искать многопользовательскую сессию, используя шаблон сессии, бункер подбора и шаблон Secure Device Association, который вы должны были предварительно создать в приборной панели XDP. Идентификатор пользователя, который вы можете получить для конкретного игровой панели, вызвав xboxlive_user_for_pad() функцию. Функция вернет уникальное значение Request ID, которое затем может быть использовано для определения правильного социального асинхронного события для этой функции. Это событие будет запущено после создания сессии и будет содержать DS Map в переменной async_load со следующими парами ключ/значение:

  • "requestid" - содержит идентификатор запроса, возвращенный вызывающей функцией
  • "status" - содержит строку "session_created" для информирования о том, какой тип события был вызван.
  • "num_results" - содержит 1 или более, если сессия была найдена, в противном случае содержит 0.
  • "sessionid<index>" - содержит идентификатор сессии, или -1 в случае неудачи (<index/gt; является значением от 0 до "num_results" - 1).
  • "sessionOwner<index>" - содержит ID хозяина сессии, если сессия была найдена (<index> - это значение от 0 до "num_results" - 1).
  • "error" - содержит 0 в случае успеха, или -1 в случае неудачи

Синтаксис:

xboxlive_matchmaking_find(user_id, template, hopper, sdatemplate, [matchattributes]);

Аргумент Описание
идентификатор пользователя Указатель идентификатора пользователя для использования
шаблон Имя шаблона сеанса
бункер Название бункера для спичек
sdatemplate Имя шаблона ассоциации защищенных устройств
matchattributes [ВАРИАНТ]

Возвращается:

Идентификатор запроса (целое число)

Пример:

var userID = xboxone_user_for_pad(global.PadIndex);
result = xboxone_matchmaking_find(userID, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic");

Вышеприведенная команда получит идентификатор пользователя на данном геймпаде, а затем попытается найти для него сеанс подбора.

network_send_broadcast

С помощью этой функции вы можете транслировать данные из буфера локально на ряд IP-адресов (более подробную информацию о буферах см. в разделе Справочник - Буферы). Диапазон ограничен адресом устройства, на котором работает сервер, например, если устройство имеет IP-адрес 92.168.11.130, то данные будут передаваться по диапазону 92.168.11.*. Функция возвращает количество байт отправленных данных или число меньше 0, если отправка не удалась.

ПРИМЕЧАНИЕ: Эта функция будет работать только при использовании UDP - ваш сервер должен быть TCP, а ваш клиент должен иметь клиентский сокет UDP, созданный с помощью кнопки network_create_socket_ext() чтобы получать любые широковещательные сообщения, отправляемые с сервера.

ПРИМЕЧАНИЕ: Эта функция не будет работать при использовании в проекте, работающем на HTML5-цели.

Синтаксис:

network_send_broadcast(socket, port, buffer, size);

Аргумент Описание
розетка Идентификатор используемого сокета.
порт Порт, который будет использовать сервер.
буфер Идентификатор буфера, из которого будут получены данные.
размер Размер (в байтах) данных.

Возвращается:

Реально

Пример:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

Приведенный выше код записывает строку имени текущего сервера (хранится в "global.ServerName"), затем записывает ее в двоичный буфер с идентификатором "broadcast_buffer". Затем эти данные локально транслируются на ряд IP-адресов (IP-адрес устройства в настоящее время подразумевается как базовый диапазон вещания) на порт 6511.

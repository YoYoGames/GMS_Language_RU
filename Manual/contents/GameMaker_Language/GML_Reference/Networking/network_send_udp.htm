network_send_udp

С помощью этой функции вы можете отправлять данные по сети с помощью UDP на сервер. Функция принимает идентификатор сокета для подключения, URL для подключения и порт для использования. Затем вы должны указать идентификатор буфера, который содержит данные для отправки (для получения информации см. (для получения дополнительной информации о буферах см. раздел " Справочник - Буферы") и, наконец, размер (в байтах) данных. UDP является "бесконнектным" в том смысле, что вы не устанавливаете соединение, вы просто посылаете пакет непосредственно на IP-адрес, а сервер получает входящие данные с IP-адреса и должен работать с ними "как есть". Функция возвращает количество байт отправленных данных или число меньше 0, если отправка не удалась. Стоит отметить, что окончательный размер отправляемых данных, возвращаемый этой функцией, будет также включать информацию заголовка GameMaker, что составляет дополнительные 12 байт.

ПРИМЕЧАНИЕ: Эта функция не будет работать при использовании в проекте, работающем на HTML5-цели, также как и HTML5-проекты не смогут принимать UDP.

Синтаксис:

network_send_udp(socket, url, port, buffer, size);

Аргумент Описание
розетка Идентификатор используемого сокета.
url url или IP для подключения (строка).
порт Порт, к которому необходимо подключиться.
буфер Идентификатор буфера, из которого будут получены данные.
размер Размер (в байтах) данных.

Возвращается:

Реально

Пример:

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

Приведенный выше код отправит UDP-пакет на сервер, заданный URL, по порту 6510. Данные берутся из буфера, индексированного в переменной "buff".

сетевой_создать_сервер

Эта функция используется для создания нового сетевого сервера для вашей игры, используя один из разрешенных протоколов подключения (см. константы, перечисленные ниже). Вы задаете тип сервера, затем указываете порт, который он будет использовать, и, наконец, количество максимальных соединений которое должно быть разрешено в любой момент времени к серверу (обратите внимание, что это число зависит от вас, но слишком большое количество подключенных клиентов насытит сеть или процессор устройства не сможет справиться с обработкой такого количества игроков, поэтому используйте его с осторожностью). осторожностью). Функция возвращает уникальный id, который должен быть сохранен в переменной и использоваться для идентификации сервера во всех дальнейших сетевых функциях, или значение меньше 0, если соединение не удалось.

Постоянная Описание
network_socket_tcp Создайте сокет с помощью TCP.
network_socket_udp Создайте сокет, используя UDP.
network_socket_ws Создайте веб-сокет(только для подключения к проектам HTML5), используя TCP.
network_socket_bluetooth Создайте гнездо Bluetooth(в настоящее время недоступно!).

Синтаксис:

network_create_server(type, port, max_client);

Аргумент Описание
тип Тип создаваемого сервера (см. константы, перечисленные ниже).
порт Порт, который будет использовать сервер.
max_client Максимальное количество клиентов, которые могут подключиться одновременно.

Возвращается:

Реально

Пример:

var port = 6510;
server = network_create_server(network_socket_tcp, port, 32);
while (server < 0 && port < 65535)
{
port++
server = network_create_server(network_socket_tcp, port, 32);
}

Приведенный выше код попытается создать сервер, используя TCP через порт 6510. Если этот порт недоступен, он будет перебирать порты, чтобы найти тот, который доступен.

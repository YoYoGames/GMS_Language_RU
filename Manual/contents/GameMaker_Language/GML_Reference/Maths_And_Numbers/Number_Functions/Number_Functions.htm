Функции чисел

Вещественные числа в GameMaker Studio 2 считаются числами с плавающей точкой двойной точности - то есть они имеют десятичную точку без фиксированного количества цифр до или после точки - или целыми числами - то есть это целые числа без десятичной точки. или целыми числами - то есть это целые числа без десятичной точки. Например, 2 - это целое число, а 2,01 - вещественное число с плавающей точкой.

ПРИМЕЧАНИЕ: В цели HTML5 все действительные числа являются двойными.

Это различие между целыми и плавающими числами очень важно при кроссплатформенной разработке, поскольку точность вычислений, выполненных на компьютере под управлением Windows, не совпадает с точностью тех же самых вычислений на мобильном устройстве. Это означает, что вы должны быть особенно внимательны при выполнении сравнений, например:

if (image_index == 1)
{
сделайте что-нибудь...
}

В приведенном выше примере, если мы устанавливали image_speed на 0.25 - например, - то после четырех шагов вы можете предположить, что значение image_index будет равно 1, но это может быть значение 1.0000002 из-за того, как работает математика с плавающей точкой, и поэтому оценка не будет верной. Подобные ошибки могут быть довольно сложными для отладки, поэтому всегда полезно знать о них и планировать заранее и использовать другие средства проверки значений или использовать соответствующие функции перекрытия или округления (перечислены ниже) для преобразования проверяемого числа в целое (более подробную информацию о математике с плавающей точкой и о том, почему это является проблемой, смотрите здесь). Например, приведенный выше код можно записать как:

if (floor(image_index) == 1)
{
сделайте что-нибудь...
}

Также стоит отметить, что при использовании целей YoYo Compiler все выражения и функции оцениваются слева направо, в то время как на всех других целевых платформах они оцениваются справа налево, Это означает, что, например, это - будет давать разные результаты в зависимости от платформы:

val = max(num, ++num, num++);

Вы также можете использовать специальную функцию, доступную в GameMaker Studio 2, чтобы установить значение эпсилон для математики с плавающей точкой. Когда вещественное число округляется до ближайшего числа с плавающей точкой, эпсилон (также известный как "машинный эпсилон") формирует верхнюю границу относительной ошибки, и вы можете получить и установить значение эпсилона с помощью следующих функций:

  • math_set_epsilon
  • math_get_epsilon

Виртуальные клавиши и виртуальные клавиатуры

При разработке игр для устройств с сенсорным экраном часто бывает важно получить от пользователя ввод с клавиатуры. При использовании GameMaker Studio 2 это можно сделать одним из двух способов:

  • Создайте свои собственные виртуальные клавиши на экране для ввода информации
  • Всплывающая виртуальная клавиатура ОС и получение ввода с нее

То, что вы будете использовать, во многом зависит от игры и того, что вам нужно, но в целом вы будете использовать виртуальные клавиши, когда вам нужны экранные кнопки для управления игроком или для меню, в то время как вы будете использовать виртуальную клавиатуру, когда вам нужен ввод игрока, например, для получения имени или установки значения.

Для того чтобы ваши игры работали должным образом на мобильных устройствах с сенсорным экраном, вам может понадобиться возможность назначить участки экрана, которые будут реагировать на определенные действия при прикосновении к ним пользователя. Это делается путем назначения "виртуальной клавиши" внутренне сопоставленной стандартной клавише клавиатуры, а затем используя генерируемые ею события клавиатуры для управления вашим приложением. Стоит отметить, что эти "виртуальные клавиши" сопоставлены с положением экрана, а не с положением комнаты и поэтому значения x/y являются абсолютными, основанными на размере дисплея. Это означает, что вам не нужно беспокоиться об использовании представлений или относительных координат помещения, а можно просто определить ключи в событии Create объекта (определять виртуальный ключ нужно только один раз для каждого помещения). виртуальный ключ только один раз для каждой комнаты), а затем нарисовать спрайты ключей в событии Draw GUI Event, установив размер слоя GUI равным размеру экрана.

ПРИМЕЧАНИЕ: Эти функции не будут работать на целевых платформах Windows, Ubuntu(Linux) и MacOS, использующих сенсорный экран, хотя они будут воспринимать щелчки мыши как "прикосновения".

Доступны следующие функции виртуальных клавиш:

  • виртуальный_ключ_добавить
  • виртуальный_ключ_показывать
  • virtual_key_hide
  • virtual_key_delete

Если вам требуется ввод текста или цифр в игре на устройстве с сенсорным экраном, то вы можете использовать функции виртуальной клавиатуры, предоставляемые для использования клавиатуры ОС устройства. Эти функции будут вызывать или скрывать клавиатуру ОС и принимать с нее ввод, однако они не будут вызывать обычные события клавиатуры, а скорее обновят переменную строка_клавиатуры переменную с вводом пользователя (это связано с тем, что виртуальная клавиатура может вводить целые слова, а не только отдельные символы).

Открытие или закрытие виртуальной клавиатуры генерирует системные асинхронные события, в которых async_load DS будет заполнена следующими парами ключ/значение:

  • " event_type" - тип полученного системного события,для виртуальных клавиатур это будет " virtualkeyboard status".
  • "screen_height" - высота виртуальной клавиатуры (в пикселях). Это значение будет равно 0, если клавиатура невидима.
  • "keyboard_status" - текущее состояние клавиатуры, возвращается в виде одной из следующих строк:
    • "скрытие"
    • "скрытый"
    • "показывающий"
    • "видимый"

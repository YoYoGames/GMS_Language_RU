точка_столкновения

Collision_point проверяет точку, указанную аргументами x1,y1, на столкновение с любым экземпляром объекта, указанного аргументом "obj". эта проверка может быть как точной, так и нет, но для включения точных столкновения должна быть включена, объект или экземпляр, который вы проверяете, также должен иметь включенные точные столкновения для своего спрайта. В противном случае проверка по умолчанию будет основана на граничных коробках. На следующем изображении показано, как это работает:

Помните, что для учета точных столкновений как спрайт объекта. и функция столкновения должны быть отмечены как точные. Следует также отметить, что возвращаемое значение функции может быть id любого из экземпляров, считающихся столкнувшимися, поэтому если три экземпляра пересекаются в этой точке, то любой из них может быть любой из их идентификаторов может быть возвращаемым значением функции.

Обратите внимание, что вместо индекса объекта вы можете указать экземпляр id для проверки конкретного экземпляра, или ключевые слова экземпляра all, или other (в зависимости от события и текущей области видимости кода).

Синтаксис:

collision_point(x, y, obj, prec, notme);

Аргумент Описание
x Координата x точки, которую нужно проверить.
y Координата y точки для проверки.
obj Объект для проверки столкновений экземпляров.
prec Основана ли проверка на точных столкновениях(true, что медленнее) или на его граничной области в целом(false, быстрее).
notme Должен ли вызывающий экземпляр, если он релевантен, быть исключен(true) или нет(false).

Возвращается:

Идентификатор экземпляра или нет

Пример:

if collision_point(x, y, obj_Cursor, false, true)
{
Score += 10S;
}

Здесь мы проверяем точку в позиции объекта, имеющего код объекта "obj_Cursor". Если такая точка есть, то мы добавляем 10 к переменной score.

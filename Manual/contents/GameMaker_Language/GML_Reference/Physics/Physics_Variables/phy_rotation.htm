фи_ротация

Эта переменная может быть использована для получения (или установки) угла установки экземпляров в градусах, аналогично установке или получению параметра угол_изображения. Однако обратите внимание, что в физическом мире вращение рассчитывается противоположным образом, чем в обычном игровом мире GameMaker Studio 2, а это означает, что векторные функции типа точка_направления()должны будут изменены возвращаемые значения (простое преобразование положительного значения в отрицательное должно решить эту проблему).

Синтаксис:

phy_rotation;

Возвращается:

Real (значение с плавающей точкой одинарной точности, или неопределено, если в экземпляре не включена физика)

Пример:

if (phy_speed_x > 0) || (phy_speed_y > 0)
{
phy_rotation += sqrt(sqr(phy_speed_x) + sqr(phy_speed_y)) /10;
}

Приведенный выше код проверяет линейную скорость, и если любой из векторов не равен 0, он вычисляет фактическую скорость и использует ее для задания вращения.

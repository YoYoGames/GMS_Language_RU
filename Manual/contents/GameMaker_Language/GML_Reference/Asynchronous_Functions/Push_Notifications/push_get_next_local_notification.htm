push_get_next_local_notification

Эта функция заполняет предварительно созданную карту DS серией пар ключ/значение для следующего локального push-уведомления в очереди на показ. Она возвращает -1, если больше нет никаких уведомлений в очереди, или реальное значение, представляющее ID для уведомления. Этот идентификатор может быть использован для отмены уведомления с помощью функции push_cancel_local_notification(). Вам следует вызвать функцию push_get_first_local_notification() для получения первого уведомления в очереди, а затем использовать эту функцию для продолжения работы.

ПРИМЕЧАНИЕ: Эта функция ограничена iOS и Android целевыми модулями.

карта DS будет содержать следующие пары ключ/значение:

  • " title" - Заголовок уведомления (он будет доступен только на данной Android платформе, поскольку iOS не хранит информацию о заголовке)
  • "сообщение" - Текст основного текста сообщения
  • " data"- Строка пакета данных

Синтаксис:

push_get_next_local_notification(map);

Аргумент Описание
карта Идентификатор карты DS, который необходимо использовать.

Возвращается:

Реально

Пример

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
var data = ds_map_find_value(map, "data");
if data == "Daily_Reward"
{
push_cancel_local_notification(ntf);
}
ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

Приведенный выше код создает карту DS Map, а затем запрашивает данные для первого доступного уведомления в очереди. Если уведомление существует, карта заполняется, а затем код проверяет ключ "data" на наличие определенной строки. Если строка найдена, уведомление отменяется и переходит к проверке следующего уведомления, пока не будут проверены все уведомления в очереди.

fps_real

В GameMaker Studio 2 есть два основных способа, которые можно использовать для определения скорости, с которой работает ваша игра. скорость игры (как указано в опциях игры) и fps (кадры в секунду). Эти значения часто путают, но в основном одно из них - это количество шагов игры, которое GameMaker Studio 2 должна выполнить за секунду (скорость игры), а другое - количество шагов процессора, которое GameMaker Studio 2 фактически выполняет за секунду (реальный fps), и это значение обычно намного выше, чем скорость игры, но будет снижаться по мере усложнения игры и использования большей вычислительной мощности для поддержания заданной скорости комнаты.

Эта переменная, доступная только для чтения, возвращает текущий fps в виде целочисленного значения. Обратите внимание, что функция будет обновляться только один раз на каждом шаге вашей игры, поэтому может показаться, что значение "прыгает" от одного к другому, но это вполне нормально.

ПРИМЕЧАНИЕ: В HTML5-цели эта переменная вместо этого будет возвращать значение, кратное частоте обновления монитора, поскольку GameMaker Studio 2 должна полагаться на браузер для определения времени и отправки.

Синтаксис:

fps_real

Возвращается:

Реально

Пример:

if debug_mode
{
draw_text(32, 32, "FPS = " + string(fps_real));
}

Приведенный выше код проверит, находится ли игра в режиме отладки, и если да, то отобразит на экране текущий реальный fps.

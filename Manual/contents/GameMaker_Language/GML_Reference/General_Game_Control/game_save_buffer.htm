игровой_сохранённый_буфер

Это вариант функции game_save() поэтому, пожалуйста, сначала прочитайте ее страницу, так как она содержит важную информацию, связанную с ее использованием, а также с использованием этой функции.

С помощью этой функции вы можете сохранить текущее состояние игры в ранее созданный буфер "grow" (см. Буферы), который затем можно снова загрузить в любое время с помощью функции game_load_buffer() функция. Эта функция предназначена для использования в одной комнате за один раз (т.е. не рекомендуется сохранять буфер в одной комнате, а затем пытаться загрузить его из другой) и в идеале должна использоваться только для контрольных точек или перезапуска уровня.

ПРИМЕЧАНИЕ: Эта функция очень ограничена и предназначена для новичков, чтобы быстро запустить систему контрольных точек, но более опытные пользователи могут предпочесть написать свою собственную систему, используя функции File, из-за того, что игра не будет сохранять динамические ресурсы, такие как структуры данных, поверхности, добавленные спрайты и т.д.

Синтаксис:

game_save_buffer(buffer);

Аргумент Описание
буфер Идентификатор буфера для сохранения.

Возвращается:

Н/Д

Пример:

if keyboard_check_pressed(ord("S"))
{
global.Checkpoint = true;
game_save_buffer(save_buff);
}

Приведенный выше код установит глобальную переменную в true, а затем при нажатии клавиши "S" сохранит текущее состояние игры в буфер, указанный в переменной "save_buff".

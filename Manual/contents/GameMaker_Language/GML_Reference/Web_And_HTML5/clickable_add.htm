clickable_add

С помощью этой функции вы можете создать пользовательскую иконку с возможностью клика, которая действует как "кнопка" вне игровой канвы, но в самом окне браузера. Функция возвращает значение id, которое должно быть сохранено в переменной для использования во всех последующих функциях кнопок, которые ссылаются на новую иконку. Эта функция очень полезна для предотвращения создания браузером всплывающего окна при нажатии на кнопку, поскольку все это делается в самой GameMaker Studio 2.

Сначала необходимо задать позицию, которая является относительной к (0,0) окна браузера (левый верхний угол), а затем задать текстуру страницы для спрайта, который должен быть у значка кнопки (см: sprite_get_tpe()). Затем вы можете указать либо URL-адрес для открытия, либо сценарий (из браузера активов GameMaker Studio 2) для открытия или запуска. Параметр "target", который вы указываете, совпадает со стандартным значением JavaScript "name", когда вы при использовании метода open() (имейте в виду, что все значения, кроме '_self', могут привести к блокировке браузера или к тому, что браузер спросит пользователя, разрешить ли ему это), а аргумент "params" совпадает со стандартным параметром JavaScript "specs" для управления свойствами нового окна/вкладки (не все браузеры могут поддерживать все функции). Если вы используете иконку для запуска внутреннего скрипта, эти два последних аргумента можно оставить пустыми строками "".

Действительными целями являются:

Цель Описание
_blank Открывает связанный документ в новом окне или вкладке.
_self Открывает связанный документ в том же фрейме, в котором он был щелкнут (по умолчанию).
_родитель Открывает связанный документ в родительском фрейме.
_top Открывает связанный документ в полном тексте окна.

Доступными параметрами являются:

Параметры Описание
'height=[px]' Высота окна, минимальное значение - 100.
'width=[px]' Ширина окна, минимальное значение - 100.
'left=[px]' Левая позиция окна.
'top=[px]' Верхнее положение окна (только в IE).
'location=[boolean]' Отображать или нет поле адреса (по умолчанию 1).
'menubar=[boolean]' Отображать или нет строку меню (по умолчанию 1).
'resizable=[boolean]' Можно ли изменять размеры окна (по умолчанию 1).
'scrollbars=[boolean]' Отображать или нет полосы прокрутки (по умолчанию 1).
'status=[boolean]' Добавлять или нет строку состояния (по умолчанию 1).
'titlebar=[boolean]' Отображать или нет строку заголовка. Этот параметр игнорируется, если вызывающее приложение не является HTML-приложением или доверенным диалоговым окном (по умолчанию 1);
'toolbar=[boolean]' Отображать или нет панель инструментов браузера (по умолчанию - да).


Стоит также отметить, что если передаваемый вами аргумент URL является именем функции скрипта (т.е. не начинается с "http://") И скрипт называется gmcallback_* то вы можете получить прямую передачу в html5-событие. Имена скриптовых функций GameMaker Studio 2 не обфусцируются, если они начинаются с gmcallback_.

Синтаксис:

clickable_add(x, y, tpe, url, target, params);

Аргумент Описание
x Позиция x в окне.
y Позиция y в окне.
tpe Запись страницы текстуры для используемого спрайта.
url URL (адрес веб-сайта) для ссылки или сценарий, который необходимо запустить.
цель Это целевая область, в которой нужно открыть URL (см. описание).
params Различные параметры для значка (см. описание).

Возвращается:

Реально

Пример:

home_but = clickable_add(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0");

Приведенный выше код создает кликабельную DOM-иконку в позиции (32, 32) страницы, на которой запущен игровой холст. Иконка использует спрайт, указанный на странице текстуры как "spr_MS_Home", и при нажатии на иконку открывается новое окно для указанного URL и с определенными свойствами для этого окна.

os_get_info

Эта функция возвращает Карта DS с подробной информацией об ОС, на которой запущена игра. Точная возвращаемая информация будет зависеть от ОС и устройства, поэтому рекомендуется протестировать эту функцию на всех платформах, на которые вы хотите ориентироваться, и вывести возвращаемые значения с помощью функции типа show_debug_message(). Обратите внимание, что карта DS не автоматически очищается из памяти, и вы должны использовать эту функцию, когда данные больше не нужны. ds_map_destroy() когда вам больше не нужны содержащиеся в ней данные.

Ниже приведены некоторые подробности о дополнительных данных, которые будут доступны к карте DS для следующих платформ. Следует отметить, что на всех платформах, кроме HTML5, карта будет иметь ключ "is64bit". который будет либо true, либо false в зависимости от того, является ли ОС платформы 64-битной или нет.

Windows / UWP

На Windows Desktop и Windows UWP вы получите некоторые дополнительные данные от DX11 со следующими клавишами, доступными в карте DS (полезно для расширений):

  • video_d3d11_device - указатель на устройство DX11
  • video_d3d11_context - указатель на контекст DX11
  • video_adapter_vendorid - строка с идентификатором производителя адаптеров
  • video_adapter_deviceid - строка с идентификатором устройства адаптера
  • video_adapter_subsysid - строка с идентификатором подсистемы адаптера
  • video_adapter_revision - строка с номером ревизии адаптера
  • video_adapter_dedicatedvideomemory - строка с размером видеопамяти
  • video_adapter_dedicatedsystemmemory - строка с размером системной памяти (используется адаптером DX11)

Android

На Android платформе вы получите некоторые дополнительные данные из различных API со следующими ключами, доступными в карте DS:

  • android_tv - возвращает true, если устройство распознано как устройство Android TV, и false в противном случае.
  • GL_VERSION - версия OpenGL, о которой сообщает драйвер
  • GL_VENDOR - поставщик OpenGL, о котором сообщает драйвер
  • GL_RENDERER - рендерер OpenGL, о котором сообщает драйвер
  • GL_EXTENSIONS - расширения OpenGL, которые доступны по данным драйвера
  • GL_SHADING_LANGUAGE_VERSION - версия GLSL, поддерживаемая данным драйвером OpenGL
  • GL_MAX_TEXTURE_SIZE - максимальный размер текстуры, поддерживаемый данным драйвером OpenGL
  • SDK_INT - значение параметра android.os.Build.VERSION.SDK_INT см. здесь для получения дополнительной информации.
  • RELEASE - значение параметра android.os.Build.VERSION.RELEASE смотрите здесь для получения дополнительной информации.
  • MODEL - значение параметра android.os.Build.MODEL см. здесь для получения дополнительной информации.
  • DEVICE - значение android.os.Build.DEVICE см. здесь для получения дополнительной информации.
  • MANUFACTURER - значение android.os.Build.MANUFACTURER смотрите здесь для получения дополнительной информации.
  • CPU_ABI - значение параметра android.os.Build.CPU_ABI, более подробную информацию смотрите здесь.
  • CPU_ABI2 - значение параметра android.os.Build.CPU_ABI2 смотрите здесь для получения дополнительной информации.
  • BOOTLOADER - значение параметра android.os.Build.BOOTLOADER см. здесь для получения дополнительной информации.
  • BOARD - значение параметра android.os.Build.BOARD см. здесь для получения дополнительной информации.
  • VERSION - значение os.version из метода Android System.getProperty
  • REGION - значения user.region из метода Android System.getProperty
  • VERSION_NAME - значение версии данного пакетаName - см. здесь для получения дополнительной информации.
  • PHYSICAL_KEYBOARD - если мы считаем, что физическая клавиатура доступна, то строка "TRUE", иначе "FALSE".

Синтаксис:

os_get_info()

Возвращается:

Real (индекс ds_map)

Пример:

if (os_type == os_android)
{
var _info = os_get_info();

if (_info[? "android_tv"])
{
global.android_tv = true;
}
}

Приведенный выше код проверяет, является ли текущая ОС Android, получает информацию об ОС и считывает возвращенную карту, чтобы проверить, запущена ли игра на Android TV. В этом случае он устанавливает глобальную переменную в true.

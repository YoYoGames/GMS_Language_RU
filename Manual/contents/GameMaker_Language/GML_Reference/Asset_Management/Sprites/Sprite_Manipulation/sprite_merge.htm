sprite_merge

Эта функция объединит спрайт, индексированный в аргументе 1 ("ind2"), со спрайтом, индексированным в аргументе 0 ("ind1"). Сами изображения НЕ объединяются вместе, а скорее объединяются индексы изображений объединяются, при этом суб-изображения из спрайта "ind2" добавляются к изображениям спрайта "ind1", т.е. добавляются в конце. Обратите внимание, что если спрайты разного размера, то добавленные спрайты растягиваются, чтобы соответствовать размеру изображения для "ind1".

Это изменение является постоянным, и с момента использования этой функции и до закрытия игры или удаления спрайта, спрайт, с которым происходит слияние, будет изменен, однако спрайт, с которым происходит слияние, останется прежним.

ПРИМЕЧАНИЕ: Ни ресурс изображения, в который происходит слияние, ни ресурс изображения, с которым происходит слияние, не могут быть одним из исходных ресурсов игры. Вы должны использовать sprite_duplicate() первый для создания двух новых ресурсов (если вы объединяете включенные игровые ресурсы), а затем объединить их.

ПРИМЕЧАНИЕ: Эта функция полезна только для растровых спрайтов и не работает со спрайтами SWF или JSON (Spine).

Синтаксис:

sprite_merge(ind1, ind2);

Аргумент Описание
инд1 Индекс спрайта, который нужно объединить.
инд2 Индекс спрайта, с которым должен быть объединен ind1.

Возвращает

Н/Д

Пример:

var tspr;
tspr[0] = spr_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0);
tspr[1] = sprite_duplicate(spr_Explosion);
sprite_merge(tspr[0], tspr[1]);
sprite_index = t_spr[0];
sprite_delete(tspr[1]);

Приведенный выше код загружает спрайт в локальную переменную, затем объединяет его с другим спрайтом, который был продублирован. Наконец, объединенный спрайт присваивается экземпляру sprite_index, а загруженный спрайт удаляется. чтобы освободить используемую им память. Обратите внимание, что в какой-то момент после этой операции, например, в событиях Destroy или Room End, вам также нужно будет использовать sprite_delete() чтобы удалить объединенный спрайт из памяти.

Генерирование событий объектов

GameMaker Studio 2 полностью ориентирована на события, то есть все действия происходят в результате определенных событий, которые запускаются в определенном порядке или в результате определенного события в игре. Существует множество различных событий, каждое из которых имеет определенную задачу или время, а некоторые из них даже разделены на отдельные "под" события. Обычно эти события запускаются программой GameMaker Studio 2, когда она обнаруживает внутриигровой триггер, или каждый игровой такт. (как в случае с событиями step и draw), но вы также можете применить событие к экземпляру в текущей комнате из части кода.

Существуют следующие функции, работающие с генерацией событий:

  • событие_наследованное
  • событие_исполнения
  • event_perform_object
  • пользователь события

Существует также ряд переменных только для чтения, которые можно использовать с перечисленными выше функциями и которые особенно полезны для отладки:

  • действие_события
  • номер_события
  • объект_события
  • тип события

Более подробную информацию о событиях можно найти в разделе События объекта.

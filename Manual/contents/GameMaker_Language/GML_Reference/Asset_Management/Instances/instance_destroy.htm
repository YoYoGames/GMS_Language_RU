уничтожение экземпляра

Вы вызываете эту функцию всякий раз, когда хотите "уничтожить" экземпляр, обычно вызывая событие Destroy , а также событие Clean Up . Это удалит его из комнаты до тех пор, пока комната не будет перезапущена (если только комната не является постоянной). Вызов функции без аргументов просто уничтожит экземпляр, который в данный момент находится в области видимости и выполняет код, но вы можете указать необязательный аргумент "id" и указать конкретный экземпляр, используя значение экземпляра id, или указать все экземпляры конкретного объекта, используя аргумент object_index. Например:

instance_destroy(other); // уничтожить экземпляр "other" в событии столкновения
instance_destroy(obj_Bullet); // уничтожить ВСЕ экземпляры объекта "obj_Bullet"

Второй необязательный флаг позволяет вам "отключить" событие Destroy Event для уничтожаемого экземпляра. По умолчанию событие Destroy всегда будет выполняться, но если вы установите этот флаг в false, то сможете заставить экземпляр экземпляр уничтожить сам себя и пропустить выполнение этого события.

ВАЖНО! При этом событие Уничтожение будет пропущено, но событие Очистка все равно будет вызвано.

Стоит отметить, что при уничтожении экземпляра его событие destroy вызывается сразу после кода или действия, вызывающего функцию destroy. Кроме того, хотя событие destroy выполняется, экземпляр не сразу удаляется из игры, и он будет продолжать выполнять код, содержащийся в текущем событии. Только после завершения текущего события он будет удален из игры.

Итак, если у вас есть, например, такой код:

f hp <= 0 instance_destroy();
score += 10;

Переменная "score" будет увеличиваться, даже если была вызвана функция instance_destroy, и экземпляр будет окончательно удален из вашей игры в конце события. Будьте внимательны, так как если вы (например) создали динамический ресурс для экземпляра, например, структуру данных, а затем уничтожили этот ресурс в событии destroy, то если после выполнения функции или действия destroy будут какие-либо ссылки на него, вы получите ошибку "unknown ресурс", поскольку событие уничтожения удалило его из игры.

Синтаксис:

instance_destroy([id, execute_event_flag]);

Аргумент Описание
id Идентификатор экземпляра или индекс_объекта для уничтожения (необязательно, по умолчанию - вызывающий экземпляр).
флаг_события Устанавливается в true или false для выполнения события уничтожения или нет (необязательно, по умолчанию true).

Возвращается:

Н/Д

Пример:

if bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height
{
instance_destroy(id, false);
}

Приведенный выше код уничтожит экземпляр, выполняющий код, без вызова события Destroy, если экземпляр находится за границами комнаты.

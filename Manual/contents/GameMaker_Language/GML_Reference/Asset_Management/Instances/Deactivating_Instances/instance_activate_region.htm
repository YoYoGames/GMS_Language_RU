экземпляр_активировать_регион

С помощью этой функции можно определить область в помещении для активации экземпляров, которые ранее были деактивированы. Эта область может быть помечена как "внутри" или "снаружи", как показано на следующем изображении:

На изображении выше видно, что экземпляр "яблоко" всегда активен. потому что, даже если сам спрайт не перекрывает регион, ограничивающая рамка перекрывает его. Таким образом, экземпляры считаются находящимися в указанном регионе, когда их ограничительная область перекрывает его, а состояние маски столкновений (точное или нет) не принимается во внимание. Обратите внимание, что активация не является мгновенной, и экземпляр, активированный таким образом, не будет считаться активным до окончания события, в котором была вызвана функция.

Синтаксис:

instance_activate_region(left, top, width, height, inside);

Аргумент Описание
слева Координата x левой части прямоугольной области для активации.
топ Координата y вершины прямоугольной области, которую нужно активировать.
ширина Ширина области, которую необходимо активировать.
высота Высота области, которую нужно активировать.
внутри Активировать ли экземпляры внутри региона (true) или снаружи (false).

Возвращается:

Н/Д

Пример:

instance_deactivate_all(true);
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Приведенный выше код деактивирует все экземпляры, кроме того, который выполняет код, а затем активирует регион в комнате.

audio_create_play_queue

Эта функция подготавливает буферную очередь для аудио. Вы передаете формат данных (в настоящее время поддерживаются только buffer_u8 или buffer_s16 ), частоту дискретизации (которая может быть от 1000гц до 48000гц), и каналы, которые требуются для аудио из одной из констант, перечисленных ниже:

Постоянная Описание
audio_mono Монофонический (одноканальный) звук.
audio_stereo Стерео (двухканальный) звук.
audio_3D 3D (5.1) аудио.


Обратите внимание, что вы должны освободить все аудио-очереди, созданные с помощью этой функции, когда они больше не нужны, чтобы предотвратить утечку памяти при использовании функции audio_free_play_queue().

Эта функция вызовет асинхронное событие воспроизведения аудио, когда очередь достигнет конца, и в этом событии специальная карта DS будет будет создана в переменной async_load со следующими парами ключ/значение:

  • " queue_id" - очередь, которая закончила воспроизведение данных
  • " buffer_id" - буфер, воспроизведение из которого прекращено
  • " queue_shutdown" - устанавливается в 0 при нормальном воспроизведении и в 1 при получении события, потому что audio_free_play_queue() было вызвано. Когда это значение установлено в 1, вы не ставить в очередь дальнейшие данные.

Синтаксис:

audio_create_play_queue(queueFormat, queueRate, queueChannels);

Аргумент Описание
queueFormat Формат данных буфера для использования(buffer_u8 или buffer_s16).
queueRate Частота дискретизации данных в буферной очереди.
queueChannels Используемые каналы из одной из перечисленных ниже констант.

Возвращается:

Real (идентификатор буфера)

Пример:

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

Приведенный выше код создает новую очередь аудио и присваивает ее ID переменной для дальнейшего использования.

audio_play_sound

С помощью этой функции вы можете воспроизводить любой звуковой ресурс в вашей игре. Вы предоставляете индекс звука и присваиваете ему приоритет, который затем используется для определения того, как звуки будут обрабатываться, когда количество воспроизводимых звуков превысит предел, установленный функцией audio_channel_num(). Звуки с более низким приоритетом будут остановлены в пользу звуков с более высоким приоритетом, а значение приоритета может быть любым реальным числом (фактическое значение произвольно и может быть от 0 до 1 или от 0 до 100, поскольку GameMaker Studio 2 будет расставлять приоритеты одинаково). расставлять приоритеты одинаково). Обратите внимание, что при работе с приоритетами чем больше число, тем выше приоритет, поэтому звук с приоритетом 100 будет предпочтительнее звука с приоритетом приоритетом 1. Последний аргумент предназначен для зацикливания звука, и если установить его в true, то звук будет воспроизводиться до тех пор, пока его не остановят, а если установить его в false, то звук будет воспроизводиться только один раз. только.

Эта функция также возвращает уникальный индексный номер воспроизводимого звука, который затем можно сохранить в переменной, чтобы затем приостановить или остановить его с помощью соответствующих функций. Это означает, что если у вас есть несколько экземпляров одного и того же в любой момент времени вы можете выбрать только один экземпляр этого звука для работы с аудиофункциями.

Синтаксис:

audio_play_sound(index, priority, loop);

Аргумент Описание
индекс Индекс воспроизводимого звука.
приоритет Установите приоритет канала для звука.
петля Устанавливает зацикливание или отсутствие зацикливания звука.

Возвращается:

Индекс (идентификатор звука)

Пример:

если здоровье <= 0
{
lives -= 1;
audio_play_sound(snd_PlayerDead, 10, false);
}

Приведенный выше код проверяет глобальную переменную "health", и если она меньше или равна 0, то удаляет 1 из глобальной переменной "lives" и воспроизводит звук.

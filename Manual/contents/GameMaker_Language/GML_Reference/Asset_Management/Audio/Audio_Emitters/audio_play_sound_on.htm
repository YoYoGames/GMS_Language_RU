audio_play_sound_on

С помощью этой функции вы можете воспроизводить любой звуковой ресурс через эмиттер, причем любые изменения усиления, положения, высоты тона или скорости эмиттера влияют на то, как пользователь слышит конечный воспроизводимый звук. Вы указываете индекс эмиттера для использования, индекс звука звука, который будет воспроизводиться, а затем указываете, должен ли звук зацикливаться или нет. Наконец, вы можете назначить приоритет, который затем используется для определения того, как звуки будут обрабатываться, когда количество воспроизводимых звуков превысит предел, установленный функцией audio_channel_num(). Звуки с более низким приоритетом будут остановлены в пользу звуков с более высоким приоритетом, а значение приоритета может быть любым реальным числом (фактическое значение произвольно и может быть от 0 до 1 или от 0 до 100, так как GameMaker Studio 2 будет расставлять приоритеты одинаково). расставлять приоритеты одинаково). Обратите внимание, что при работе с приоритетами чем больше число, тем выше приоритет, поэтому звук с приоритетом 100 будет предпочтительнее звука с приоритетом приоритет 1.

Эта функция также возвращает уникальный индексный номер воспроизводимого звука, который затем можно сохранить в переменной, чтобы затем приостановить или остановить его с помощью соответствующих функций. Это означает, что если у вас есть несколько экземпляров одного и того же в любой момент времени вы можете выбрать только один экземпляр этого звука для работы с ним с помощью аудиофункций.

Синтаксис:

audio_play_sound_on(emitter, sound, loop, priority);

Аргумент Описание
эмиттер Индекс эмиттера, который необходимо использовать.
звук Индекс используемого звука.
петля Помечает звук как зацикленный или нет.
приоритет Установите приоритет канала для звука.

Возвращается:

Индекс

Пример:

if global.SFX
{
audio_play_sound_on(s_emit, snd_Explode, false, 1);
}

Приведенный выше код проверяет глобальную переменную "SFX", и если она возвращает true, то звук , индексированный в переменной "snd_Explode", будет воспроизведен через эмиттер, индексированный в переменной "s_emit" с низким приоритетом и без зацикливания.

Аудио

GameMaker Studio 2 имеет полноценный звуковой движок, основанный на звуковых форматах *.ogg, *.mp3 и *.wav. Звуки этих типов, добавленные в IDE, могут быть использованы в вашей игре с помощью основных аудиофункций, показанных ниже. Для более сложных вещей, чем базовые звуковые эффекты или воспроизведение одного музыкального фрагмента, вы можете обратиться к расширенным аудиофункциям, которые позволяют изменить способ воспроизведения звука. Есть также ряд более специализированных функций, предназначенных для потоковой передачи аудио, позиционированию аудио - для получения 3D-звука - и группировке аудио.

ВАЖНО! При использовании звука в HTML5-цели следует помнить, что не все браузеры поддерживают Web Audio, поэтому при запуске ваш проект может не воспроизводить звук. Вы можете получить общее представление о поддержке Web Audio по следующей ссылке: Могу ли я использовать Web Audio?

  • аудио_пауза_звук
  • audio_pause_all

Веб-аудио

При создании игр для целевой платформы HTML5 звуковой движок требует поддержки Web Audio, а это, в свою очередь, означает, что иногда звук не будет воспроизводиться, когда или как вы ожидаете. Это происходит потому, что контекст Web Audio может быть не запущен или может перестать работать во время воспроизведения игры. Причины этого сильно различаются в разных браузерах и даже в разных версиях одного и того же браузера, поэтому обнаружение изменения состояния контекста веб-аудио очень важно, например: для обнаружения когда контекст меняется и приостановить/запустить зацикленное аудио, например, фоновую музыку.

Чтобы помочь решить эту проблему, в GameMaker Studio 2 есть два отдельных способа обнаружения изменения состояния контекста Web Audio, либо с помощью следующей функции:

  • audio_system_is_available

Вы также можете использовать асинхронное системное событие, которое будет срабатывать всякий раз, когда изменяется статус Web Audio. В этом событии вы получите встроенную async_load DS map, заполненную ключом " event_type", который в свою очередь будет содержать строку " audio_system_status", если это аудиособытие. Когда этот ключ существует, также будет существовать дополнительныйключ"status", который будет будет либо "доступен", либо"недоступен". Обратите внимание, что это событие будет срабатывать на всех платформах, но на всех, кроме HTML5, оно будет срабатывать только один раз при запуске игры, когда аудиодвижок будет впервые инициализируется.

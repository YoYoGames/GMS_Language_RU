ds_priority_read

С помощью этой функции вы можете воссоздать сохраненный приоритет DS (тот, который ранее был записан в виде строки с помощью функции ds_priority_write()). Сначала вы должны создать новый приоритет DS для чтения строки, а если DS приоритет уже существует и в нем хранится информация, то она будет очищена перед считыванием. Эта функция имеет жизненно важное значение при создании механизмов сохранения/загрузки для вашей игры. Обратите внимание, что если структура данных была создана в предыдущих версии GameMaker вам следует добавить необязательный аргумент "legacy", установив его в true, так как формат строки в этой версии изменился.

Синтаксис:

ds_priority_read(id, str [, legacy]);

Аргумент Описание
id Идентификатор структуры данных для записи.
str Строка, с которой будет производиться запись.
наследие(необязательно) Может быть как истинным, так и ложным или полностью отсутствовать.

Возвращается:

Н/Д

Пример:

p_queue = ds_priority_create();
ini_open("save.ini");
var str = ini_read_string("P_Queues", "0", "");
if (str != "")
{
ds_priority_read(p_queue, str);
}
ini_close();

Приведенный выше код создает приоритетную очередь и сохраняет индекс в переменной "p_queue". Затем он открывает ini-файл и считывает из него строку, проверяя, что сначала строка не возвращается пустой. Затем эта строка считывается во вновь созданный приоритет DS.

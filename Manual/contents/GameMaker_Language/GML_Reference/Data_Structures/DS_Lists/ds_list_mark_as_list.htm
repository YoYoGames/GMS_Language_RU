ds_list_mark_as_list

Эта функция "пометит" (или "отметит") заданную позицию в ранее созданном списке DS как содержащую другой список DS. Эта функция необходима при кодировании строк JSON (которые вы можете создать с помощью функции json_encode()), но также может быть полезна при вложении структур данных, поскольку помеченные таким образом элементы будут автоматически собираться (уничтожаться) при уничтожении родительского списка DS. Это означает, что вам не нужно вручную просматривать содержимое списка и уничтожать помеченные структуры данных по отдельности перед уничтожением "родительского" списка. Однако, если вы удалите позицию списка по отдельности, содержащийся в ней список DS не будет собран в мусор, и вам следует вызвать соответствующую функцию DS list destroy перед удалением элемента из позиции родительского списка. Также обратите внимание, что если вы вызываете функцию ds_list_clear() на списке, все элементы, помеченные как списки, также будут уничтожены, когда список будет очищен.

Также стоит отметить, что если позиция, которая была отмечена, меняется в списке, "метка" перемещается вместе с ней, поэтому если вы отметили позицию 3 в списке (например), а затем вставили еще 2 элемента под ней, чтобы она переместилась на позицию 5, он все равно будет помечен как список.

Синтаксис:

ds_list_mark_as_list(id, pos);

Аргумент Описание
id Идентификатор списка, который нужно отметить.
поз Позиция в списке, которую нужно отметить.

Возвращается:

Real (целое число - ID списка DS в случае успеха или -1 в случае неудачи)

Пример:

var j_list = ds_list_create();
var sub_list = ds_list_create();
ds_list_add(sub_list, health);
ds_list_add(sub_list, lives);
ds_list_add(sub_list, score);
ds_list_add(j_list, sub_list);
ds_list_mark_as_list(j_list, 0);

Приведенный выше код создает два списка DS, затем заполняет один из них различными значениями из глобальных переменных. Затем этот список добавляется во второй список, а позиция "помечается" как таковая, чтобы впоследствии ее можно было правильно закодировать.

ds_stack_read

С помощью этой функции можно воссоздать сохраненный DS-стек (тот, который ранее был записан в виде строки с помощью функции ds_stack_write()). Сначала необходимо создать новый DS-стек для чтения строки, и если DS-стек уже существует и в нем хранится информация, то она будет очищена перед чтением. Эта функция имеет жизненно важное значение при создании механизмов сохранения/загрузки для вашей игры. Обратите внимание, что если структура данных была создана с помощью предыдущих версий программы GameMaker вам следует добавить необязательный аргумент "legacy", установив его в true, поскольку формат строк в этой версии изменился.

Синтаксис:

ds_stack_read(id, str [, legacy] );

Аргумент Описание
id Идентификатор структуры данных для чтения.
str Строка для чтения.
наследие(необязательно) Может быть как истинным, так и ложным или полностью отсутствовать.

Возвращается:

Н/Д

Пример:

stack = ds_stack_create();
ini_open("save.ini");
var str = ini_read_string("Stacks", "0", "");
if (str != "")
{
ds_stack_read(stack, str);
}
ini_close();

Приведенный выше код создает стек и сохраняет индекс в переменной "stack". Затем он открывает ini-файл и считывает из него строку, проверяя, что сначала строка не возвращается пустой. Затем эта строка считывается во только что созданный стек DS.

Назад: ds_stack_create
Далее: ds_stack_write

рабочий_каталог

Working_directory может возвращать два разных значения в зависимости от того, для чего вы его используете. Если вы записываете файл на диск, working_directory указывает на область локального хранилища, зарезервированную для вашей игры на целевом устройстве (в windows это " ", где "gamedir" - каталог с именем вашей игры). (в windows это "%LOCALAPPDATA%/gamedir/", где "gamedir" - это каталог с именем вашей игры). Однако, при чтении с диска, working_directory может быть как локальным хранилищем , так и областью, где хранятся включенные файлы.

ПРИМЕЧАНИЕ: Переменная working_directory возвращает путь, включая последний обратный слеш.

Например, если вы включили в игру стандартный файл *.txt и прочитали его в память с помощью working_directory, то этот файл будет взят из области, где хранятся включенные файлы. Если вы затем решите записать эту информацию в текстовый файл с помощью working_directory, это приведет к записи файла в локальное хранилище целевой платформы, откуда он будет считан в следующий раз, если только вы не прикажете GameMaker Studio 2 удалить файл, и в этом случае working_directory снова будет указывать на область, где хранятся включенные файлы.

Стоит отметить, что в общем случае вам не нужно использовать это и достаточно просто назвать имя файла (плюс любая дополнительная информация о пути), например, так:

file_text_open_read(working_directory + "my_file.txt");

было бы лучше сделать так:

file_text_open_read("my_file.txt");

ВНИМАНИЕ! Эта функция может работать не так, как вы ожидаете, из-за того, что GameMaker Studio 2 находится в "песочнице"! Пожалуйста, обратитесь к разделу о файловой системе для получения дополнительной информации.

Пример:

ini_open(working_directory + "temp_ini.ini");

Это откроет ini-файл из рабочего каталога игры (создав его, если он еще не существует). Это может быть локальное хранилище или область, где находятся включенные файлы, в зависимости от того, существует ли искомый файл в любом из этих мест.

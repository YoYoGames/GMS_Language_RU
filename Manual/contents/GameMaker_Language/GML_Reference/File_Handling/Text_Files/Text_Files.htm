Текстовые файлы

Текстовые файлы - это полезный способ хранения больших объемов данных вне игры. В отличие от ini файлов, вы можете иметь до 32 текстовых файлов, открытых одновременно, и вы можете читать и записывать в них большие куски данных. Однако вы сами должны решать, как эти данные должны быть структурированы, поскольку не существует установленной структуры "секций" или "ключей", как в *.ini файлах.

Обратите внимание, что для игр с локализациями очень важно, чтобы акцентированные буквы могли быть прочитаны из внешних файлов. Это означает, что вам следует сначала создать txt-файл в формате UTF8 и затем добавить его в GameMaker Studio 2 в качестве включенного файла, чтобы он экспортировался при запуске игры и использовался вместо стандартного txt-файла в формате ANSI, который создается файловыми функциями GameMaker Studio 2, когда файл не предоставлен. если файл не был предоставлен ранее. Таким образом, вы можете читать и писать в нем правильно, с сохранением всех ударений и нелатинских букв.

Существуют следующие функции, работающие с файлами:

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • строка_текста_записи_файла
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • файл_текст_закрыть

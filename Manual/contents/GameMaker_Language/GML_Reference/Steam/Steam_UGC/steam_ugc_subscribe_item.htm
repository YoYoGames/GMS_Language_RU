steam_ugc_subscribe_item

Эта функция может быть использована для подписки на элемент пользовательского контента. Вы предоставляете ID опубликованного файла, а затем функция возвращает уникальное значение async ID, которое можно использовать для проверки того, когда подписанный элемент был загружен в событииSteam Async. Это событие будет иметь async_load DS map, которая будет заполнена следующими парами ключ/значение:

  • " id" - Идентификатор async, возвращаемый функцией, которая вызвала событие
  • "result" - Результат операции (вещественное значение). Это будет либо GML-константа ugc_result_success, либо какое-то другое вещественное число. Поэтому вы должны проверить наличие этой константы, чтобы убедиться, что вызов был успешным, и если в противном случае что-то сработало неправильно. Остальные возможные возвращаемые значения отображаются как результат значения Steam "EResult", и вы должны увидеть steamclientpublic .h в заголовках SDK для всех 89 возможных значений.
  • " event_type" - Строка " ugc_subscribe_item".
  • "published_file_id" - Уникальный идентификатор для опубликованного элемента.

Синтаксис:

steam_ugc_subscribe_item(published_file_id);

Аргумент Описание
опубликованный_файл_ид Уникальный идентификатор файла для UGC, на который нужно подписаться.

Возвращается:

Асинхронный идентификатор

Пример:

steam_sub= steam_ugc_subscribe_item(global.pubFileID);

Приведенный выше код подпишется (и загрузит) элемент с идентификатором файла, хранящимся в глобальной переменной "pubFileID".

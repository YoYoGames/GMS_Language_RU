буферный_пик

С помощью функции buffer_read() вы можете читать данные из заданного буфера в текущей позиции "seek", при этом каждая часть данных читается, продвигаясь вперед по этой позиции на считанные или записанные байты. Однако может возникнуть необходимость прочитать определенный фрагмент данных без изменения текущей позиции поиска, и тогда вы можете использовать эту функцию. Вы просто сообщаете функции идентификатор буфера, а затем позицию смещения (от начала буфера) в буфере. начала буфера) внутри этого буфера для чтения, а также тип данных, которые вы хотите прочитать.

ПРИМЕЧАНИЕ: Использование неправильного типа данных для считываемых данных приведет к получению ошибочных значений!

При успешном выполнении функции будет возвращено значение (Real, string, int и т.д...) в зависимости от типа данных, в то время как при неудаче будет возвращен просто 0.

Синтаксис:

buffer_peek(buffer, offset, type);

Аргумент Описание
буфер Индекс используемого буфера.
смещение Позиция смещения (в байтах) в буфере для считывания заданных данных.
тип Тип данных, которые должны быть считаны из буфера (см. список констант здесь).

Возвращается:

Real/String или 0 в случае неудачи.

Пример:

var red = buffer_peek(buff, 1, buffer_u8);
var green = buffer_peek(buff, 2, buffer_u8);
var blue = buffer_peek(buff, 3, buffer_u8);
image_blend = make_colour_rgb(red, green, blue);

Приведенный выше код получит три значения из трех различных позиций в буфере, индексированных в переменной "buff", а затем использует эти значения для установки смешивания изображения экземпляра.

buffer_async_group_begin

Эта функция вызывается, когда вы хотите начать сохранение из нескольких буферов в несколько файлов. Имя группы" - это строка, которая будет использоваться как имя каталога, куда будут сохранены файлы, и должна использоваться как часть пути к файлу при загрузке файлов обратно в систему. при последующей загрузке файлов обратно в IDE (с помощью любой из buffer_load() функции). Эта функция предназначена только для использования с buffer_save_async() функция и вы также должны завершить определение сохранения вызовом функции buffer_async_group_end() иначе файлы не будут сохранены.

Обратите внимание, что для консольных платформ (например, PS4) "имя группы" будет использоваться в качестве описания слота сохранения, и использование этой функции поможет вам избежать отображения пользовательского интерфейса для каждого сохраняемого файла. Также обратите внимание, что при использовании UWP вы можете перемещать свои сохранения игр между родными окнами магазинов XBox и Microsoft, создав группу, которая будет такой же, как группа ERA Xbox, например:

if os_type == os_uwp
{
buffer_async_group_begin("root/yourgroupname");
}
else
{
buffer_async_group_begin("yourgroupname");
}

Синтаксис:

buffer_async_group_begin(groupname);

Аргумент Описание
имя группы Имя группы (в виде строки).

Возвращается:

Н/Д

Пример:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

Приведенный выше код запускает буферную группу, затем устанавливает ее для асинхронного сохранения 4 файлов. Затем определение группы завершается (в этот момент начнется сохранение).

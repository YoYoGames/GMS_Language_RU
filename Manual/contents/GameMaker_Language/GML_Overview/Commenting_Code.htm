Код комментирования

При совместной работе с другими людьми, при работе с большим проектом или даже просто для собственного ознакомления и отладки очень важно оставлять комментарии в коде событий и скриптах. В GameMaker Studio 2 у вас есть различные механизмы. позволяющие оставлять заметки и комментарии к участкам кода и даже целым блокам кода, которые видны в списке событий объекта, который вы редактируете в данный момент. Таким образом, вы можете оставлять заметки для себя и своих коллег, или объяснять или объяснить особенно сложный кусок кода, или даже просто оставить напоминание о том, что что-то делает, или о том, что вы хотите, чтобы что-то делалось в будущем.

Чтобы оставить простой однострочный комментарий, первое, что вы можете сделать, это использовать // перед текстом. Например:

//инициализация переменных
sr = 10;
hp = 100;

Вы также можете оставлять многострочные комментарии, чтобы отдать должное, опустить целый раздел кода для отладки или даже оставить заметки о том, как вы хотите изменить код в будущем. Для этого вы можете использовать /* ... */ например:

/*
Установите переменные GIF в начальные значения (их можно редактировать, поэтому вам не нужно постоянно использовать функции setter, если вы всегда используете одни и те же значения)

ВАЖНО! Пожалуйста, смотрите отдельные скрипты "set" для более подробной информации о каждой из переменных.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

Помимо комментариев вы также можете использовать специальные теги #region, чтобы отметить участок кода для Code Folding, и каждый регион может содержать комментарий. Это работает следующим образом: вы добавляете тег #region - вместе с любым текстом комментария - в вверху участка кода, а затем добавляете соответствующий тег #endregion внизу участка и затем вы можете "свернуть" код, т.е. спрятать его, чтобы сократить количество строк кода, видимых в редакторе в любой момент времени. один раз:

Фактический код выглядит примерно так:

#region Это пример региона

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Дальнейшие комментарии здесь, если необходимо.

Наконец, стоит упомянуть, что при работе с событиями в редакторе объектов можно использовать специальный комментарий "description" в начале события, чтобы сказать. общее содержание события, и оно будет отображаться в окне событий редактора объектов. Этот комментарий должен располагаться в самой первой строке редактора кода и имеет следующий формат:

/// @description <This Is An Event Description>.

Наконец, если вы пишете скриптовые функции и хотите иметь некоторый контроль над тем, как они обрабатываются IDE (например, иметь автозаполнение или показывать параметры скрипта), то вам следует писать комментарии, используя нотацию JSDoc, как объяснено здесь.

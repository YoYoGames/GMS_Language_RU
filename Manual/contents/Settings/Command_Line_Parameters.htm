Параметры командной строки

При создании игр очень важно протестировать исполняемый файл, чтобы убедиться, что окончательная компиляция корректна и не содержит непредвиденных ошибок или различий. Однако создание исполняемого файла означает, что, поскольку у вас нет доступа ни к отладочной консоли, ни к к окну компилятора, может быть трудно определить и отладить любые проблемы, которые вы можете найти. Однако для этого есть решение - использовать параметры командной строки для установки определенных свойств при запуске исполняемой версии игры. Это позволяет создавать файлы выходных журналов и тестировать другие специальные ситуации.

ПРИМЕЧАНИЕ: Это доступно только на стандартном Windows только на стандартном целевом модуле.

Чтобы это заработало, сначала нужно создать исполняемый файл игры для Windows и установить его на тестовой машине. После этого необходимо создать ярлык игры на рабочем столе. Именно здесь мы будем добавлять параметры для передачи на *.exe (чтобы создать ярлык, щелкните правой кнопкой мыши на исполняемом файле игры и выберите Отправить на > Рабочий стол (создать ярлык)) .

После создания ярлыка вы можете использовать любой из следующих параметров в целевом пути:

-noaudio: Это отключает все звуки в вашей игре.

-inawindow: Заставляет игру запускаться в окне, даже если она запущена в полноэкранном режиме.

-output <filename>: Отправляет консольный вывод на имя файла, включая любые отладочные сообщения, которые есть в вашей кодовой базе.

-debugoutput <filename>: Отправляет консольный вывод в указанный файл, исключая любые пользовательские отладочные сообщения, но включая дополнительную информацию от бегуна для сообщения об ошибке.

-software: Заставляет игру использовать программную обработку вершин вместо аппаратной для рендеринга игровой графики. Это особенно полезно для тех компьютеров, которые используют встроенный чипсет Intel GFX или демонстрируют проблемы с дисплеем на старых машинах.

-intel: Это включит исправление для плохих драйверов Intel, независимо от того, какой GPU используется. Это решение проблемы плохих драйверов замедлит работу, что приведет к заметному снижению производительности игры, поэтому если вам не нужно вам не нужно исправление, не используйте его.

-vanillaGFX: Использование этого параметра отключает любую проверку производителя драйвера и может быть очень полезно для тестирования вашей игры "как есть".

Стоит отметить, что <filename> в -output и -debugoutput при желании может быть одним и тем же, так как файл открывается и закрывается при каждой записи.

Теперь вы должны перейти к тому месту, где вы создали ярлык исполняемого файла игры, а затем щелкнуть на нем правой кнопкой мыши и выбрать пункт "Свойства".

Откроется окно, аналогичное приведенному выше. Здесь вы можете добавить в Target path вашу командную строку. Просто прокрутите до конца заданного пути, добавьте пробел, а затем строку параметров. Так, например, для создания вывода файла отладки вы можете сделать следующее:

D:\Users\Me\GameMaker\CustomLoadBar-Default-1.0.0.0\CustomLoadBar.exe -debugoutput debug.txt

В результате будет создан файл под названием "debug.txt" в том же месте, что и исполняемый файл (не в месте расположения ярлыка, а там, где установлена игра) с отладочной информацией и информацией компилятора из вашей готовой игры. Нажмите на "OK" и затем используйте ярлык для запуска вашей игры, чтобы она использовала эти параметры и помогла вам с отладкой.

Более подробную информацию о том, как отладить вашу игру, вы найдете в разделе руководства " Отладка".

А/Б тестирование с помощью командной строки

YoYo runner также получает преимущества от специального параметра командной строки:

-game <filename>: загружает заданный файл игры

Зачем вам это нужно? Обычно вы этого не делаете, но если вы хотите иметь несколько версий вашей игры, запущенных одновременно, это можно сделать с помощью этого метода, что значительно упрощает сравнительные тесты A/B. С помощью этого метода вы можете провести фокус-тест с кем-то или, как дизайнер, получить лучшее представление о вещах, проводя тесты бок о бок.

Чтобы настроить его, необходимо знать несколько вещей. Для начала вам понадобится путь к самому Runner. Его можно найти в папке Program Data вашей ОС Windows (по умолчанию она может быть скрыта, так что вам чтобы увидеть ее, необходимо изменить свойства просмотра), с типичным форматом:

C:\ProgramData\GameMakerStudio2\Cache\runtimes\< runtime_version>\windows\Runner.exe

Обратите внимание, что папка runtime_version будет зависеть от установленных у вас в данный момент времен выполнения. Вам также понадобится путь к файлу <yourgame>.win, поскольку именно на него мы будем указывать бегунку. Самый простой способ найти его - это зайти в папку Temp (обычно она находится по адресу %localappdata%\Temp\GameMakerStudio\<имя_игры>\) и просто найти *.win, так как это покажет вам все имеющиеся файлы, и вы сможете найти их. покажет все доступные папки, и вы можете просто выбрать ту, к которой вам нужен полный путь.

Теперь мы можем создать ярлык бегущей строки, как мы это сделали для исполняемого файла, а затем изменять параметры по мере необходимости, но это означает необходимость открывать и редактировать ее между тестами, что сводит на нет цель использования этой функциональности для параллельного тестирования. тестирования. Вместо этого мы создадим пару маленьких "пакетных" файлов, чтобы справиться с этой задачей.

Для этого сначала создайте новый текстовый документ на рабочем столе и откройте его в Блокноте (или любом другом текстовом редакторе). Добавьте следующее:

START C:\ProgramData\GameMakerStudio2\Cache\runtimes\< runtime_version>\windows\Runner.exe -game <LOCAL TEMP PATH>\<GAMENAME>.win

Вам нужно установить временный путь к файлу игры *.win, а затем сохранить его как "GM_Test_1.bat" на рабочем столе (имейте в виду, что Блокнот автоматически сохранит файл *.txt, если вы не выберете "все файлы" в опциях диалога сохранения). Пример того, как должна выглядеть ваша конечная команда:

START C:\ProgramData\GameMakerStudio2\Cache\runtimes\runtime-9.9.1.199\windows\Runner.exe -game D:\Users\Mark\AppData\Local\Temp\GameMakerStudio2\AlphaDog_Reboot\AlphaDog_Reboot.win

Теперь создайте другой текстовый файл и добавьте ту же строку, только на этот раз укажите путь ко второй версии игры, с которой вы хотите провести тестирование бок о бок, и сохраните его как "GM_Test_2.bat". Вы можете дважды щелкнуть на любом из них, чтобы открыть различные версии вашей игры, и Наличие пакетных файлов означает, что вы можете легко открыть их снова, чтобы отредактировать и изменить тестируемую игру.

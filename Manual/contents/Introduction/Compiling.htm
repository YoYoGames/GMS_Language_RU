Компиляция

Компиляция вашей игры может означать одно из двух: компиляция для тестирования или компиляция для создания игры в формате исполняемый пакет или конкретной целевой платформы. Компиляция игры для тестирования может быть выполнена простым нажатием кнопки Play в верхней части IDE, что приведет к запуску игры для тестирования используя указанную цель. Вы также можете запустить игру в режиме отладки, протестировав ее с помощью кнопки Debug . При этом игра будет запущена, но также откроется Окно отладки, в котором можно проследить за работой игры и увидеть любые проблемы (более подробную информацию см. в разделе " Отладка" ).

По умолчанию GameMaker Studio 2 запускается и отлаживается с помощью встроенной виртуальной машины (VM), что более или менее аналогично запуску на используемой настольной ОС. Однако GameMaker Studio 2 является кроссплатформенным движком, и вы можете тестировать, отлаживать и компилировать исполняемые пакеты ваших проектов на ряде различных целевых платформ (конкретные доступные платформы зависят от деталей вашей лицензии). Чтобы изменить текущую целевую платформу, вы можете щелкнуть на кнопке Targets , чтобы открыть окно Targets Window, которое будет выглядеть примерно так (точные детали зависят от типа вашей лицензии):

В верхней части окна, рядом с кнопкой "Цели", отображаются текущие настройки, которые сообщают вам о платформе и конкретных настройках, используемых в данный момент, а затем остальная часть окна занята деталями и опциями для всех доступных целей, которые вы можете выбрать вместо них. которые вы можете выбрать для использования вместо них. Выход

Каждая целевая платформа может иметь один или несколько форматов вывода, основными из которых являются:

  • VM: Цель VM (Virtual Machine) использует общий запуск для каждой платформы, а затем интерпретирует код для вашей игры. Обычно этот вариант используется для тестирования из-за более быстрого времени сборки, но он не дает такого такого прироста производительности, как при использовании опции YYC (если она доступна). Вы можете использовать его для компиляции исполняемых пакетов для небольших игр или игр, в которых производительность не является проблемой.
  • YYC: YYC(YoYo Compiler) берет обычный выход GameMaker Studio 2 и компилирует его в родной код для целевой платформы, "вычеркивая" ненужные функции и выполняя множество других методов оптимизации. для создания более компактного и производительного исполняемого пакета. Это может повысить производительность ваших игр как минимум в два-три раза, особенно игр с большим количеством логики, и идеально подходит для крупных проектов с интенсивным использованием процессора. Время компиляции может и всегда следует очищать кэш компилятора перед сборкой окончательного полного пакета активов для целевой платформы. Обратите внимание, что YYC-цель может потребовать установки дополнительных инструментов для выбранной платформы, иначе она не будет работать. Вы можете найти дополнительную информацию об этом в Справочном центреYoYo Games, а также на страницах настроек отдельных целей. подробно описанных в данном руководстве.
  • JavaScript: Цель JavaScript будет доступна только для некоторых целей, таких как цель HTML5, и задает компиляцию игры в чистый JavaScript.

Создание конечного исполняемого пакета

Компиляция вашего игрового проекта для любой целевой платформы потребует, чтобы вы установили правильные настройки инструменты сборки (см. здесь) , а также заполнить соответствующие Platform Preferences. После этого вы можете просто нажать кнопку Create Executable в IDE, чтобы начать сборку компилятора, или выбрать Create Executable из меню Build Menu. Любой из вариантов откроет окно проводника файлов, где вы можете дать окончательное имя, которое для исполняемого файла игры, после чего нажмите кнопку Save , чтобы запустить процесс компиляции и сборки. После этого будут созданы необходимые файлы, которые вы сможете распространять по своему усмотрению.

ВАЖНО! Если у вас бесплатная 30-дневная пробная лицензия, то компиляция конечного исполняемого файла будет недоступна, и вам придется перейти на одну из платных лицензий, чтобы скомпилировать вашу игру для распространения.

ПРИМЕЧАНИЕ: Перед окончательной сборкой проекта для релиза всегда следует очистить кэш компилятора ассетов (с помощью значка "метлы" в верхней части IDE). GameMaker Studio 2 кэширует многие игровые файлы, чтобы снизить скорость компиляции до минимума.

Каждый целевой параметр сохраняется в формате, характерном для конкретной платформы, перечисленных ниже:

  • Windows - При компиляции для общей ОС Windows вам сначала будет предложено выбрать между созданием инсталлятора и Zip-пакета , где инсталлятор будет представлять собой один исполняемый файл, который установит вашу игру, а Zip-файл будет представлять собой один сжатый файл формата со всеми файлами игры, хранящимися внутри (файлы нужно будет извлечь). Zip-файл будет представлять собой один сжатый файл формата *.zip со всеми файлами вашей игры, хранящимися внутри (для запуска игры файлы необходимо извлечь). Если вы отметите флажок Remember Packaging Option, то GameMaker Studio 2 запомнит этот выбор для всех последующих компиляций (его можно сбросить или изменить в Windows Preferences). Более подробную информацию вы можете найти в Справочном центреYoYo Games. Обратите внимание, что Windows-цель может компилировать 32-разрядные или 64-разрядные исполняемые файлы в зависимости от того, что вы выбрали в параметре общих параметрах игры. Мы настоятельно рекомендуем всем новым проектам использовать x64 Runtime для создания 64-битных исполняемых файлов, так как 32-битная среда выполнения будет устаревать и удаляться. в будущем.
  • Ubuntu (Linux ) - Приложения для Ubuntu(Linux) могут быть скомпилированы в виде файлов *.zip, содержащих исполняемый файл x64 или ARM - в зависимости от устройства, на которое они нацелены. Более подробную информацию вы можете получить в Справочном центреYoYo Games.
  • HTML5 - Если вы выбрали сборку HTML5, то файл index.html (это имя по умолчанию, но вы можете дать собственное имя в Настройках платформы) вместе с папкой, содержащей файлы вашей игры будут созданы и сохранены в указанном месте. Для того чтобы ваша игра работала, необходимо, чтобы оба эти файла были загружены на сервер. Сайт index.html также может быть настроен чтобы показать вашу игру с другим цветом фона, или в другой позиции и т.д... но для этого необходимо знание HTML, и вы также можете указать свой собственный индексный файл при сборке пакета (см. Опции игры HTML5). Вы можете узнать больше в Справочном центреYoYo Games.
  • Android - Для устройств Android вы можете выбрать файл *.apk или *.aab (Android App Bundle) в окне, которое отображается при сохранении игры:

    Тип выбранного вами файла зависит от магазина, на который вы хотите ориентироваться: файл *.aab требуется для Google Play, а файл *.apk можно использовать в других магазинах. Более подробную информацию вы можете получить в Справочном центреYoYo Games.
  • iOS - При компиляции для iOS создается файл xarchive, который затем используется в Xtools для создания конечного пакета iOS. Обратите внимание, что для компиляции под iOS вам потребуется компьютер Apple Mac под управлением OSX или выше, а также соответствующие сертификаты и разрешения. Более подробную информацию вы можете получить в Справочном центреYoYo Games.
  • macOS - При компиляции для macOS сначала будет предложено выбрать между созданием DMG и Zip-файла :
    "Package as DMG" создаст пакет установщика .dmg, который можно использовать для установки игры на компьютере Mac. компьютере Mac. Обратите внимание, что в процессе сборки на Mac появится программа установки, которая исчезнет после завершения процесса сборки.

    "Package as Zip" создаст либо файл *.app, либо файл *.pkg, в зависимости от того, хотите ли вы впоследствии загрузить игру в Mac App Store или нет. Как и в случае с iOS, вам потребуется компьютер Apple Mac под управлением OSX или выше, а также соответствующие сертификаты и разрешения. Более подробную информацию вы можете получить в Справочном центреYoYo Games.
  • Windows UWP - Для Windows UWP GameMaker Studio 2 создаст пакет *.appxbundle, который затем можно загрузить в Windows Store или распространять в других местах. Когда вы нажмете кнопку Создать исполняемый файл вам будет предложено указать GameMaker Studio 2, какой тип пакета вы хотите создать (может быть для архитектур ARM, x64 и/или x86), и вам следует выбрать те цели, которые на которых будет поддерживаться игра (вы должны выбрать хотя бы одну, но можете выбрать все три или любую комбинацию):

    Если вы отметите флажок Package For Store Upload, то окончательно созданный пакет будет представлять собой файл *.appxupload, который, как указано Microsoft, должен использоваться для отправки приложений в их магазин, как объясняется в этой статье здесь (все выбранные архитектуры будут включены, как и в случае с *.appxbundle). Вы можете узнать больше о настройке и компиляции под платформу UWP в Справочном центреYoYo Games.

После создания исполняемого пакета активов вы можете передать файл другим людям или разместить его на своем сайте для скачивания, или загрузить эти файлы на различные хостинги для индивидуального распространения, или даже в интернет-магазины (такие как Google Play, iTunes или MS Store) для общего распространения и розничной продажи.

Обратите внимание, что вы можете свободно распространять игры, созданные с помощью GameMaker Studio 2, любым способом, включая их продажу. Конечно, это предполагает, что спрайты, изображения и звуки, которые вы использовали для создания игры, также могут быть распространены или проданы и что у вас есть законные права на все активы, а также предполагается, что игра соответствует ЛСКПYoYo Games для GameMaker Studio 2.

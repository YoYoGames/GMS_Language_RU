Языковые предпочтения

Параметры языка используются для определения того, как некоторые функции языка программирования будут отображаться в IDE GameMaker Studio 2. Опции для GML следующие:

  • Отображать типы аргументов в строке состояния: Эта опция относится к случаям, когда вы используете собственные пользовательские сценарии и используете нотацию JSDoc для идентификации различных аргументов и т.д... (см. здесь для более подробную информацию). Если эта опция отмечена, и вы указали типы аргументов в сценарии, то они будут показаны в строке состояния в нижней части редактора, заключенные в {}. По умолчанию эта опция выключена.
  • Отображать описания аргументов в строке состояния: Если этот флажок установлен, то любые описания, которые вы дали своим пользовательским скриптам, используя нотацию JSDoc, будут отображаться в строке состояния в нижней части окна редактора. По умолчанию эта опция выключена.
  • Показывать американское написание для системных функций: По умолчанию GameMaker Studio 2 использует английское написание функций, например, draw_text_colour(), однако это может быть не совсем то, к чему вы привыкли, и поэтому вы можете переключиться на американское написание, используя эту опцию. можно переключиться на американское написание с помощью этой опции, чтобы функции стали, например, draw_text_color(). По умолчанию эта опция выключена.
  • Максимальное количество отображаемых синтаксических ошибок: Это значение устанавливает максимальное количество синтаксических ошибок, которые будут отображаться в любой момент времени. Если вы обнаружите, что IDE отстает, попробуйте уменьшить это значение. По умолчанию это значение равно 50.
  • Включить предупреждения о ссылках на переменные: Если эта функция включена, она будет отмечать любые потенциальные проблемы с переменными, например, когда переменная определена где-то, но фактически не используется в коде. При отключении эти предупреждения больше не будут показываться. По умолчанию эта функция включена.
